<h1 class="firstHeading mw-first-heading" id="firstHeading"><span class="mw-page-title-main">Турбокод</span></h1>
<div class="vector-body" id="bodyContent">
<div class="noprint" id="siteSub">Материал из Википедии — свободной энциклопедии</div>
<div id="contentSub"><div id="mw-content-subtitle"><span class="mw-redirectedfrom">(перенаправлено с «<a class="mw-redirect" href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE-%D0%BA%D0%BE%D0%B4&amp;redirect=no" title="Турбо-код">Турбо-код</a>»)</span><br/>

<div class="mw-body-content" id="mw-content-text"><div class="mw-content-ltr mw-parser-output" dir="ltr" lang="ru"><p><b>Ту́рбокод</b> — параллельный каскадный блоковый <a class="new" href="/w/index.php?title=%D0%A1%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0%D1%82%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%BA%D0%BE%D0%B4&amp;action=edit&amp;redlink=1" title="Систематический код (страница отсутствует)">систематический код</a>, способный исправлять ошибки, возникающие при передаче цифровой <a href="/wiki/%D0%98%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%86%D0%B8%D1%8F" title="Информация">информации</a> по <a href="/wiki/%D0%9A%D0%B0%D0%BD%D0%B0%D0%BB_%D1%81%D0%B2%D1%8F%D0%B7%D0%B8" title="Канал связи">каналу связи</a> с <a href="/wiki/%D0%A8%D1%83%D0%BC" title="Шум">шумами</a>. Синонимом турбокода является известный в теории кодирования термин — <b>каскадный код</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">concatenated code</span>) (предложен <a href="/wiki/%D0%A4%D0%BE%D1%80%D0%BD%D0%B8,_%D0%94%D0%B6%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D0%B8" title="Форни, Джованни">Д. Форни</a> в 1966 году).
</p><p>Турбокод состоит из каскада параллельно соединённых систематических кодов. Эти составляющие называются компонентными кодами. В качестве компонентных кодов могут использоваться <a class="mw-redirect" href="/wiki/%D0%A1%D0%B2%D1%91%D1%80%D1%82%D0%BE%D1%87%D0%BD%D1%8B%D0%B5_%D0%BA%D0%BE%D0%B4%D1%8B" title="Свёрточные коды">свёрточные коды</a>, <a href="/wiki/%D0%9A%D0%BE%D0%B4_%D0%A5%D1%8D%D0%BC%D0%BC%D0%B8%D0%BD%D0%B3%D0%B0" title="Код Хэмминга">Код Хэмминга</a>, <a href="/wiki/%D0%9A%D0%BE%D0%B4_%D0%A0%D0%B8%D0%B4%D0%B0_%E2%80%94_%D0%A1%D0%BE%D0%BB%D0%BE%D0%BC%D0%BE%D0%BD%D0%B0" title="Код Рида — Соломона">код Рида — Соломона</a>, <a href="/wiki/%D0%9A%D0%BE%D0%B4_%D0%91%D0%BE%D1%83%D0%B7%D0%B0_%E2%80%94_%D0%A7%D0%BE%D1%83%D0%B4%D1%85%D1%83%D1%80%D0%B8_%E2%80%94_%D0%A5%D0%BE%D0%BA%D0%B2%D0%B8%D0%BD%D0%B3%D0%B5%D0%BC%D0%B0" title="Код Боуза — Чоудхури — Хоквингема">код Боуза — Чоудхури — Хоквингема</a> и другие. В зависимости от выбора компонентного кода турбокоды делятся на свёрточные турбокоды (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Turbo Convolutional Codes, ТСС</span>) и блоковые коды-произведения (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Turbo Product Codes, TPC</span>)<sup class="reference" id="cite_ref-ref-1_1-0"><a href="#cite_note-ref-1-1">[1]</a></sup>.
</p><p>Турбокоды были разработаны в <a href="/wiki/1993_%D0%B3%D0%BE%D0%B4" title="1993 год">1993 году</a> и являются классом высокоэффективных помехоустойчивых <a class="mw-redirect" href="/wiki/%D0%9E%D0%B1%D0%BD%D0%B0%D1%80%D1%83%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5_%D0%B8_%D0%B8%D1%81%D0%BF%D1%80%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D0%B5_%D0%BE%D1%88%D0%B8%D0%B1%D0%BE%D0%BA" title="Обнаружение и исправление ошибок">кодов с коррекцией ошибок</a>, используются в <a href="/wiki/%D0%AD%D0%BB%D0%B5%D0%BA%D1%82%D1%80%D0%BE%D1%82%D0%B5%D1%85%D0%BD%D0%B8%D0%BA%D0%B0" title="Электротехника">электротехнике</a> и <a href="/wiki/%D0%A6%D0%B8%D1%84%D1%80%D0%BE%D0%B2%D0%B0%D1%8F_%D1%81%D0%B2%D1%8F%D0%B7%D1%8C" title="Цифровая связь">цифровой связи</a>, а также нашли своё применение в <a href="/wiki/%D0%A1%D0%BF%D1%83%D1%82%D0%BD%D0%B8%D0%BA%D0%BE%D0%B2%D0%B0%D1%8F_%D1%81%D0%B2%D1%8F%D0%B7%D1%8C" title="Спутниковая связь">спутниковой связи</a> и в других областях, в которых необходимо достижение максимальной <a class="mw-redirect" href="/wiki/%D0%A1%D0%BA%D0%BE%D1%80%D0%BE%D1%81%D1%82%D1%8C_%D0%BF%D0%B5%D1%80%D0%B5%D0%B4%D0%B0%D1%87%D0%B8_%D0%B8%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%86%D0%B8%D0%B8" title="Скорость передачи информации">скорости передачи данных</a> по каналу связи с шумами в ограниченной <a href="/wiki/%D0%9F%D0%BE%D0%BB%D0%BE%D1%81%D0%B0_%D0%BF%D1%80%D0%BE%D0%BF%D1%83%D1%81%D0%BA%D0%B0%D0%BD%D0%B8%D1%8F" title="Полоса пропускания">полосе частот</a>.
</p>

<h2><span id=".D0.98.D1.81.D1.82.D0.BE.D1.80.D0.B8.D1.8F"></span><span class="mw-headline" id="История">История</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;veaction=edit&amp;section=1" title="Редактировать раздел «История»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;action=edit&amp;section=1" title="Редактировать код раздела «История»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>До момента возникновения турбокода был широко распространён метод каскадного кодирования, при котором данные кодировались сначала <a href="/wiki/%D0%9A%D0%BE%D0%B4_%D0%A0%D0%B8%D0%B4%D0%B0_%E2%80%94_%D0%A1%D0%BE%D0%BB%D0%BE%D0%BC%D0%BE%D0%BD%D0%B0" title="Код Рида — Соломона">кодом Рида — Соломона</a>, а затем <a href="/wiki/%D0%A1%D0%B2%D1%91%D1%80%D1%82%D0%BE%D1%87%D0%BD%D1%8B%D0%B9_%D0%BA%D0%BE%D0%B4" title="Свёрточный код">свёрточным кодом</a>. Он достаточно близко подходит к <a href="/wiki/%D0%A2%D0%B5%D0%BE%D1%80%D0%B5%D0%BC%D1%8B_%D0%A8%D0%B5%D0%BD%D0%BD%D0%BE%D0%BD%D0%B0_%D0%B4%D0%BB%D1%8F_%D0%BA%D0%B0%D0%BD%D0%B0%D0%BB%D0%B0_%D1%81_%D1%88%D1%83%D0%BC%D0%B0%D0%BC%D0%B8" title="Теоремы Шеннона для канала с шумами">границе Шеннона</a> и объединяет в себе блок коррекции ошибок, использующий <a href="/wiki/%D0%9A%D0%BE%D0%B4_%D0%A0%D0%B8%D0%B4%D0%B0_%E2%80%94_%D0%A1%D0%BE%D0%BB%D0%BE%D0%BC%D0%BE%D0%BD%D0%B0" title="Код Рида — Соломона">код Рида — Соломона</a> и блок <a href="/wiki/%D0%A1%D0%B2%D1%91%D1%80%D1%82%D0%BE%D1%87%D0%BD%D1%8B%D0%B9_%D0%BA%D0%BE%D0%B4" title="Свёрточный код">свёрточных кодов</a>, декодируемых с помощью <a href="/wiki/%D0%90%D0%BB%D0%B3%D0%BE%D1%80%D0%B8%D1%82%D0%BC_%D0%92%D0%B8%D1%82%D0%B5%D1%80%D0%B1%D0%B8" title="Алгоритм Витерби">алгоритма Витерби</a>.
</p><p>Турбокоды были предложены К. Берроу (<i>C. Berrou</i>), А. Главьё (<i>A. Glavieux</i>) и П. Ситимашимой (<i>P. Thitimajshima</i>) из (<a href="/wiki/%D0%A4%D1%80%D0%B0%D0%BD%D1%86%D1%83%D0%B7%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Французский язык">фр.</a> <span lang="fr" style="font-style:italic;">École Nationale Supérieure des Télécommunications de Bretagne (ENST Bretagne)</span>, Высшая национальная школа <a class="mw-redirect" href="/wiki/%D0%A2%D0%B5%D0%BB%D0%B5%D0%BA%D0%BE%D0%BC%D0%BC%D1%83%D0%BD%D0%B8%D0%BA%D0%B0%D1%86%D0%B8%D0%B8" title="Телекоммуникации">телекоммуникаций</a> <a href="/wiki/%D0%91%D1%80%D0%B5%D1%82%D0%B0%D0%BD%D1%8C" title="Бретань">Бретани</a> (<a href="/wiki/%D0%A4%D1%80%D0%B0%D0%BD%D1%86%D0%B8%D1%8F" title="Франция">Франция</a>)) в 1993 году в статье «<a href="/wiki/%D0%9A%D0%BE%D0%B4" title="Код">Кодирование</a> и <a class="mw-redirect" href="/wiki/%D0%94%D0%B5%D0%BA%D0%BE%D0%B4%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Декодирование">декодирование</a> с исправлением ошибок вблизи предела Шеннона: турбокоды» (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">«Near Shannon Limit Error-correcting Coding and Decoding: Turbo-code»</span>)<sup class="reference" id="cite_ref-ref-2_2-0"><a href="#cite_note-ref-2-2">[2]</a></sup>,
опубликованной в трудах <a class="mw-redirect" href="/wiki/IEEE" title="IEEE">IEEE</a>. В последующей статье<sup class="reference" id="cite_ref-ref-3_3-0"><a href="#cite_note-ref-3-3">[3]</a></sup> Берроу отдаёт должное интуиции Г. Бэттэйла (<i>G. Battail</i>), Дж. Хагенауэра (<i>J. Hagenauer</i>) и П. Хёера (<i>P. Hoeher</i>), которые в конце <a class="mw-redirect" href="/wiki/1980-%D0%B5" title="1980-е">1980-х</a> теоретически предсказали <a href="/wiki/%D0%92%D0%B5%D1%80%D0%BE%D1%8F%D1%82%D0%BD%D0%BE%D1%81%D1%82%D1%8C" title="Вероятность">вероятностную</a> обработку данных. Также Берроу упоминает, что <a href="/wiki/%D0%93%D0%B0%D0%BB%D0%BB%D0%B0%D0%B3%D0%B5%D1%80,_%D0%A0%D0%BE%D0%B1%D0%B5%D1%80%D1%82" title="Галлагер, Роберт">Роберт Галлагер</a> и М. Таннер (<i>M. Tanner</i>) ещё в своё время представляли методы кодирования и декодирования с общими <a href="/wiki/%D0%9F%D1%80%D0%B8%D0%BD%D1%86%D0%B8%D0%BF" title="Принцип">принципами</a>, очень близкими к турбокодам, но в то время не были доступны необходимые <a class="new" href="/w/index.php?title=%D0%92%D1%8B%D1%87%D0%B8%D1%81%D0%BB%D0%B8%D1%82%D0%B5%D0%BB%D1%8C%D0%BD%D1%8B%D0%B5_%D1%80%D0%B5%D1%81%D1%83%D1%80%D1%81%D1%8B&amp;action=edit&amp;redlink=1" title="Вычислительные ресурсы (страница отсутствует)">вычислительные возможности</a>.
</p>
<h2><span id=".D0.A1.D1.82.D1.80.D1.83.D0.BA.D1.82.D1.83.D1.80.D0.B0_.D1.82.D1.83.D1.80.D0.B1.D0.BE.D0.BA.D0.BE.D0.B4.D0.B0"></span><span class="mw-headline" id="Структура_турбокода">Структура турбокода</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;veaction=edit&amp;section=2" title="Редактировать раздел «Структура турбокода»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;action=edit&amp;section=2" title="Редактировать код раздела «Структура турбокода»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<table class="plainlinks metadata ambox ambox-style" role="presentation"><tbody><tr><td class="mbox-image"><div style="width:52px"><span typeof="mw:File"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:Nuvola_apps_important_recycle.svg"><img class="mw-file-element" data-file-height="500" data-file-width="600" decoding="async" height="38" src="//upload.wikimedia.org/wikipedia/commons/thumb/0/09/Nuvola_apps_important_recycle.svg/45px-Nuvola_apps_important_recycle.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/0/09/Nuvola_apps_important_recycle.svg/68px-Nuvola_apps_important_recycle.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/0/09/Nuvola_apps_important_recycle.svg/90px-Nuvola_apps_important_recycle.svg.png 2x" width="45"/></a></span></div></td><td class="mbox-text"><div class="mbox-text-div"><b>Эта статья или раздел нуждается в переработке.</b></div><div class="mbox-textsmall-div hide-when-compact" style="font-size:85%">Пожалуйста, улучшите статью в соответствии с <a class="mw-redirect" href="/wiki/%D0%92%D0%B8%D0%BA%D0%B8%D0%BF%D0%B5%D0%B4%D0%B8%D1%8F:%D0%9F%D1%80%D0%B0%D0%B2%D0%B8%D0%BB%D0%B0_%D0%B8_%D1%83%D0%BA%D0%B0%D0%B7%D0%B0%D0%BD%D0%B8%D1%8F" title="Википедия:Правила и указания">правилами написания статей</a>.</div></td></tr></tbody></table>
<p>Согласно <a class="mw-redirect" href="/wiki/%D0%A8%D0%B5%D0%BD%D0%BD%D0%BE%D0%BD,_%D0%9A%D0%BB%D0%BE%D0%B4_%D0%AD%D0%BB%D0%B2%D1%83%D0%B4" title="Шеннон, Клод Элвуд">Шеннону</a>, наилучшим кодом является <a href="/wiki/%D0%9A%D0%BE%D0%B4" title="Код">код</a>, который передает сообщение за бесконечно большое время, формируя в каждый момент времени случайные кодовые элементы<sup class="ts-fix-template noprint">[<i><a href="/wiki/%D0%92%D0%B8%D0%BA%D0%B8%D0%BF%D0%B5%D0%B4%D0%B8%D1%8F:%D0%A1%D1%81%D1%8B%D0%BB%D0%BA%D0%B8_%D0%BD%D0%B0_%D0%B8%D1%81%D1%82%D0%BE%D1%87%D0%BD%D0%B8%D0%BA%D0%B8" title="Википедия:Ссылки на источники"><span title="нет источника (5 января 2020)">источник не указан 1515 дней</span></a></i>]</sup>. У приёмника есть бесконечные версии сообщения, искажённого случайным образом. Из этих копий декодер должен выбрать копию, наиболее близкую к переданному сообщению. Это представляет собой теоретически идеальный <a href="/wiki/%D0%9A%D0%BE%D0%B4" title="Код">код</a>, который может <a class="mw-redirect" href="/wiki/%D0%9E%D0%B1%D0%BD%D0%B0%D1%80%D1%83%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5_%D0%B8_%D0%B8%D1%81%D0%BF%D1%80%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D0%B5_%D0%BE%D1%88%D0%B8%D0%B1%D0%BE%D0%BA" title="Обнаружение и исправление ошибок">исправить</a> все ошибки в сигнале. Турбокод является шагом в этом направлении. Ясно, что мы не должны посылать информационное сообщение в течение бесконечного времени. Для приемлемой работы достаточно удвоить или утроить время передачи, что обеспечит довольно приличные результаты для каналов связи.
</p><p>Особенностью турбокодов является параллельная структура, состоящая из рекурсивных систематических <a href="/wiki/%D0%A1%D0%B2%D1%91%D1%80%D1%82%D0%BE%D1%87%D0%BD%D1%8B%D0%B9_%D0%BA%D0%BE%D0%B4" title="Свёрточный код">сверточных</a> (RSC) <a href="/wiki/%D0%9A%D0%BE%D0%B4" title="Код">кодов</a>, работающих параллельно и использующих создание <i>случайной</i> версии сообщения. Параллельная структура использует два или больше <a href="/wiki/%D0%9A%D0%BE%D0%B4" title="Код">кодов</a> RSC, каждый с различным <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%B6%D0%B8%D1%82%D0%B5%D0%BB%D1%8C" title="Перемежитель">перемежителем</a>. Цель <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%B6%D0%B8%D1%82%D0%B5%D0%BB%D1%8C" title="Перемежитель">перемежителя</a> состоит в том, чтобы предложить каждому кодеру <a href="/wiki/%D0%9A%D0%BE%D1%80%D1%80%D0%B5%D0%BB%D1%8F%D1%86%D0%B8%D1%8F" title="Корреляция">некоррелированную</a> или <i>случайную</i> версию <a href="/wiki/%D0%98%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%86%D0%B8%D1%8F" title="Информация">информации</a>, в результате чего паритетные биты каждого RSC становятся <a href="/wiki/%D0%9D%D0%B5%D0%B7%D0%B0%D0%B2%D0%B8%D1%81%D0%B8%D0%BC%D0%BE%D1%81%D1%82%D1%8C_(%D1%82%D0%B5%D0%BE%D1%80%D0%B8%D1%8F_%D0%B2%D0%B5%D1%80%D0%BE%D1%8F%D1%82%D0%BD%D0%BE%D1%81%D1%82%D0%B5%D0%B9)" title="Независимость (теория вероятностей)">независимыми</a>.
</p><p>В турбокодах блоки имеют длину порядка нескольких Кбит. Цель такой длины состоит в том, чтобы эффективно рандомизировать последовательность, идущую на второе кодирующее устройство. Чем длиннее размер блока, тем лучше его <a href="/wiki/%D0%9A%D0%BE%D1%80%D1%80%D0%B5%D0%BB%D1%8F%D1%86%D0%B8%D1%8F" title="Корреляция">корреляция</a> с сообщением первого кодера, то есть <a href="/wiki/%D0%9A%D0%BE%D1%80%D1%80%D0%B5%D0%BB%D1%8F%D1%86%D0%B8%D1%8F" title="Корреляция">корреляция</a> мала.
</p><p>Существует несколько схем турбокодов:
</p>
<ul><li><b>PCCC</b> — в случае конкатенации параллельных сверточных кодов</li>
<li><b>SCCC</b> — схема с последовательным соединением сверточных кодов, коды <i>SCCC</i> имеют высокие характеристики при больших отношениях сигнал/шум</li>
<li><b>TPC</b> — <b>турбокод-произведение</b>, использует блочные коды вместо сверточных; два различных блочных кода (обычно <a href="/wiki/%D0%9A%D0%BE%D0%B4_%D0%A5%D1%8D%D0%BC%D0%BC%D0%B8%D0%BD%D0%B3%D0%B0" title="Код Хэмминга">коды Хемминга</a>) соединены последовательно без промежуточного <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%B6%D0%B8%D1%82%D0%B5%D0%BB%D1%8C" title="Перемежитель">перемежителя</a>. Так как два кода независимы и работают в рядах и колонках, что само по себе обеспечивает достаточно хорошую рандомизацию, то применение <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%B6%D0%B8%D1%82%D0%B5%D0%BB%D1%8C" title="Перемежитель">перемежителя</a> не требуется.</li></ul>

<p>В <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%B6%D0%B8%D1%82%D0%B5%D0%BB%D1%8C" title="Перемежитель">перемежителях</a> по псевдослучайному закону происходит перемешивание поступающих бит. В отличие от посимвольного прямоугольного <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%B6%D0%B8%D1%82%D0%B5%D0%BB%D1%8C" title="Перемежитель">перемежителя</a>, используемого в <a href="/wiki/%D0%9A%D0%BE%D0%B4_%D0%A0%D0%B8%D0%B4%D0%B0_%E2%80%94_%D0%A1%D0%BE%D0%BB%D0%BE%D0%BC%D0%BE%D0%BD%D0%B0" title="Код Рида — Соломона">кодах Рида-Соломона</a>, в турбокодах используется <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%B6%D0%B8%D1%82%D0%B5%D0%BB%D1%8C" title="Перемежитель">перемежение</a> отдельных <a href="/wiki/%D0%91%D0%B8%D1%82" title="Бит">бит</a>, которое подобно случайным перестановкам. Причём впоследствии, при операциях декодирования этот закон <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%B6%D0%B8%D1%82%D0%B5%D0%BB%D1%8C" title="Перемежитель">перемежения</a> будет считаться известным. Полученные последовательности поступают на входы кодеров.
</p><p>Задача <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%B6%D0%B8%D1%82%D0%B5%D0%BB%D1%8C" title="Перемежитель">перемежителя</a> — преобразовать входную последовательность так, чтобы комбинации бит <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle X_{0}}" xmlns="http://www.w3.org/1998/Math/MathML">

Таким образом кодеры получают на выходе кодовые слова с различными весами. При кодировании формируются кодовые слова так, чтобы получалось максимально возможное среднее расстояние между ними (<b>расстоянием</b> между двумя кодовыми словами называется число бит, в которых они различаются). Из-за того что кодовые блоки формируются из почти независимых частей, на выходе турбокодера среднее расстояние между кодовыми словами больше, чем минимальное расстояние для каждого компонентного кодера, а следовательно растёт эффективность кодирования.

<h3><span id=".D0.90.D0.BB.D0.B3.D0.BE.D1.80.D0.B8.D1.82.D0.BC_.D0.B4.D0.B5.D0.BA.D0.BE.D0.B4.D0.B8.D1.80.D0.BE.D0.B2.D0.B0.D0.BD.D0.B8.D1.8F_.D0.BF.D0.BE_.D0.BC.D0.B0.D0.BA.D1.81.D0.B8.D0.BC.D1.83.D0.BC.D1.83_.D0.B0.D0.BF.D0.BE.D1.81.D1.82.D0.B5.D1.80.D0.B8.D0.BE.D1.80.D0.BD.D0.BE.D0.B9_.D0.B2.D0.B5.D1.80.D0.BE.D1.8F.D1.82.D0.BD.D0.BE.D1.81.D1.82.D0.B8_.28MAP.29"></span><span class="mw-headline" id="Алгоритм_декодирования_по_максимуму_апостериорной_вероятности_(MAP)"><a href="/wiki/%D0%9E%D1%86%D0%B5%D0%BD%D0%BA%D0%B0_%D0%B0%D0%BF%D0%BE%D1%81%D1%82%D0%B5%D1%80%D0%B8%D0%BE%D1%80%D0%BD%D0%BE%D0%B3%D0%BE_%D0%BC%D0%B0%D0%BA%D1%81%D0%B8%D0%BC%D1%83%D0%BC%D0%B0" title="Оценка апостериорного максимума">Алгоритм декодирования по максимуму апостериорной вероятности (MAP)</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;veaction=edit&amp;section=7" title="Редактировать раздел «Алгоритм декодирования по максимуму апостериорной вероятности (MAP)»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;action=edit&amp;section=7" title="Редактировать код раздела «Алгоритм декодирования по максимуму апостериорной вероятности (MAP)»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>При осуществлении декодирования с <a class="mw-redirect" href="/wiki/%D0%9E%D0%B1%D0%BD%D0%B0%D1%80%D1%83%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5_%D0%B8_%D0%B8%D1%81%D0%BF%D1%80%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D0%B5_%D0%BE%D1%88%D0%B8%D0%B1%D0%BE%D0%BA" title="Обнаружение и исправление ошибок">исправлением ошибок</a> существенен анализ <a href="/wiki/%D0%90%D0%BF%D1%80%D0%B8%D0%BE%D1%80%D0%B8" title="Априори">априорной</a> и <a href="/wiki/%D0%90%D0%BF%D0%BE%D1%81%D1%82%D0%B5%D1%80%D0%B8%D0%BE%D1%80%D0%B8" title="Апостериори">апостериорной</a> вероятностей прихода верного кодового слова. <b>Априорной</b> называется информация, которой обладает декодер до прихода кодового слова, а <b>апостериорной</b> называется информация, полученная после обработки кодового слова.
</p><p>В своей работе Берроу предлагает для использования в турбодекодерах <a href="/wiki/%D0%9E%D1%86%D0%B5%D0%BD%D0%BA%D0%B0_%D0%B0%D0%BF%D0%BE%D1%81%D1%82%D0%B5%D1%80%D0%B8%D0%BE%D1%80%D0%BD%D0%BE%D0%B3%D0%BE_%D0%BC%D0%B0%D0%BA%D1%81%D0%B8%D0%BC%D1%83%D0%BC%D0%B0" title="Оценка апостериорного максимума">алгоритм максимума апостериорной вероятности</a> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Maximum of A-posteriori Probability, MAP</span>), также известный под названием алгоритма Бала (<a href="/wiki/BCJR-%D0%B0%D0%BB%D0%B3%D0%BE%D1%80%D0%B8%D1%82%D0%BC" title="BCJR-алгоритм">Bahl — Cocke — Jelinek — Raviv (BCJR)</a>). Алгоритм Бала дает «<b>мягкую</b>» оценку <a href="/wiki/%D0%94%D0%BE%D1%81%D1%82%D0%BE%D0%B2%D0%B5%D1%80%D0%BD%D0%BE%D1%81%D1%82%D1%8C" title="Достоверность">достоверности</a> декодированного бита. То есть предъявляет на выходе степень доверия результату декодирования. В противоположность «<b>жёсткой</b>» структуре, при которой на выходе декодера формируется лишь наиболее вероятное значение декодированного бита («0» или «1»), при вынесении «мягкого» решения используется более подробная <a href="/wiki/%D0%94%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F" title="Дискретизация">дискретизация</a> выходного сигнала, характеризующая вероятность корректного приема бита. Благодаря использованию «мягких» решений в турбодекодерах оказывается эффективным использование нескольких <a class="mw-disambig" href="/wiki/%D0%98%D1%82%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D1%8F" title="Итерация">итераций</a> декодирования. Апостериорная информация, полученная о кодовом слове на выходе первой итерации декодирования, поступает на вход блока следующей итерации и является для него уже априорной вероятностью. Такой подход позволяет улучшать <a class="mw-disambig" href="/wiki/%D0%9A%D0%B0%D1%87%D0%B5%D1%81%D1%82%D0%B2%D0%BE" title="Качество">качество</a> декодирования от итерации к итерации. Таким образом, изменяя число итераций декодирования, можно адаптировать декодер к текущему состоянию канала передачи и достичь требуемой вероятности ошибки на бит<sup class="reference" id="cite_ref-ref-5_6-1"><a href="#cite_note-ref-5-6">[6]</a></sup>.
</p>

<p>Рассмотрим информационный <a href="/wiki/%D0%91%D0%B8%D1%82" title="Бит">бит</a> как бинарную переменную <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle {\boldsymbol {u}}_{k}}" xmlns="http://www.w3.org/1998/Math/MathML">
<semantics>

<p>С выхода каждой итерации решение переходит на вход следующей. Организация работы трёхитерационного турбодекодера показана на рис. 3. От итерации к итерации происходит уточнение решения. При этом каждая итерация работает с «мягкими» оценками и на выход отдает также «мягкие». Поэтому такие схемы получили название <a class="new" href="/w/index.php?title=%D0%94%D0%B5%D0%BA%D0%BE%D0%B4%D0%B5%D1%80%D1%8B_%D1%81_%D0%BC%D1%8F%D0%B3%D0%BA%D0%B8%D0%BC_%D0%B2%D1%85%D0%BE%D0%B4%D0%BE%D0%BC_%D0%B8_%D0%BC%D1%8F%D0%B3%D0%BA%D0%B8%D0%BC_%D0%B2%D1%8B%D1%85%D0%BE%D0%B4%D0%BE%D0%BC&amp;action=edit&amp;redlink=1" title="Декодеры с мягким входом и мягким выходом (страница отсутствует)">декодеров с мягким входом и мягким выходом</a> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Soft Input Soft Output (SISO)</span>)<sup class="reference" id="cite_ref-ref-7_8-0"><a href="#cite_note-ref-7-8">[8]</a></sup>.
Процесс декодирования прекращается либо после выполнения всех итераций, либо когда вероятность ошибки на бит достигнет требуемого значения. После декодирования из полученного «мягкого» решения производится окончательное «жёсткое»<sup class="reference" id="cite_ref-ref-6_7-1"><a href="#cite_note-ref-6-7">[7]</a></sup>.
</p>
<h2><span id=".D0.9F.D1.80.D0.B5.D0.B8.D0.BC.D1.83.D1.89.D0.B5.D1.81.D1.82.D0.B2.D0.B0_.D0.B8_.D0.BD.D0.B5.D0.B4.D0.BE.D1.81.D1.82.D0.B0.D1.82.D0.BA.D0.B8_.D1.82.D1.83.D1.80.D0.B1.D0.BE.D0.BA.D0.BE.D0.B4.D0.BE.D0.B2"></span><span class="mw-headline" id="Преимущества_и_недостатки_турбокодов">Преимущества и недостатки турбокодов</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;veaction=edit&amp;section=11" title="Редактировать раздел «Преимущества и недостатки турбокодов»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;action=edit&amp;section=11" title="Редактировать код раздела «Преимущества и недостатки турбокодов»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span id=".D0.9F.D1.80.D0.B5.D0.B8.D0.BC.D1.83.D1.89.D0.B5.D1.81.D1.82.D0.B2.D0.B0"></span><span class="mw-headline" id="Преимущества">Преимущества</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;veaction=edit&amp;section=12" title="Редактировать раздел «Преимущества»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;action=edit&amp;section=12" title="Редактировать код раздела «Преимущества»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Среди всех практически используемых современных методов коррекции ошибок турбокоды и <a class="mw-redirect" href="/wiki/LDPC" title="LDPC">коды с низкой плотностью проверок на чётность</a> наиболее близко подходят к границе <a class="mw-redirect" href="/wiki/%D0%A8%D0%B5%D0%BD%D0%BD%D0%BE%D0%BD,_%D0%9A%D0%BB%D0%BE%D0%B4_%D0%AD%D0%BB%D0%B2%D1%83%D0%B4" title="Шеннон, Клод Элвуд">Шеннона</a>, теоретическому пределу максимальной <a href="/wiki/%D0%9F%D1%80%D0%BE%D0%BF%D1%83%D1%81%D0%BA%D0%BD%D0%B0%D1%8F_%D1%81%D0%BF%D0%BE%D1%81%D0%BE%D0%B1%D0%BD%D0%BE%D1%81%D1%82%D1%8C" title="Пропускная способность">пропускной способности</a> зашумленного канала. Турбокоды позволяют увеличить скорость передачи информации, не требуя увеличения <a href="/wiki/%D0%9C%D0%BE%D1%89%D0%BD%D0%BE%D1%81%D1%82%D1%8C" title="Мощность">мощности</a> передатчика, или они могут быть использованы для уменьшения требуемой мощности при передаче с заданной скоростью. Важным преимуществом турбокодов является независимость сложности декодирования от длины информационного блока, что позволяет снизить вероятность ошибки декодирования путём увеличения его длины<sup class="reference" id="cite_ref-ref-8_9-0"><a href="#cite_note-ref-8-9">[9]</a></sup>.
</p>
<h3><span id=".D0.9D.D0.B5.D0.B4.D0.BE.D1.81.D1.82.D0.B0.D1.82.D0.BA.D0.B8"></span><span class="mw-headline" id="Недостатки">Недостатки</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;veaction=edit&amp;section=13" title="Редактировать раздел «Недостатки»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;action=edit&amp;section=13" title="Редактировать код раздела «Недостатки»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Основной недостаток турбокодов — это относительно высокая сложность декодирования и большая задержка, которые делают их неудобными для некоторых применений. Но, например, для использования в спутниковых каналах этот недостаток не является определяющим, так как длина канала связи сама по себе вносит задержку, вызванную конечностью <a href="/wiki/%D0%A1%D0%BA%D0%BE%D1%80%D0%BE%D1%81%D1%82%D1%8C_%D1%81%D0%B2%D0%B5%D1%82%D0%B0" title="Скорость света">скорости света</a>.
</p><p>Ещё один важный недостаток турбокодов — сравнительно небольшое кодовое расстояние (то есть минимальное расстояние между двумя кодовыми словами в смысле выбранной <a class="mw-disambig" href="/wiki/%D0%9C%D0%B5%D1%82%D1%80%D0%B8%D0%BA%D0%B0" title="Метрика">метрики</a>). Это приводит к тому, что, хотя при большой входной вероятности ошибки (то есть в плохом канале) эффективность турбокода высока, при малой входной вероятности ошибки эффективность турбокода крайне ограничена.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> Поэтому в хороших каналах для дальнейшего уменьшения вероятности ошибки применяют не турбокоды, а <a class="mw-redirect" href="/wiki/LDPC-%D0%BA%D0%BE%D0%B4%D1%8B" title="LDPC-коды">LDPC-коды</a>.
</p><p>Хотя сложность используемых алгоритмов турбокодирования и недостаток открытого <a href="/wiki/%D0%9F%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Программное обеспечение">программного обеспечения</a> препятствуют внедрению турбокодов, в настоящее время многие современные системы используют турбокоды.
</p>
<h2><span id=".D0.9F.D1.80.D0.B8.D0.BC.D0.B5.D0.BD.D0.B5.D0.BD.D0.B8.D0.B5_.D1.82.D1.83.D1.80.D0.B1.D0.BE.D0.BA.D0.BE.D0.B4.D0.BE.D0.B2"></span><span class="mw-headline" id="Применение_турбокодов">Применение турбокодов</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;veaction=edit&amp;section=14" title="Редактировать раздел «Применение турбокодов»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%A2%D1%83%D1%80%D0%B1%D0%BE%D0%BA%D0%BE%D0%B4&amp;action=edit&amp;section=14" title="Редактировать код раздела «Применение турбокодов»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Компании <a href="/wiki/France_T%C3%A9l%C3%A9com" title="France Télécom">France Télécom</a> и Telediffusion de France запатентовали широкий класс турбокодов, что ограничивает возможность их свободного применения и, в то же время, стимулирует развитие новых методов кодирования таких, как, например, <a class="mw-redirect" href="/wiki/LDPC" title="LDPC">LDPC</a>.
</p><p>Турбокоды активно применяются в системах <a href="/wiki/%D0%A1%D0%BF%D1%83%D1%82%D0%BD%D0%B8%D0%BA%D0%BE%D0%B2%D0%B0%D1%8F_%D1%81%D0%B2%D1%8F%D0%B7%D1%8C" title="Спутниковая связь">спутниковой</a> и <a class="mw-redirect" href="/wiki/%D0%9C%D0%BE%D0%B1%D0%B8%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D1%81%D0%B2%D1%8F%D0%B7%D1%8C" title="Мобильная связь">мобильной связи</a>, беспроводного <a class="mw-redirect" href="/wiki/%D0%A8%D0%B8%D1%80%D0%BE%D0%BA%D0%BE%D0%BF%D0%BE%D0%BB%D0%BE%D1%81%D0%BD%D1%8B%D0%B9_%D0%B4%D0%BE%D1%81%D1%82%D1%83%D0%BF" title="Широкополосный доступ">широкополосного доступа</a> и <a href="/wiki/%D0%A6%D0%B8%D1%84%D1%80%D0%BE%D0%B2%D0%BE%D0%B5_%D1%82%D0%B5%D0%BB%D0%B5%D0%B2%D0%B8%D0%B4%D0%B5%D0%BD%D0%B8%D0%B5" title="Цифровое телевидение">цифрового телевидения</a>.<sup class="reference" id="cite_ref-ref-7_8-1"><a href="#cite_note-ref-7-8">[8]</a></sup> Турбокоды утверждены в стандарте спутниковой связи <a href="/wiki/DVB-RCS" title="DVB-RCS">DVB-RCS</a>. Турбокоды также нашли широкое применение в <a href="/wiki/3G" title="3G">мобильных системах связи третьего поколения</a> (стандарты <a class="mw-redirect" href="/wiki/CDMA2000" title="CDMA2000">CDMA2000</a> и <a href="/wiki/UMTS" title="UMTS">UMTS</a>).<sup class="reference" id="cite_ref-ref-8_9-1"><a href="#cite_note-ref-8-9">[9]</a></sup>
</p>
