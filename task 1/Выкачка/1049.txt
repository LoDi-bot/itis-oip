<p><b>Передискретиза́ция</b> в <a href="/wiki/%D0%9E%D0%B1%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D0%BA%D0%B0_%D1%81%D0%B8%D0%B3%D0%BD%D0%B0%D0%BB%D0%BE%D0%B2" title="Обработка сигналов">обработке сигналов</a> — изменение <a href="/wiki/%D0%A7%D0%B0%D1%81%D1%82%D0%BE%D1%82%D0%B0_%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D0%B8" title="Частота дискретизации">частоты дискретизации</a> <a href="/wiki/%D0%94%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%BD%D1%8B%D0%B9_%D1%81%D0%B8%D0%B3%D0%BD%D0%B0%D0%BB" title="Дискретный сигнал">дискретного</a> (чаще всего <a href="/wiki/%D0%A6%D0%B8%D1%84%D1%80%D0%BE%D0%B2%D0%BE%D0%B9_%D1%81%D0%B8%D0%B3%D0%BD%D0%B0%D0%BB" title="Цифровой сигнал">цифрового</a>) сигнала. Алгоритмы передискретизации широко применяются при обработке <a href="/wiki/%D0%97%D0%B2%D1%83%D0%BA" title="Звук">звуковых сигналов</a>, <a class="mw-redirect" href="/wiki/%D0%A0%D0%B0%D0%B4%D0%B8%D0%BE%D1%81%D0%B8%D0%B3%D0%BD%D0%B0%D0%BB" title="Радиосигнал">радиосигналов</a> и изображений (передискретизация <a class="mw-redirect" href="/wiki/%D0%A0%D0%B0%D1%81%D1%82%D1%80%D0%BE%D0%B2%D0%BE%D0%B5_%D0%B8%D0%B7%D0%BE%D0%B1%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5" title="Растровое изображение">растрового изображения</a> — это изменение его разрешения в <a class="mw-redirect" href="/wiki/%D0%9F%D0%B8%D0%BA%D1%81%D0%B5%D0%BB" title="Пиксел">пикселах</a>).
</p><p>Отсчёты сигнала, соответствующие новой частоте дискретизации, вычисляются по уже имеющимся отсчётам и не содержат новой информации.
</p><p>Повышение частоты дискретизации называется <i><a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D1%80%D0%BF%D0%BE%D0%BB%D1%8F%D1%86%D0%B8%D1%8F" title="Интерполяция">интерполяцией</a></i>, понижение — <i><a href="/wiki/%D0%94%D0%B5%D1%86%D0%B8%D0%BC%D0%B0%D1%86%D0%B8%D1%8F_(%D0%BE%D0%B1%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D0%BA%D0%B0_%D1%81%D0%B8%D0%B3%D0%BD%D0%B0%D0%BB%D0%BE%D0%B2)" title="Децимация (обработка сигналов)">децимацией</a></i>.<sup class="reference" id="cite_ref-_3b6b6bfb4da6a216_1-0"><a href="#cite_note-_3b6b6bfb4da6a216-1">[1]</a></sup>
</p>

Синяя кривая — исходный непрерывный сигнал, зелёная — <a class="mw-redirect" href="/wiki/%D0%98%D0%BC%D0%BF%D1%83%D0%BB%D1%8C%D1%81%D0%BD%D0%B0%D1%8F_%D1%85%D0%B0%D1%80%D0%B0%D0%BA%D1%82%D0%B5%D1%80%D0%B8%D1%81%D1%82%D0%B8%D0%BA%D0%B0" title="Импульсная характеристика">импульсная характеристика</a> идеального ФНЧ. Для интерполяции суммируются значения импульсной характеристики в точках отсчётов, умноженные на величину соответствующих отсчётов.</figcaption></figure>
<p>Согласно <a href="/wiki/%D0%A2%D0%B5%D0%BE%D1%80%D0%B5%D0%BC%D0%B0_%D0%9A%D0%BE%D1%82%D0%B5%D0%BB%D1%8C%D0%BD%D0%B8%D0%BA%D0%BE%D0%B2%D0%B0" title="Теорема Котельникова">теореме Котельникова</a>, любой непрерывный сигнал с <a href="/wiki/%D0%A4%D0%B8%D0%BD%D0%B8%D1%82%D0%BD%D0%B0%D1%8F_%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D1%8F" title="Финитная функция">конечным</a> <a href="/wiki/%D0%A1%D0%BF%D0%B5%D0%BA%D1%82%D1%80" title="Спектр">спектром</a> (то есть таким спектром, в котором спектральные составляющие, соответствующие <a href="/wiki/%D0%A7%D0%B0%D1%81%D1%82%D0%BE%D1%82%D0%B0" title="Частота">частотам</a> выше или равным некоторой частоты <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle f_{0}}" xmlns="http://www.w3.org/1998/Math/MathML">

</p><p>При передискретизации отсчёты сигнала, соответствующие одной частоте дискретизации, вычисляются по имеющимся отсчётам этого же сигнала, соответствующим другой частоте дискретизации (при этом предполагается, что обе частоты дискретизации соответствуют условиям теоремы Котельникова). Идеальная передискретизация эквивалентна восстановлению непрерывного сигнала по его отсчётам с последующей дискретизацией его на новой частоте.<sup class="reference" id="cite_ref-_9e2a9304f2256d27_3-0"><a href="#cite_note-_9e2a9304f2256d27-3">[3]</a></sup>

<p>Процесс уменьшения частоты дискретизации сигнала называется децимацией.<sup class="reference" id="cite_ref-_3b6b6bfb4da6a216_1-1"><a href="#cite_note-_3b6b6bfb4da6a216-1">[1]</a></sup> Иногда этот термин употребляют только для уменьшения частоты дискретизации в целое число раз (далее <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle N}" xmlns="http://www.w3.org/1998/Math/MathML">

<ol><li><a href="/wiki/%D0%A6%D0%B8%D1%84%D1%80%D0%BE%D0%B2%D0%BE%D0%B9_%D1%84%D0%B8%D0%BB%D1%8C%D1%82%D1%80" title="Цифровой фильтр">Цифровая фильтрация</a> сигнала с целью удаления высокочастотных составляющих, не удовлетворяющих условиям теоремы Котельникова для новой частоты дискретизации;</li>

<p>В англоязычной литературе второй из этих этапов иногда обозначают термином <i>downsampling</i>.<sup class="reference" id="cite_ref-upsamp_16-0"><a href="#cite_note-upsamp-16">[16]</a></sup> В обиходе этот термин может употребляться как синоним термина «децимация».
</p><p>Первый этап необходим для исключения <a class="mw-redirect" href="/wiki/%D0%9D%D0%B0%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5_%D1%81%D0%BF%D0%B5%D0%BA%D1%82%D1%80%D0%BE%D0%B2" title="Наложение спектров">наложения спектров</a>, природа которого аналогична наложению спектров при первоначальной <a href="/wiki/%D0%94%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F" title="Дискретизация">дискретизации</a> <a href="/wiki/%D0%90%D0%BD%D0%B0%D0%BB%D0%BE%D0%B3%D0%BE%D0%B2%D1%8B%D0%B9_%D1%81%D0%B8%D0%B3%D0%BD%D0%B0%D0%BB" title="Аналоговый сигнал">аналогового сигнала</a>.<sup class="reference" id="cite_ref-_9e2a9204f2256b53_15-1"><a href="#cite_note-_9e2a9204f2256b53-15">[15]</a></sup> Наложение спектров особенно заметно на тех участках сигнала, которые содержат значительные высокочастотные спектральные составляющие. Так, на приведённых в начале статьи фотографиях небо практически не подвергнулось наложению спектров, но эффект становится заметным, если обратить внимание на резкие переходы.
</p><p>При <a href="/wiki/%D0%9F%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Программное обеспечение">программной</a> реализации алгоритма децимации «лишние» отсчёты не удаляются, а просто не вычисляются. При этом число обращений к цифровому фильтру уменьшается в <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle N}" xmlns="http://www.w3.org/1998/Math/MathML">

При аппаратной реализации экономии можно достичь путём использования <a class="new" href="/w/index.php?title=%D0%9F%D0%BE%D0%BB%D0%B8%D1%84%D0%B0%D0%B7%D0%BD%D1%8B%D0%B9_%D1%84%D0%B8%D0%BB%D1%8C%D1%82%D1%80&amp;action=edit&amp;redlink=1" title="Полифазный фильтр (страница отсутствует)">полифазных фильтров</a>.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup>

<h3><span id=".D0.98.D0.BD.D1.82.D0.B5.D1.80.D0.BF.D0.BE.D0.BB.D1.8F.D1.86.D0.B8.D1.8F_.D1.81_.D1.86.D0.B5.D0.BB.D1.8B.D0.BC_.D0.BA.D0.BE.D1.8D.D1.84.D1.84.D0.B8.D1.86.D0.B8.D0.B5.D0.BD.D1.82.D0.BE.D0.BC"></span><span class="mw-headline" id="Интерполяция_с_целым_коэффициентом">Интерполяция с целым коэффициентом</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%9F%D0%B5%D1%80%D0%B5%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F&amp;veaction=edit&amp;section=5" title="Редактировать раздел «Интерполяция с целым коэффициентом»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%9F%D0%B5%D1%80%D0%B5%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F&amp;action=edit&amp;section=5" title="Редактировать код раздела «Интерполяция с целым коэффициентом»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Интерполяцией называют увеличение частоты в целое или дробное число раз<sup class="reference" id="cite_ref-_3b6b6bfb4da6a216_1-2"><a href="#cite_note-_3b6b6bfb4da6a216-1">[1]</a></sup> путём вычисления промежуточных отсчетов по уже имеющимся. Идеальная интерполяция позволяет точно восстановить значения сигнала в промежуточных отсчётах.
</p><p>Стандартный алгоритм интерполяции сигнала с целым коэффициентом заключается в следующем:<sup class="reference" id="cite_ref-_fb4fb2b1cc5cc2bc_18-0"><a href="#cite_note-_fb4fb2b1cc5cc2bc-18">[18]</a></sup>
</p>
<ol><li>вставка нулевых отсчетов на место отсчетов, которые необходимо вычислить;</li>
<li>фильтрация сигнала цифровым фильтром нижних частот для того, чтобы убрать спектральные составляющие сигнала, которых заведомо не могло быть в исходном сигнале согласно теореме Котельникова; выход фильтра умножается на коэффициент интерполяции для <a class="mw-disambig" href="/wiki/%D0%9D%D0%BE%D1%80%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Нормирование">нормирования</a>.</li></ol>
<p>В англоязычной литературе первый из этих этапов иногда обозначается термином <i>upsampling</i>.<sup class="reference" id="cite_ref-upsamp_16-1"><a href="#cite_note-upsamp-16">[16]</a></sup> При этом в обиходе этот термин может употребляться как синоним термина «интерполяция».
</p><p>При программной реализации интерполяции нулевые отсчёты не участвуют в вычислении выходного сигнала фильтра, что позволяет оптимизировать процесс вычисления. При аппаратной реализации для экономии ресурсов возможно использование полифазных фильтров.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup><sup class="reference" id="cite_ref-_fb4fb1b1cc5cc0e9_20-0"><a href="#cite_note-_fb4fb1b1cc5cc0e9-20">[20]</a></sup>
</p>

<p>Метод передискретизации с помощью <a class="new" href="/w/index.php?title=%D0%9F%D0%BE%D0%BB%D0%B8%D1%84%D0%B0%D0%B7%D0%BD%D1%8B%D0%B9_%D1%84%D0%B8%D0%BB%D1%8C%D1%82%D1%80&amp;action=edit&amp;redlink=1" title="Полифазный фильтр (страница отсутствует)">полифазных фильтров</a> аналогичен предыдущему, с тем отличием, что в нём вместо одного фильтра, работающего на высокой частоте дискретизации, используется несколько фильтров, работающих на низкой частоте. При этом удаётся добиться сокращения количества необходимых вычислений, так как для каждого отсчёта необходимо вычислить выходной сигнал только одного из этих фильтров.<sup class="reference" id="cite_ref-_fb4fb1b1cc5cc0e9_20-1"><a href="#cite_note-_fb4fb1b1cc5cc0e9-20">[20]</a></sup><sup class="reference" id="cite_ref-_9e2a8e04f22564af_23-0"><a href="#cite_note-_9e2a8e04f22564af-23">[23]</a></sup>
</p><p>Полифазный фильтр представляет собой набор небольших фильтров, работающих параллельно, каждый из которых обрабатывает только подмножество отсчётов сигнала (если всего имеется <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle N}" xmlns="http://www.w3.org/1998/Math/MathML">

</p><p>Полифазные фильтры применяются для передискретизации как с целым, так и с дробным коэффициентом.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup>

<p>Передискретизация с помощью <a href="/wiki/%D0%94%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%B5%D0%BE%D0%B1%D1%80%D0%B0%D0%B7%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_%D0%A4%D1%83%D1%80%D1%8C%D0%B5" title="Дискретное преобразование Фурье">ДПФ</a> используется для повышения частоты дискретизации в целое или дробное число раз. Алгоритм работает только с конечными <a href="/wiki/%D0%9E%D1%82%D1%80%D0%B5%D0%B7%D0%BE%D0%BA" title="Отрезок">отрезками</a> сигнала. </p>
<h2><span id=".D0.9F.D1.80.D0.B8.D0.BC.D0.B5.D0.BD.D0.B5.D0.BD.D0.B8.D1.8F"></span><span class="mw-headline" id="Применения">Применения</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%9F%D0%B5%D1%80%D0%B5%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F&amp;veaction=edit&amp;section=9" title="Редактировать раздел «Применения»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%9F%D0%B5%D1%80%D0%B5%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F&amp;action=edit&amp;section=9" title="Редактировать код раздела «Применения»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Широко применяется как аппаратная (на основе специализированных <a class="mw-redirect" href="/wiki/%D0%9C%D0%B8%D0%BA%D1%80%D0%BE%D1%81%D1%85%D0%B5%D0%BC%D0%B0" title="Микросхема">микросхем</a><sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup><sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> или <a class="mw-redirect" href="/wiki/FPGA" title="FPGA">FPGA</a><sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup>), так и программная (на базе <a class="mw-redirect" href="/wiki/%D0%9F%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80" title="Процессор">процессоров</a> общего назначения (см. ниже) или <a href="/wiki/%D0%A6%D0%B8%D1%84%D1%80%D0%BE%D0%B2%D0%BE%D0%B9_%D1%81%D0%B8%D0%B3%D0%BD%D0%B0%D0%BB%D1%8C%D0%BD%D1%8B%D0%B9_%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80" title="Цифровой сигнальный процессор">сигнальных процессоров</a><sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup>) реализация алгоритмов передискретизации.
</p><p>Выбор конкретной реализации алгоритма передискретизации является результатом компромисса между качеством преобразования и его <a href="/wiki/%D0%92%D1%8B%D1%87%D0%B8%D1%81%D0%BB%D0%B8%D1%82%D0%B5%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D1%81%D0%BB%D0%BE%D0%B6%D0%BD%D0%BE%D1%81%D1%82%D1%8C" title="Вычислительная сложность">вычислительной сложностью</a>. Основным параметром, влияющим на эти характеристики, является близость используемых цифровых фильтров к идеальным. Более качественные фильтры требуют больших ресурсов для вычисления.<sup class="reference" id="cite_ref-_fb4936b1cc578418_32-0"><a href="#cite_note-_fb4936b1cc578418-32">[32]</a></sup>
</p><p>На практике передискретизация в большинстве случаев ведёт к потере <a href="/wiki/%D0%98%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%86%D0%B8%D1%8F" title="Информация">информации</a> о сигнале по следующим причинам:
</p>
<ul><li>при уменьшении частоты дискретизации сигнал необходимо отфильтровать с целью удаления высокочастотных спектральных составляющих, которые не соответствуют условиям теоремы Котельникова для новой частоты дискретизации;</li>
<li>неизбежная неидеальность применяемых цифровых фильтров;<sup class="reference" id="cite_ref-_9e2a9204f2256b51_33-0"><a href="#cite_note-_9e2a9204f2256b51-33">[33]</a></sup></li>
<li>вычисления, производимые над <a href="/wiki/%D0%A6%D0%B8%D1%84%D1%80%D0%BE%D0%B2%D0%BE%D0%B9_%D1%81%D0%B8%D0%B3%D0%BD%D0%B0%D0%BB" title="Цифровой сигнал">цифровыми</a> (<a href="/wiki/%D0%9A%D0%B2%D0%B0%D0%BD%D1%82%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_(%D0%BE%D0%B1%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D0%BA%D0%B0_%D1%81%D0%B8%D0%B3%D0%BD%D0%B0%D0%BB%D0%BE%D0%B2)" title="Квантование (обработка сигналов)">квантованными</a> по уровню) сигналами ведут к необратимым ошибкам <a href="/wiki/%D0%9E%D0%BA%D1%80%D1%83%D0%B3%D0%BB%D0%B5%D0%BD%D0%B8%D0%B5" title="Округление">округления</a>.<sup class="reference" id="cite_ref-_7595f909db6ece38_34-0"><a href="#cite_note-_7595f909db6ece38-34">[34]</a></sup></li></ul>
<p>Таким образом, при увеличении частоты дискретизации с последующим уменьшением её до исходного значения качество сигнала будет потеряно (если только высокая частота не кратна низкой).
</p>
<h3><span id=".D0.94.D0.B8.D1.81.D0.BA.D1.80.D0.B5.D1.82.D0.B8.D0.B7.D0.B0.D1.86.D0.B8.D1.8F_.D1.81.D0.B8.D0.B3.D0.BD.D0.B0.D0.BB.D0.BE.D0.B2_.D1.81_.D0.B7.D0.B0.D0.BF.D0.B0.D1.81.D0.BE.D0.BC_.D0.BF.D0.BE_.D1.87.D0.B0.D1.81.D1.82.D0.BE.D1.82.D0.B5_.D0.B4.D0.B8.D1.81.D0.BA.D1.80.D0.B5.D1.82.D0.B8.D0.B7.D0.B0.D1.86.D0.B8.D0.B8"></span><span class="mw-headline" id="Дискретизация_сигналов_с_запасом_по_частоте_дискретизации">Дискретизация сигналов с запасом по частоте дискретизации</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%9F%D0%B5%D1%80%D0%B5%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F&amp;veaction=edit&amp;section=10" title="Редактировать раздел «Дискретизация сигналов с запасом по частоте дискретизации»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%9F%D0%B5%D1%80%D0%B5%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F&amp;action=edit&amp;section=10" title="Редактировать код раздела «Дискретизация сигналов с запасом по частоте дискретизации»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote navigation-not-searchable dabhide">См. также: <a href="/wiki/%D0%94%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F" title="Дискретизация">Дискретизация</a></div>
<p>Дискретизация сигналов с запасом по частоте дискретизации (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">oversampling</span>) подразумевает дискретизацию сигнала на частоте, в несколько раз превышающей <a href="/wiki/%D0%A2%D0%B5%D0%BE%D1%80%D0%B5%D0%BC%D0%B0_%D0%9A%D0%BE%D1%82%D0%B5%D0%BB%D1%8C%D0%BD%D0%B8%D0%BA%D0%BE%D0%B2%D0%B0" title="Теорема Котельникова">частоту Котельникова</a> с последующей децимацией. Такой подход позволяет добиться следующих преимуществ<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup>:
</p>
<ul><li>возможность использовать более простой и дешёвый <a href="/wiki/%D0%A4%D0%B8%D0%BB%D1%8C%D1%82%D1%80_(%D1%8D%D0%BB%D0%B5%D0%BA%D1%82%D1%80%D0%BE%D0%BD%D0%B8%D0%BA%D0%B0)" title="Фильтр (электроника)">аналоговый фильтр</a> для защиты от <a class="mw-redirect" href="/wiki/%D0%9D%D0%B0%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5_%D1%81%D0%BF%D0%B5%D0%BA%D1%82%D1%80%D0%BE%D0%B2" title="Наложение спектров">наложения спектров</a>.</li>
<li>возможность использовать АЦП с меньшей разрядностью.</li></ul>
<p>Аналогичный подход применяется и при восстановлении сигнала по его отсчётам для упрощения аналогового <a class="mw-redirect" href="/wiki/%D0%A4%D0%B8%D0%BB%D1%8C%D1%82%D1%80_%D0%B2%D0%BE%D1%81%D1%81%D1%82%D0%B0%D0%BD%D0%BE%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F" title="Фильтр восстановления">фильтра восстановления</a>.<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup>
</p>
<h3><span id=".D0.9F.D1.80.D0.B8_.D0.BE.D0.B1.D1.80.D0.B0.D0.B1.D0.BE.D1.82.D0.BA.D0.B5_.D0.B7.D0.B2.D1.83.D0.BA.D0.B0"></span><span class="mw-headline" id="При_обработке_звука">При обработке звука</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%9F%D0%B5%D1%80%D0%B5%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F&amp;veaction=edit&amp;section=11" title="Редактировать раздел «При обработке звука»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%9F%D0%B5%D1%80%D0%B5%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F&amp;action=edit&amp;section=11" title="Редактировать код раздела «При обработке звука»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Оборудование, предназначенное для воспроизведения цифрового звука, как правило, рассчитано на вполне определённую частоту дискретизации сигнала непосредственно перед <a class="mw-redirect" href="/wiki/%D0%A6%D0%B8%D1%84%D1%80%D0%BE-%D0%B0%D0%BD%D0%B0%D0%BB%D0%BE%D0%B3%D0%BE%D0%B2%D0%BE%D0%B5_%D0%BF%D1%80%D0%B5%D0%BE%D0%B1%D1%80%D0%B0%D0%B7%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Цифро-аналоговое преобразование">цифро-аналоговым преобразованием</a>. Все звуковые сигналы с другими частотами дискретизации должны быть рано или поздно передискретизованы<sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup>.
</p><p>Передискретизация звукового сигнала на требуемую частоту может осуществляться <a class="mw-redirect" href="/wiki/%D0%9C%D0%B5%D0%B4%D0%B8%D0%B0%D0%BF%D0%BB%D0%B5%D0%B5%D1%80" title="Медиаплеер">медиаплеером</a>, <a href="/wiki/%D0%94%D1%80%D0%B0%D0%B9%D0%B2%D0%B5%D1%80" title="Драйвер">драйвером</a> звуковой карты или самой звуковой картой. Использование программы-проигрывателя для данной цели может быть оправдано, если есть желание избежать аппаратной передискретизации звука (или передискретизации драйвером) с целью добиться более высокого качества (при большей загрузке <a href="/wiki/%D0%A6%D0%B5%D0%BD%D1%82%D1%80%D0%B0%D0%BB%D1%8C%D0%BD%D1%8B%D0%B9_%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80" title="Центральный процессор">центрального процессора</a>). Однако программная передискретизация воспроизводимого материала на частоту, отличную от частоты, поддерживаемой оборудованием, не имеет смысла и приводит только к потере качества сигнала.
</p><p>Существуют программные передискретизаторы звуковых сигналов с <a href="/wiki/%D0%9E%D1%82%D0%BA%D1%80%D1%8B%D1%82%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Открытое программное обеспечение">открытым исходным кодом</a>:
</p>
<ul><li>SRC (Secret Rabbit Code) или libsamplerate<sup class="reference" id="cite_ref-38"><a href="#cite_note-38">[38]</a></sup> — существует <a href="/wiki/%D0%9F%D0%BB%D0%B0%D0%B3%D0%B8%D0%BD" title="Плагин">плагин</a> для <a href="/wiki/Foobar2000" title="Foobar2000">foobar2000</a>;</li>
<li>SSRC<sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[39]</a></sup> — существуют плагины для <a href="/wiki/Winamp" title="Winamp">Winamp</a> и <a href="/wiki/Foobar2000" title="Foobar2000">foobar2000</a>.</li>
<li>SOXR<sup class="reference" id="cite_ref-40"><a href="#cite_note-40">[40]</a></sup> — качественный, быстрый, с минимальными требованиями к ресурсам<sup class="reference" id="cite_ref-41"><a href="#cite_note-41">[41]</a></sup>. Содержит библиотеку для замены SRC, подключается к <a href="/wiki/Foobar2000" title="Foobar2000">foobar2000</a>, используется в <a href="/wiki/FFmpeg" title="FFmpeg">FFmpeg</a> (с версии 1.1.1), <a href="/wiki/Audacity" title="Audacity">Audacity</a> и других проектах.</li></ul>
<p>Также передискретизация поддерживается программами-редакторами звука (такими как <a href="/wiki/Adobe_Audition" title="Adobe Audition">Adobe Audition</a>, Sony <a href="/wiki/Sound_Forge" title="Sound Forge">Sound Forge</a> или <a href="/wiki/Audacity" title="Audacity">Audacity</a>).
</p>
<h3><span id=".D0.9F.D1.80.D0.B8_.D0.BE.D0.B1.D1.80.D0.B0.D0.B1.D0.BE.D1.82.D0.BA.D0.B5_.D0.B8.D0.B7.D0.BE.D0.B1.D1.80.D0.B0.D0.B6.D0.B5.D0.BD.D0.B8.D0.B9"></span><span class="mw-headline" id="При_обработке_изображений">При обработке изображений</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%9F%D0%B5%D1%80%D0%B5%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F&amp;veaction=edit&amp;section=12" title="Редактировать раздел «При обработке изображений»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%9F%D0%B5%D1%80%D0%B5%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F&amp;action=edit&amp;section=12" title="Редактировать код раздела «При обработке изображений»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<figure typeof="mw:File/Thumb"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:Decimation-moire.png"><img class="mw-file-element" data-file-height="384" data-file-width="256" decoding="async" height="384" src="//upload.wikimedia.org/wikipedia/commons/0/02/Decimation-moire.png" width="256"/></a><figcaption>Иллюстрация эффекта <a href="/wiki/%D0%9C%D1%83%D0%B0%D1%80%D0%BE%D0%B2%D1%8B%D0%B9_%D1%83%D0%B7%D0%BE%D1%80" title="Муаровый узор">муара</a>, вызванного <a class="mw-redirect" href="/wiki/%D0%9D%D0%B0%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5_%D1%81%D0%BF%D0%B5%D0%BA%D1%82%D1%80%D0%BE%D0%B2" title="Наложение спектров">наложением спектров</a> при уменьшении разрешения изображения. Сверху — оригинал, снизу слева — изображение, уменьшенное в 2 раза без фильтрации, снизу справа — изображение, уменьшенное в 2 раза с фильтрацией.</figcaption></figure>
<p>Изменение разрешения является одной из распространённых операций обработки изображений. Передискретизация, приближенная к идеальной, не всегда является желательной. Наоборот, результаты работы фильтров с частотной характеристикой, далёкой от идеальной, могут визуально восприниматься как хорошие.<sup class="reference" id="cite_ref-42"><a href="#cite_note-42">[42]</a></sup> Выбор фильтра для передискретизации является результатом компромисса между типом и выраженностью <a class="mw-disambig" href="/wiki/%D0%90%D1%80%D1%82%D0%B5%D1%84%D0%B0%D0%BA%D1%82" title="Артефакт">артефактов</a> и вычислительной сложностью преобразования (актуальной для приложений <a class="mw-redirect" href="/wiki/%D0%A0%D0%B5%D0%B0%D0%BB%D1%8C%D0%BD%D0%BE%D0%B5_%D0%B2%D1%80%D0%B5%D0%BC%D1%8F" title="Реальное время">реального времени</a>).
</p><p>Типичные артефакты при изменении разрешения изображения:<sup class="reference" id="cite_ref-im_12-1"><a href="#cite_note-im-12">[12]</a></sup>:
</p>
<ul><li>Пикселизация (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">blocking</span>);</li>
<li>Звон (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">ringing</span>);</li>
<li><a href="/wiki/%D0%90%D0%BB%D0%B8%D0%B0%D1%81%D0%B8%D0%BD%D0%B3" title="Алиасинг">Алиасинг</a> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">aliasing</span>) и связанный с ним эффект <a href="/wiki/%D0%9C%D1%83%D0%B0%D1%80%D0%BE%D0%B2%D1%8B%D0%B9_%D1%83%D0%B7%D0%BE%D1%80" title="Муаровый узор">муара</a>;</li>
<li>Размытие (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">blurring</span>).</li></ul>
<p>Для передискретизации изображений применяется большое число фильтров, которые можно классифицировать следующим образом<sup class="reference" id="cite_ref-im_12-2"><a href="#cite_note-im-12">[12]</a></sup>:
</p>
<ol><li><i>Фильтры интерполяционного типа</i>, обладающие сравнительно узкой импульсной характеристикой. К ним относятся, в частности, треугольный фильтр, производящий <a href="/wiki/%D0%91%D0%B8%D0%BB%D0%B8%D0%BD%D0%B5%D0%B9%D0%BD%D0%B0%D1%8F_%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D0%BF%D0%BE%D0%BB%D1%8F%D1%86%D0%B8%D1%8F" title="Билинейная интерполяция">билинейную интерполяцию</a> и <a class="mw-redirect" href="/wiki/%D0%9F%D0%BE%D0%BB%D0%B8%D0%BD%D0%BE%D0%BC_%D0%9B%D0%B0%D0%B3%D1%80%D0%B0%D0%BD%D0%B6%D0%B0" title="Полином Лагранжа">полином Лагранжа</a>, с помощью которого можно реализовать <a href="/wiki/%D0%91%D0%B8%D0%BA%D1%83%D0%B1%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B0%D1%8F_%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D0%BF%D0%BE%D0%BB%D1%8F%D1%86%D0%B8%D1%8F" title="Бикубическая интерполяция">бикубическую интерполяцию</a>. Применение таких фильтров позволяет осуществить передискретизацию изображения достаточно быстро.</li>
<li><i>Фильтры с колоколообразной характеристикой</i>, такие как <a href="/wiki/%D0%A4%D0%B8%D0%BB%D1%8C%D1%82%D1%80_%D0%93%D0%B0%D1%83%D1%81%D1%81%D0%B0" title="Фильтр Гаусса">фильтр Гаусса</a>. Эти фильтры хорошо справляются с пикселизацией, звоном и алиасингом, а также отфильтровывают высокочастотные шумы. Их недостаток — заметное размытие изображения.</li>
<li><i>Оконные <a href="/wiki/Sinc-%D1%84%D0%B8%D0%BB%D1%8C%D1%82%D1%80" title="Sinc-фильтр">sinc-фильтры</a></i>. Sinc-фильтр — это идеальный фильтр нижних частот. Как говорилось выше, он не может быть реализован. Однако если частотную характеристику sinc-фильтра умножить на <a class="mw-redirect" href="/wiki/%D0%9E%D0%BA%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D1%8F" title="Оконная функция">оконную функцию</a>, получится реализуемый фильтр с хорошими спектральными свойствами. При применении данных фильтров к изображениям удаётся сохранить относительно высокую чёткость (даже при увеличении разрешения), но может быть сильно заметен эффект звона. Одним из наиболее часто применяемых фильтров данного типа является <a href="/wiki/%D0%A4%D0%B8%D0%BB%D1%8C%D1%82%D1%80_%D0%9B%D0%B0%D0%BD%D1%86%D0%BE%D1%88%D0%B0" title="Фильтр Ланцоша">фильтр Ланцоша</a>.</li></ol>
<p>Нижеприведённые изображения иллюстрируют применение наиболее часто используемых фильтров изменения размера изображений. При увеличении размера изображения без фильтра изображение получается чётким, но пикселизованным. При билинейной интерполяции пикселизация меньше заметна, но изображение размыто. При использовании фильтра Гаусса изображение размыто, но пикселизации практически не заметно. При использовании фильтра Ланцоша пикселизация отсутствует, изображение также размыто и заметен звон (видимый как светлая каёмка вокруг фигур).
</p>

<h3><span id=".D0.9F.D1.80.D0.B8_.D0.BE.D0.B1.D1.80.D0.B0.D0.B1.D0.BE.D1.82.D0.BA.D0.B5_.D1.80.D0.B0.D0.B4.D0.B8.D0.BE.D1.81.D0.B8.D0.B3.D0.BD.D0.B0.D0.BB.D0.BE.D0.B2"></span><span class="mw-headline" id="При_обработке_радиосигналов">При обработке радиосигналов</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%9F%D0%B5%D1%80%D0%B5%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F&amp;veaction=edit&amp;section=13" title="Редактировать раздел «При обработке радиосигналов»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%9F%D0%B5%D1%80%D0%B5%D0%B4%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F&amp;action=edit&amp;section=13" title="Редактировать код раздела «При обработке радиосигналов»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>При <a href="/wiki/%D0%94%D0%B5%D0%BC%D0%BE%D0%B4%D1%83%D0%BB%D1%8F%D1%86%D0%B8%D1%8F" title="Демодуляция">демодуляции</a> цифровых сигналов желательно, чтобы частота дискретизации сигнала была кратна его <a href="/wiki/%D0%A1%D0%BA%D0%BE%D1%80%D0%BE%D1%81%D1%82%D1%8C_%D0%BC%D0%B0%D0%BD%D0%B8%D0%BF%D1%83%D0%BB%D1%8F%D1%86%D0%B8%D0%B8" title="Скорость манипуляции">скорости манипуляции</a> (иначе говоря, чтобы на каждый <a href="/wiki/%D0%A1%D0%B8%D0%BC%D0%B2%D0%BE%D0%BB" title="Символ">символ</a> приходилось одинаковое число отсчётов сигнала). Однако частота дискретизации входного сигнала с <a class="mw-redirect" href="/wiki/%D0%90%D0%A6%D0%9F" title="АЦП">АЦП</a>, как правило, фиксирована, а скорость манипуляции может меняться. Решением является передискретизация сигнала.<sup class="reference" id="cite_ref-43"><a href="#cite_note-43">[43]</a></sup>
</p>
