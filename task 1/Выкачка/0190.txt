<div class="vector-body" id="bodyContent">
<div class="noprint" id="siteSub">Материал из Википедии — свободной энциклопедии</div>

<div id="mw-fr-revisiondetails-wrapper" style="position:relative;"><div class="flaggedrevs_short_details" id="mw-fr-revisiondetails" style="display:none">Текущая версия страницы пока <a href="/wiki/%D0%92%D0%B8%D0%BA%D0%B8%D0%BF%D0%B5%D0%B4%D0%B8%D1%8F:%D0%9F%D1%80%D0%BE%D0%B2%D0%B5%D1%80%D0%BA%D0%B0_%D1%81%D1%82%D0%B0%D1%82%D0%B5%D0%B9/%D0%9F%D0%BE%D1%8F%D1%81%D0%BD%D0%B5%D0%BD%D0%B8%D0%B5_%D0%B4%D0%BB%D1%8F_%D1%87%D0%B8%D1%82%D0%B0%D1%82%D0%B5%D0%BB%D0%B5%D0%B9" title="Википедия:Проверка статей/Пояснение для читателей">не проверялась</a> опытными участниками и может значительно отличаться от <a class="external text" href="https://ru.wikipedia.org/w/index.php?title=Itanium&amp;stable=1">версии</a>, проверенной 28 февраля 2021 года; проверки требуют <a class="external text" href="https://ru.wikipedia.org/w/index.php?title=Itanium&amp;oldid=112681061&amp;diff=cur&amp;diffonly=0">14 правок</a>.</div>

<p><b>Itanium</b> (произносится: <i>Айтэниум</i>) — <a href="/wiki/%D0%9C%D0%B8%D0%BA%D1%80%D0%BE%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80" title="Микропроцессор">микропроцессор</a> с архитектурой <a class="mw-redirect" href="/wiki/IA-64" title="IA-64">IA-64</a> для <a href="/wiki/%D0%A1%D0%B5%D1%80%D0%B2%D0%B5%D1%80_(%D0%B0%D0%BF%D0%BF%D0%B0%D1%80%D0%B0%D1%82%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5)" title="Сервер (аппаратное обеспечение)">серверов</a> и <a href="/wiki/%D0%A0%D0%B0%D0%B1%D0%BE%D1%87%D0%B0%D1%8F_%D1%81%D1%82%D0%B0%D0%BD%D1%86%D0%B8%D1%8F" title="Рабочая станция">рабочих станций</a>, разработанный совместно компаниями <a href="/wiki/Intel" title="Intel">Intel</a> и <a href="/wiki/Hewlett-Packard" title="Hewlett-Packard">Hewlett-Packard</a>. Впервые был представлен 29 мая 2001 года.
</p><p>Производство Itanium прекращено в июле 2002 года одновременно с выходом <b>Itanium 2</b>. В ноябре 2007 года Intel переименовала серию процессоров Itanium 2 обратно в Itanium. По состоянию на весну 2011 года процессорам присваивались четырёхзначные номера, подобно <a class="mw-redirect" href="/wiki/Intel_Xeon" title="Intel Xeon">Intel Xeon</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup>. В мае 2017 года было выпущено последнее семейство процессоров с архитектурой Itanium, предназначенное для Hewlett-Packard Enterprise<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>.
</p><p>В 2019 году Intel объявила, что производство процессоров Itanium завершится 30 января 2020 года, а поставки прекратятся 29 июля 2021 года<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup>. Это произошло в срок<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup>.
</p>

<h2><span id=".D0.98.D1.81.D1.82.D0.BE.D1.80.D0.B8.D1.8F"></span><span class="mw-headline" id="История">История</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=1" title="Редактировать раздел «История»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=1" title="Редактировать код раздела «История»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span id=".D0.9F.D1.80.D0.B5.D0.B8.D0.BC.D1.83.D1.89.D0.B5.D1.81.D1.82.D0.B2.D0.B0"></span><span class="mw-headline" id="Преимущества">Преимущества</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=2" title="Редактировать раздел «Преимущества»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=2" title="Редактировать код раздела «Преимущества»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Itanium был специально разработан для предоставления очень высокого уровня производительности в <a href="/wiki/%D0%9F%D0%B0%D1%80%D0%B0%D0%BB%D0%BB%D0%B5%D0%BB%D1%8C%D0%BD%D1%8B%D0%B5_%D0%B2%D1%8B%D1%87%D0%B8%D1%81%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F" title="Параллельные вычисления">параллельных вычислениях</a> без увеличения частот. Ключевые преимущества архитектуры Itanium:
</p>
<ul><li>Выполнение до 6 инструкций за 1 <a class="mw-redirect" href="/wiki/%D0%9A%D0%BE%D0%BD%D0%B2%D0%B5%D0%B9%D0%B5%D1%80_(%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80)" title="Конвейер (процессор)">цикл</a>.</li>
<li>Выполнение двух <a href="/wiki/SIMD" title="SIMD">SIMD</a>-операций с <a class="mw-redirect" href="/wiki/%D0%A7%D0%B8%D1%81%D0%BB%D0%B0_%D1%81_%D0%BF%D0%BB%D0%B0%D0%B2%D0%B0%D1%8E%D1%89%D0%B5%D0%B9_%D0%B7%D0%B0%D0%BF%D1%8F%D1%82%D0%BE%D0%B9" title="Числа с плавающей запятой">плавающей точкой</a> c 82-битными <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D0%BD%D0%B4" title="Операнд">операндами</a> за один цикл.</li>
<li>Увеличенные вычислительные ресурсы <a href="/wiki/%D0%AF%D0%B4%D1%80%D0%BE_%D0%BC%D0%B8%D0%BA%D1%80%D0%BE%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80%D0%B0" title="Ядро микропроцессора">ядра</a>: 256 <a href="/wiki/%D0%A0%D0%B5%D0%B3%D0%B8%D1%81%D1%82%D1%80_%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80%D0%B0" title="Регистр процессора">регистров</a> (128 целочисленных, 128 вещественных) и 64 предикатных регистра.</li>
<li>Большой <a href="/wiki/%D0%9A%D1%8D%D1%88" title="Кэш">кэш</a>: 24 <a href="/wiki/%D0%9C%D0%B5%D0%B3%D0%B0%D0%B1%D0%B0%D0%B9%D1%82" title="Мегабайт">МБ</a> у двухъядерной версии (по 12 МБ на ядро), предоставляющий данные каждому ядру со скоростью до 48 <a href="/wiki/%D0%A1%D0%BA%D0%BE%D1%80%D0%BE%D1%81%D1%82%D1%8C_%D0%BF%D0%B5%D1%80%D0%B5%D0%B4%D0%B0%D1%87%D0%B8_%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85" title="Скорость передачи данных">ГБ/с</a></li>
<li>За счёт размещения в оперативной или <a href="/wiki/%D0%9A%D1%8D%D1%88_%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80%D0%B0" title="Кэш процессора">кэш-памяти</a> больших объёмов данных (задачи сложного <a href="/wiki/%D0%9C%D0%B0%D1%82%D0%B5%D0%BC%D0%B0%D1%82%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%B0%D0%BD%D0%B0%D0%BB%D0%B8%D0%B7" title="Математический анализ">анализа</a>, <a href="/wiki/Data_mining" title="Data mining">Data mining</a>) позволяет добиться прироста производительности при их обработке.</li>
<li>Большое адресное пространство: 50-битная <a href="/wiki/%D0%90%D0%B4%D1%80%D0%B5%D1%81%D0%B0%D1%86%D0%B8%D1%8F_%D0%BF%D0%B0%D0%BC%D1%8F%D1%82%D0%B8" title="Адресация памяти">адресация физической памяти</a> / 64-битная <a href="/wiki/%D0%91%D0%BB%D0%BE%D0%BA_%D1%83%D0%BF%D1%80%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F_%D0%BF%D0%B0%D0%BC%D1%8F%D1%82%D1%8C%D1%8E" title="Блок управления памятью">адресация виртуальной памяти</a>.</li>
<li>Маленькое, энергоэффективное ядро: с тех пор, как функции распараллеливания передали от Itanium к <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D0%B8%D0%BB%D1%8F%D1%82%D0%BE%D1%80" title="Компилятор">компилятору</a>, в ядре уменьшили количество <a href="/wiki/%D0%A2%D1%80%D0%B0%D0%BD%D0%B7%D0%B8%D1%81%D1%82%D0%BE%D1%80" title="Транзистор">транзисторов</a>.</li>
<li>В приложениях с несколькими <a href="/wiki/%D0%9C%D0%BD%D0%BE%D0%B3%D0%BE%D0%BF%D0%BE%D1%82%D0%BE%D1%87%D0%BD%D0%BE%D1%81%D1%82%D1%8C" title="Многопоточность">потоками задач</a> (за счет истинного <a href="/wiki/%D0%9F%D0%B0%D1%80%D0%B0%D0%BB%D0%BB%D0%B5%D0%BB%D0%B8%D0%B7%D0%BC_(%D0%B8%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%82%D0%B8%D0%BA%D0%B0)" title="Параллелизм (информатика)">параллелизма</a>) производительность их обработки может быть резко увеличена.</li>
<li>Хорошо подходит для создания вычислительных систем, содержащих от 4 до 32 ЦП в <a href="/wiki/%D0%A1%D0%B5%D1%80%D0%B2%D0%B5%D1%80_(%D0%B0%D0%BF%D0%BF%D0%B0%D1%80%D0%B0%D1%82%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5)" title="Сервер (аппаратное обеспечение)">сервере</a>.</li></ul>
<h3><span id=".D0.9D.D0.B5.D0.B4.D0.BE.D1.81.D1.82.D0.B0.D1.82.D0.BA.D0.B8"></span><span class="mw-headline" id="Недостатки">Недостатки</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=3" title="Редактировать раздел «Недостатки»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=3" title="Редактировать код раздела «Недостатки»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<h4><span id=".D0.9D.D0.B5.D1.83.D0.B4.D0.B0.D1.87.D0.BD.D1.8B.D0.B9_.D0.BC.D0.B0.D1.80.D0.BA.D0.B5.D1.82.D0.B8.D0.BD.D0.B3"></span><span class="mw-headline" id="Неудачный_маркетинг">Неудачный маркетинг</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=4" title="Редактировать раздел «Неудачный маркетинг»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=4" title="Редактировать код раздела «Неудачный маркетинг»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Продажи процессора оказались менее успешными, чем предполагалось. Основными причинами этому были проблемы с производительностью и малое количество <a href="/wiki/%D0%9E%D0%BF%D1%82%D0%B8%D0%BC%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F_(%D0%B8%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%82%D0%B8%D0%BA%D0%B0)" title="Оптимизация (информатика)">оптимизированного</a> под Merced (кодовое имя первого поколения процессоров Itanium) <a href="/wiki/%D0%9F%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Программное обеспечение">программного обеспечения</a>.
</p><p>В режиме <a class="mw-redirect" href="/wiki/IA-64" title="IA-64">IA-64</a> это был самый быстрый процессор для <a class="new" href="/w/index.php?title=%D0%92%D1%8B%D1%87%D0%B8%D1%81%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F_%D1%81_%D0%BF%D0%BB%D0%B0%D0%B2%D0%B0%D1%8E%D1%89%D0%B5%D0%B9_%D0%B7%D0%B0%D0%BF%D1%8F%D1%82%D0%BE%D0%B9&amp;action=edit&amp;redlink=1" title="Вычисления с плавающей запятой (страница отсутствует)">вычислений с плавающей запятой</a>, имеющийся на рынке. В то же время в <a href="/wiki/%D0%A6%D0%B5%D0%BB%D0%BE%D1%87%D0%B8%D1%81%D0%BB%D0%B5%D0%BD%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Целочисленное программирование">целочисленных вычислениях</a> он лишь немного превосходил процессоры равной частоты с системой команд <a href="/wiki/X86" title="X86">x86</a>.
При выполнении неоптимизированного под Itanium программного кода для x86-систем его производительность была в 8 раз меньше, чем у x86-процессоров на той же частоте.
Программная эмуляция системы команд x86 работает быстрее, что демонстрирует Itanium 2, который показывает производительность не оптимизированного под Itanium программного кода, сравнимую с x86-процессорами на той же частоте, что не соответствовало ценам на Merced.
</p><p>Выбор Itanium 2 в основном определяется доступностью приложений, и несмотря на то, что аппаратная архитектура позволяет исполнять 32-разрядные приложения, в этом случае не удастся добиться оптимальной производительности и существенных преимуществ по сравнению с более дешёвым семейством <a href="/wiki/Xeon" title="Xeon">Xeon</a>, а принимая во внимание постоянно растущую производительность и функциональность последнего, применение Itanium всё чаще становится неоправданным.
</p><p>Дополнительным важным фактором стал <a href="/wiki/%D0%9F%D1%83%D0%B7%D1%8B%D1%80%D1%8C_%D0%B4%D0%BE%D1%82%D0%BA%D0%BE%D0%BC%D0%BE%D0%B2" title="Пузырь доткомов">коллапс рынка доткомов</a> и соответствующее ему падение продаж серверов.
</p><p>Итаник (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Itanic</span>) — ироничное название, введённое в публикации издания <a href="/wiki/The_Register" title="The Register">The Register</a>. Оно созвучно со словом <a href="/wiki/%D0%A2%D0%B8%D1%82%D0%B0%D0%BD%D0%B8%D0%BA" title="Титаник">Титаник</a>, именующим известный затонувший гигантский трансатлантический <a class="mw-redirect" href="/wiki/%D0%9B%D0%B0%D0%B9%D0%BD%D0%B5%D1%80_(%D1%81%D1%83%D0%B4%D0%BD%D0%BE)" title="Лайнер (судно)">лайнер</a>. Архитектура IA-64 некоторыми журнальными критиками считается неудачей, которая стоила Intel и HP многих миллиардов долларов и не смогла достичь ожидаемых продаж в изначально запланированное время.
</p>
<h4><span id=".D0.A2.D0.B5.D1.85.D0.BD.D0.B8.D1.87.D0.B5.D1.81.D0.BA.D0.B8.D0.B5_.D0.BF.D1.80.D0.BE.D0.B1.D0.BB.D0.B5.D0.BC.D1.8B"></span><span class="mw-headline" id="Технические_проблемы">Технические проблемы</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=5" title="Редактировать раздел «Технические проблемы»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=5" title="Редактировать код раздела «Технические проблемы»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Главным структурным недостатком первых версий Itanium были большие задержки (<a href="/wiki/%D0%A2%D0%B0%D0%B9%D0%BC%D0%B8%D0%BD%D0%B3%D0%B8_(%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%B8%D0%B2%D0%BD%D0%B0%D1%8F_%D0%BF%D0%B0%D0%BC%D1%8F%D1%82%D1%8C)" title="Тайминги (оперативная память)">латентность</a>) <a href="/wiki/%D0%9A%D1%8D%D1%88_%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80%D0%B0" title="Кэш процессора">СОЗУ</a> 3-го уровня. Инженеры Intel, вероятно, надеялись, что большая пропускная способность шины нового процессора скомпенсирует этот недостаток, но задержки были настолько велики, что это действительно замедляло работу кэша вплоть до уровня, когда он был немногим быстрее ОЗУ. Из-за относительно небольшого размера СОЗУ I и II уровней (32 кБ и 96 кБ соответственно) это могло привести к повышенной нагрузке на системную шину.
</p><p>Первоначально процессор планировали выпустить в 1998—1999 годах, но продолжительные задержки проекта и усилившаяся конкуренция со стороны <a href="/wiki/AMD" title="AMD">AMD</a> на рынке процессоров архитектуры x86 привели к тому, что Itanium устарел ещё до своего поступления в продажу. Таким образом, Itanium был неконкурентоспособным уже после выпуска в 2001 году, хотя этого могло не случиться, если бы он был выпущен двумя годами раньше, как и планировалось.
</p>
<h4><span id=".D0.A1.D0.B8.D0.BB.D1.8C.D0.BD.D0.B0.D1.8F_.D0.BA.D0.BE.D0.BD.D0.BA.D1.83.D1.80.D0.B5.D0.BD.D1.86.D0.B8.D1.8F"></span><span class="mw-headline" id="Сильная_конкуренция">Сильная конкуренция</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=6" title="Редактировать раздел «Сильная конкуренция»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=6" title="Редактировать код раздела «Сильная конкуренция»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h4>
<p>На рынке серверов начального уровня Itanium конкурирует с системами на основе <a class="mw-redirect" href="/wiki/AMD64" title="AMD64">AMD64</a> и <a class="mw-redirect" href="/wiki/EM64T" title="EM64T">EM64T</a>, а на рынке высокопроизводительных <a href="/wiki/%D0%A1%D0%B5%D1%80%D0%B2%D0%B5%D1%80_(%D0%B0%D0%BF%D0%BF%D0%B0%D1%80%D0%B0%D1%82%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5)" title="Сервер (аппаратное обеспечение)">серверов</a> — с процессорами архитектуры <a href="/wiki/POWER" title="POWER">POWER</a> от <a href="/wiki/IBM" title="IBM">IBM</a> и <a href="/wiki/SPARC" title="SPARC">SPARC</a> от <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun</a>. Некоторые распространители IA-64, например <a href="/wiki/Dell" title="Dell">Dell</a> и IBM, прекратили или сильно сократили поддержку архитектуры. Dell выбрал AMD64-совместимые процессоры, а IBM продолжает разрабатывать серверы на базе архитектуры POWER, в частности, на процессорах Power и PowerPC.
</p>
<h3><span id=".D0.A0.D1.8B.D0.BD.D0.BE.D1.87.D0.BD.D1.8B.D0.B5_.D1.82.D0.B5.D0.BD.D0.B4.D0.B5.D0.BD.D1.86.D0.B8.D0.B8"></span><span class="mw-headline" id="Рыночные_тенденции">Рыночные тенденции</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=7" title="Редактировать раздел «Рыночные тенденции»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=7" title="Редактировать код раздела «Рыночные тенденции»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Общая выручка от Itanium утроилась в 2003—2004, достигнув 1,4 млрд $ в 2004 году и 2,4 млрд $ в 2005. Участники <a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Itanium Solutions Alliance</span> («Альянс решений Itanium») утверждают, что общие доходы от продаж систем на основе Itanium составляют приблизительно 58 % общих доходов от продаж <a class="mw-redirect" href="/wiki/Sun_SPARC" title="Sun SPARC">Sun SPARC</a> и приблизительно 33 % общего дохода от продаж <a class="mw-redirect" href="/wiki/IBM_Power_2" title="IBM Power 2">IBM Power 2</a>.
</p><p>В ноябре 2005 года основные изготовители серверов на базе Itanium присоединились к Intel и ряду продавцов программного обеспечения, чтобы сформировать <i>Itanium Solutions Alliance</i>, целью которого было совместно способствовать продвижению архитектуры и ускорять процесс переноса программного обеспечения<sup class="reference" id="cite_ref-ISA_5-0"><a href="#cite_note-ISA-5">[5]</a></sup>. Альянс ожидал, что до конца десятилетия его члены инвестируют $10 млрд в решения для Itanium<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup>.
</p>
<h2><span id=".D0.AF.D0.B4.D1.80.D0.B0_.D0.B8_.D0.BF.D0.BE.D0.BA.D0.BE.D0.BB.D0.B5.D0.BD.D0.B8.D1.8F"></span><span class="mw-headline" id="Ядра_и_поколения">Ядра и поколения</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=8" title="Редактировать раздел «Ядра и поколения»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=8" title="Редактировать код раздела «Ядра и поколения»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="2001._Itanium">2001. Itanium</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=9" title="Редактировать раздел «2001. Itanium»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=9" title="Редактировать код раздела «2001. Itanium»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<figure class="mw-default-size mw-halign-left" typeof="mw:File/Thumb"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:Itanium.png"><img class="mw-file-element" data-file-height="280" data-file-width="308" decoding="async" height="200" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/e3/Itanium.png/220px-Itanium.png" srcset="//upload.wikimedia.org/wikipedia/commons/e/e3/Itanium.png 1.5x" width="220"/></a><figcaption>Процессор Intel Itanium</figcaption></figure>
<figure class="mw-halign-right" typeof="mw:File/Thumb"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:Itanium_architecture.svg"><img class="mw-file-element" data-file-height="640" data-file-width="855" decoding="async" height="187" src="//upload.wikimedia.org/wikipedia/commons/thumb/a/ae/Itanium_architecture.svg/250px-Itanium_architecture.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/a/ae/Itanium_architecture.svg/375px-Itanium_architecture.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/a/ae/Itanium_architecture.svg/500px-Itanium_architecture.svg.png 2x" width="250"/></a><figcaption>Архитектура Intel Itanium</figcaption></figure>
<p><a href="/wiki/Hewlett-Packard" title="Hewlett-Packard">HP</a> и <a href="/wiki/Intel" title="Intel">Intel</a> начали сотрудничество в области микропроцессоров в <a href="/wiki/1989_%D0%B3%D0%BE%D0%B4" title="1989 год">1989 году</a>. HP требовался процессор следующего поколения для замены удачных серий <a href="/wiki/%D0%A0%D0%B0%D0%B1%D0%BE%D1%87%D0%B0%D1%8F_%D1%81%D1%82%D0%B0%D0%BD%D1%86%D0%B8%D1%8F" title="Рабочая станция">рабочих станций</a> и <a href="/wiki/%D0%A1%D0%B5%D1%80%D0%B2%D0%B5%D1%80_(%D0%B0%D0%BF%D0%BF%D0%B0%D1%80%D0%B0%D1%82%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5)" title="Сервер (аппаратное обеспечение)">серверов</a>, построенных на базе процессоров с архитектурой <a href="/wiki/PA-RISC" title="PA-RISC">PA-RISC</a>, и компания хотела воспользоваться достижениями и опытом Intel в разработке и производстве <a class="mw-redirect" href="/wiki/%D0%9C%D0%B8%D0%BA%D1%80%D0%BE%D1%87%D0%B8%D0%BF" title="Микрочип">микрочипов</a>.
</p><p>Новый процессор должен был использовать набор инструкций с явным параллелизмом (<a class="mw-redirect" href="/wiki/EPIC_(%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80)" title="EPIC (процессор)">EPIC</a>), в котором <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D0%B8%D0%BB%D1%8F%D1%82%D0%BE%D1%80" title="Компилятор">компилятор</a> должен выстраивать инструкции для параллельного исполнения. Были добавлены набор команд ISA (Instruction Set Architecture) и возможности для <a href="/wiki/%D0%A1%D0%BE%D0%B2%D0%BC%D0%B5%D1%81%D1%82%D0%B8%D0%BC%D0%BE%D1%81%D1%82%D1%8C_(%D0%B8%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%82%D0%B8%D0%BA%D0%B0)" title="Совместимость (информатика)">совместимости</a> с приложениями, разработанными как для Intel <a href="/wiki/X86" title="X86">x86</a>, так и для PA-RISC. Ожидалось, что разрабатываемый процессор будет доминировать на рынке серверов, рабочих станций и, возможно, даже настольных <a href="/wiki/%D0%9F%D0%B5%D1%80%D1%81%D0%BE%D0%BD%D0%B0%D0%BB%D1%8C%D0%BD%D1%8B%D0%B9_%D0%BA%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80" title="Персональный компьютер">ПК</a>, вытеснив вездесущую архитектуру x86. Предполагалось, что конкуренты Intel, в первую очередь <a href="/wiki/AMD" title="AMD">AMD</a>, не смогут повторить новую архитектуру.
</p><p>Первая версия процессора под кодовым именем <i>Merced</i> (назван в честь города вблизи <a href="/wiki/%D0%A1%D0%B0%D0%BD-%D0%A5%D0%BE%D1%81%D0%B5_(%D0%9A%D0%B0%D0%BB%D0%B8%D1%84%D0%BE%D1%80%D0%BD%D0%B8%D1%8F)" title="Сан-Хосе (Калифорния)">Сан-Хосе</a>, <a class="mw-redirect" href="/wiki/%D0%A1%D0%A8%D0%90" title="США">США</a>) поступила в продажу в июне <a href="/wiki/2001_%D0%B3%D0%BE%D0%B4" title="2001 год">2001 года</a>. В отличие от всех последующих поколений, он мог производить четыре целочисленных или три вещественных инструкции за один такт.
Произведённый по <a class="new" href="/w/index.php?title=180-%D0%BD%D0%BC&amp;action=edit&amp;redlink=1" title="180-нм (страница отсутствует)">180-нм</a> технологии с площадью кристалла в 250 мм², напряжением на ядре 2 В и тепловыделением в 150 Вт, он работал на частотах 733 и 800 МГц с частотой системной шины 266 МГц, кэш-памятью III уровня размером в 2 или в 4 МБ. Поддерживались <a href="/wiki/SIMD" title="SIMD">SIMD</a> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Single Instruction Multiple Data</span>)-инструкции <a href="/wiki/MMX" title="MMX">MMX</a> и <a href="/wiki/SSE" title="SSE">SSE</a>. Рассчитан на установку в <a class="new" href="/w/index.php?title=Slot_M&amp;action=edit&amp;redlink=1" title="Slot M (страница отсутствует)">Slot M</a> и память <a href="/wiki/SDRAM" title="SDRAM">SDRAM</a> (PC 100). Стоили процессоры от 1200 $ до более чем 4000 $.
</p>

<h4><span class="mw-headline" id="McKinley">McKinley</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=11" title="Редактировать раздел «McKinley»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=11" title="Редактировать код раздела «McKinley»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Первый Itanium 2, получивший кодовое имя <i>McKinley</i>, был выпущен совместными усилиями HP и Intel. Это позволило решить многие из проблем с производительностью оригинального процессора Itanium, которые по большей части вызвала неэффективная подсистема памяти. McKinley содержит 221 миллион транзисторов, 25 миллионов из которых были для <a class="mw-redirect" href="/wiki/%D0%90%D0%9B%D0%A3" title="АЛУ">блоков логики</a>, имеет размеры 19,5 мм на 21,6 мм (т. о. его площадь составила 421 мм²) и производился по технологическому процессу 180-нм объёмного <a href="/wiki/%D0%9A%D0%9C%D0%9E%D0%9F" title="КМОП">КМОП</a> с шестью уровнями алюминиевой металлизации<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> McKinley, как и все последующие ядра Itanium, может производить шесть целочисленных или две (четыре при «зацеплении») вещественных инструкции за один такт.
</p><p>Начиная с McKinley процессоры Itanium стали показывать конкурентоспособные производительность и тепловыделение. С проникновением на рынок постепенно улучшилась поддержка архитектуры и ПО для неё, что отразилось в значительном росте продаж, начиная с 2004—2005 годов.
</p>
<h4><span class="mw-headline" id="Madison">Madison</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=12" title="Редактировать раздел «Madison»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=12" title="Редактировать код раздела «Madison»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h4>
<p>В дальнейшем Intel и HP продолжили разработку архитектуры, результатом чего стало появление в 2003 году  значительно доработанного двухъядерного McKinley. В производстве использовался <a href="/wiki/%D0%A2%D0%B5%D1%85%D0%BD%D0%BE%D0%BB%D0%BE%D0%B3%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81_%D0%B2_%D1%8D%D0%BB%D0%B5%D0%BA%D1%82%D1%80%D0%BE%D0%BD%D0%BD%D0%BE%D0%B9_%D0%BF%D1%80%D0%BE%D0%BC%D1%8B%D1%88%D0%BB%D0%B5%D0%BD%D0%BD%D0%BE%D1%81%D1%82%D0%B8" title="Технологический процесс в электронной промышленности">техпроцесс</a> 130 нм, ставший базой для всех новых процессоров Itanium до выхода в июне 2006 года процессора <i>Montecito</i>.
Характеристики <i>Madison</i>:
</p>
<ul><li>норма техпроцесса — 0,13 мкм,</li>
<li>тактовые частоты (от) порядка 1,5 ГГц,</li>
<li>кэш L3 объёмом до 6 Мб,</li>
<li>размер кристалла до 374 мм²,</li>
<li>количество транзисторов — до 410 млн,</li>
<li>рассеиваемая мощность — до 130 Вт.</li></ul>
<p>Характеристики, ожидаемые Intel от Madison, позволяли поднять производительность на 50 % по сравнению с предыдущим Itanium 2. Ожидалось, что процессоры Madison будут совместимы с чипами Itanium 2 и апгрейд систем будет достаточно экономным (Itanium 2 стоит 4226 $, первое поколение Itanium обходилось по 4227 $)<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup>
</p><p>Стало известно, что на рынок будут выпущены три модификации чипа Madison, с различными тактовыми частотами и размером кэша.
Madison 9M — топовая серия процессора; Deerfield — экономичная версия чипа Madison для стоечных серверов<sup class="reference" id="cite_ref-rol.ru_9-0"><a href="#cite_note-rol.ru-9">[9]</a></sup>.
</p><p>Изначально Intel планировала выпустить в 2004 году процессор Montecito, но задержала его выход на год, до 2005 года, решив внести в его архитектуру изменения (новая версия должна была иметь двойное 64-битное ядро, выполненное по 0,09-микронной технологии).
Чтобы заполнить пустоту, вызванную отсрочкой выхода процессора под кодовым названием Montecito, в своем плане по выпуску процессоров Intel заявил о новом процессоре под кодовым названием Madison 9M.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup>
</p><p>Производительность Itanium 2 Madison 9M, совместимого с процессорными разъемами и <a href="/wiki/%D0%A7%D0%B8%D0%BF%D1%81%D0%B5%D1%82" title="Чипсет">чипсетами</a> HP <a href="/wiki/Zx1" title="Zx1">zx1</a> и <a class="new" href="/w/index.php?title=Sx1000&amp;action=edit&amp;redlink=1" title="Sx1000 (страница отсутствует)">sx1000</a> средних и старших моделей ранее выпускаемых серверов серии <a class="new" href="/w/index.php?title=HP_Integrity&amp;action=edit&amp;redlink=1" title="HP Integrity (страница отсутствует)">HP Integrity</a> (rx7620-16, rx8620-32 и самый мощный, <a href="/wiki/HP_Superdome" title="HP Superdome">HP Superdome</a> — системы с числом процессоров от 16 до 128) — 51,9 <a class="mw-redirect" href="/wiki/%D0%A2%D0%B5%D1%80%D0%B0%D1%84%D0%BB%D0%BE%D0%BF%D1%81" title="Терафлопс">терафлопс</a> на частоте 1,6 ГГц (против 42,7 терафлопс процессоров <a class="mw-redirect" href="/wiki/SGI" title="SGI">SGI</a>). К середине 2005 года на платформу Intel Itanium 2 ожидался перевод серверов <a class="new" href="/w/index.php?title=HP_NonStop&amp;action=edit&amp;redlink=1" title="HP NonStop (страница отсутствует)">HP NonStop</a>. Годовой объём рынка <a href="/wiki/RISC" title="RISC">RISC</a>-систем, с которыми конкурируют серверы серии HP Integrity, оценивается аналитиками IDC приблизительно в 20 млрд долл.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup>
</p>
<figure class="mw-halign-right" typeof="mw:File/Thumb"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:Architettura_Itanium_2.png"><img class="mw-file-element" data-file-height="1412" data-file-width="1637" decoding="async" height="216" src="//upload.wikimedia.org/wikipedia/commons/thumb/1/1a/Architettura_Itanium_2.png/250px-Architettura_Itanium_2.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/1/1a/Architettura_Itanium_2.png/375px-Architettura_Itanium_2.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/1/1a/Architettura_Itanium_2.png/500px-Architettura_Itanium_2.png 2x" width="250"/></a><figcaption>Архитектура процессора <i>Itanium 2</i></figcaption></figure>
<p>Дополнительно к этой же серии планировался выпуск lower-end-решения для двухпроцессорных систем с низким номиналом питающего напряжения<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup>.
</p>

<p>Заказы на Itanium 2 Madison 9M, выпускавшиеся по 130-нм техпроцессу, принимались до 16 ноября, а последние партии планировалось отгрузить не позднее 16 февраля 2008 года. При этом ОЕМ-версии были доступны до 21 мая 2010 года<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup>.
</p>
<h4><span class="mw-headline" id="Montecito">Montecito</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=13" title="Редактировать раздел «Montecito»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=13" title="Редактировать код раздела «Montecito»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Первый двухъядерный Itanium с кодовым именем Montecito, рекламировавшийся как серия Itanium 2 9000, был отгружен Intel заказчикам в июле <a href="/wiki/2006_%D0%B3%D0%BE%D0%B4" title="2006 год">2006 года</a>. Intel и её партнёры обещали увеличение производительности процессора в 2 раза, в то же время снижение энергопотребления — приблизительно на 20 % по сравнению с одноядерным предшественником<sup class="reference" id="cite_ref-intel_20050707_15-0"><a href="#cite_note-intel_20050707-15">[15]</a></sup><sup>,</sup><sup class="reference" id="cite_ref-CW1_16-0"><a href="#cite_note-CW1-16">[16]</a></sup>
Судя по первым опубликованным тестам производительности, похоже, что в основном эти заявления подтверждаются<sup class="reference" id="cite_ref-tpcc_17-0"><a href="#cite_note-tpcc-17">[17]</a></sup>.
</p><p>Чтобы максимально эффективно использовать более 100 миллионов транзисторов в ядре Montecito, Intel в 90-нм техпроцессе решила добавить седьмой металлический слой и технологию «растягивания кремния»<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup>.
</p><p>Ожидалось, что каждое процессорное ядро будет обладать своим собственным кэшем первого, второго и третьего уровня, при этом суммарный объём кэш-памяти составит не менее 18 МБ на корпус, а весь конструктив будет содержать порядка 1 млрд транзисторов.
</p><p>С выпуском Montecito ожидался дебют новой технологии внутренней «распределительной» (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">"arbiter"</span>) шины (системного общепроцессорного интерфейса с пропускной способностью до 6,4 Гб/с и производительностью до 400 млн транзакций в секунду), предназначенной для управления двумя и более процессорными ядрами в едином корпусе. По словам представителей Intel, применение такой шины позволит удвоить объём кэш-памяти, поддерживаемой каждым процессором<sup class="reference" id="cite_ref-rol.ru_9-1"><a href="#cite_note-rol.ru-9">[9]</a></sup>.
</p><p>Официальный план выпуска процессоров Intel включает двухъядерные процессоры следующего поколения, произведенные по 65-нм технологическому процессу (Montecito произведен по 90-нм технологии) и два будущих четырёхъядерных процессора. Стоит заметить, что один или более из этих процессоров будут произведены с использованием 45-нм технологического процесса<sup class="reference" id="cite_ref-intel_45nm_19-0"><a href="#cite_note-intel_45nm-19">[19]</a></sup>.
</p>
<h4><span class="mw-headline" id="Montvale">Montvale</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=14" title="Редактировать раздел «Montvale»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=14" title="Редактировать код раздела «Montvale»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Выпуск серии Itanium 2 9100 (кодовое имя Montvale, ядро — Montecito, техпроцесс — 90 нм, выход на рынок ожидался в конце 2007 года, замена в 2008 году 4- и 8-ядерным Tukwila (65-нм)) Intel начала в ноябре <a href="/wiki/2007_%D0%B3%D0%BE%D0%B4" title="2007 год">2007 года</a><sup class="reference" id="cite_ref-IW1_20-0"><a href="#cite_note-IW1-20">[20]</a></sup>.
На май 2009 года был запланирован выпуск <i>Tukwila</i>, его наследника, но дата была снова исправлена выпуском версии для <a href="/wiki/OEM" title="OEM">сборщиков систем</a>, планируемым на первый квартал 2010 года<sup class="reference" id="cite_ref-INQ09_21-0"><a href="#cite_note-INQ09-21">[21]</a></sup>.
</p>
<h3><span class="mw-headline" id="2010._Itanium_9300_Tukwila">2010. Itanium 9300 Tukwila</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=15" title="Редактировать раздел «2010. Itanium 9300 Tukwila»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=15" title="Редактировать код раздела «2010. Itanium 9300 Tukwila»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote navigation-not-searchable">Основная статья: <b><a href="/wiki/Tukwila_(%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80)" title="Tukwila (процессор)">Tukwila (процессор)</a></b></div>
<p>В марте 2005 года Intel анонсировал работы над новым процессором Itanium с кодовым именем <a href="/wiki/Tukwila_(%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80)" title="Tukwila (процессор)">Tukwila</a>, представление которого было запланировано на 2007 год. Tukwila имеет четыре процессорных ядра и заменит шину процессора Itanium новой, <a class="mw-redirect" href="/wiki/Common_System_Interface" title="Common System Interface">Common System Interface</a>, которая будет также использоваться процессорами <a href="/wiki/Xeon" title="Xeon">Xeon</a><sup class="reference" id="cite_ref-CSI_22-0"><a href="#cite_note-CSI-22">[22]</a></sup> Спустя год, Intel перенесла сроки начала отгрузки процессоров Tukwila заказчику на 2008 год<sup class="reference" id="cite_ref-zdnet_2005_slip_23-0"><a href="#cite_note-zdnet_2005_slip-23">[23]</a></sup>
</p><p>Серия процессоров Itanium 9300, получившая кодовое имя Tukwila, представлена 8 февраля 2010 года<sup class="reference" id="cite_ref-eweek-tukwila_24-0"><a href="#cite_note-eweek-tukwila-24">[24]</a></sup>
</p><p>Процессор производится по 65 нм технологии, содержит четыре ядра<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup>, до 24 МБ кэша на кристалле процессора, поддерживает технологию <a class="mw-redirect" href="/wiki/Hyper-Threading" title="Hyper-Threading">Hyper-Threading</a> и имеет интегрированный <a href="/wiki/%D0%91%D0%BB%D0%BE%D0%BA_%D1%83%D0%BF%D1%80%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F_%D0%BF%D0%B0%D0%BC%D1%8F%D1%82%D1%8C%D1%8E" title="Блок управления памятью">контроллер памяти</a>. Реализовано исправление ошибок, возникающих во время циклов обмена с памятью. Также, в процессоре Tukwila реализована поддержка новой процессорной шины <a class="mw-redirect" href="/wiki/QPI" title="QPI">QPI</a>, что является серьёзным прорывом для построенных на базе Itanium систем. Пиковая межпроцессорная производительность составила 96 ГБ/сек, пиковая производительность подсистемы памяти 34 ГБ/сек. Совместно с <a class="mw-redirect" href="/wiki/QuickPath" title="QuickPath">QuickPath</a>, процессор содержит интегрированный контроллер памяти и интерфейс памяти напрямую использует интерфейс QPI для прямого доступа к другим процессорам и контроллерам ввода-вывода (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">I/O hubs</span>). QuickPath используемый также в процессорных микроархитектурах Intel <a href="/wiki/Nehalem" title="Nehalem">Nehalem</a>, делая вероятным тот факт, что процессоры Tukwila и Nehalem смогут использовать один <a href="/wiki/%D0%A7%D0%B8%D0%BF%D1%81%D0%B5%D1%82" title="Чипсет">чипсет</a><sup class="reference" id="cite_ref-Kittson_26-0"><a href="#cite_note-Kittson-26">[26]</a></sup>. <i>Tukwila</i> объединит четыре диспетчера памяти, каждый из которых через отдельные контроллеры памяти<sup class="reference" id="cite_ref-TukwilaDelay_27-0"><a href="#cite_note-TukwilaDelay-27">[27]</a></sup> поддерживает множество модулей памяти <a href="/wiki/DDR3_SDRAM" title="DDR3 SDRAM">DDR3</a> <a href="/wiki/DIMM" title="DIMM">DIMM</a>, подобно выходящему на рынок процессору <i><a href="/wiki/Xeon" title="Xeon">Xeon</a></i>, с кодовым именем <i>Beckton</i>, основанному на ядре <i>Nehalem</i><sup class="reference" id="cite_ref-DailyTech_Server_28-0"><a href="#cite_note-DailyTech_Server-28">[28]</a></sup>.
</p><p>Первый в мире микропроцессор, содержащий 2 миллиарда транзисторов<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup>.
</p>

<h2><span id="Itanium_.D0.B2_.D1.81.D1.83.D0.BF.D0.B5.D1.80.D0.BA.D0.BE.D0.BC.D0.BF.D1.8C.D1.8E.D1.82.D0.B5.D1.80.D0.B0.D1.85"></span><span class="mw-headline" id="Itanium_в_суперкомпьютерах">Itanium в <a href="/wiki/%D0%A1%D1%83%D0%BF%D0%B5%D1%80%D0%BA%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80" title="Суперкомпьютер">суперкомпьютерах</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=19" title="Редактировать раздел «Itanium в суперкомпьютерах»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=19" title="Редактировать код раздела «Itanium в суперкомпьютерах»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<figure class="mw-halign-right" typeof="mw:File/Thumb"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:Columbia_Supercomputer_-_NASA_Advanced_Supercomputing_Facility.jpg"><img class="mw-file-element" data-file-height="1524" data-file-width="2100" decoding="async" height="203" src="//upload.wikimedia.org/wikipedia/commons/thumb/1/1f/Columbia_Supercomputer_-_NASA_Advanced_Supercomputing_Facility.jpg/280px-Columbia_Supercomputer_-_NASA_Advanced_Supercomputing_Facility.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/1/1f/Columbia_Supercomputer_-_NASA_Advanced_Supercomputing_Facility.jpg/420px-Columbia_Supercomputer_-_NASA_Advanced_Supercomputing_Facility.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/1/1f/Columbia_Supercomputer_-_NASA_Advanced_Supercomputing_Facility.jpg/560px-Columbia_Supercomputer_-_NASA_Advanced_Supercomputing_Facility.jpg 2x" width="280"/></a><figcaption>Суперкомпьютер «Колумбия»</figcaption></figure>
<p>Обновлённый суперкомпьютер <a class="new" href="/w/index.php?title=%D0%A2%D0%B8%D1%85%D0%BE%D0%BE%D0%BA%D0%B5%D0%B0%D0%BD%D1%81%D0%BA%D0%B0%D1%8F_%D1%81%D0%B5%D0%B2%D0%B5%D1%80%D0%BE-%D0%B7%D0%B0%D0%BF%D0%B0%D0%B4%D0%BD%D0%B0%D1%8F_%D0%BD%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%B0%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D0%BB%D0%B0%D0%B1%D0%BE%D1%80%D0%B0%D1%82%D0%BE%D1%80%D0%B8%D1%8F_%D0%A1%D0%A8%D0%90&amp;action=edit&amp;redlink=1" title="Тихоокеанская северо-западная национальная лаборатория США (страница отсутствует)">Тихоокеанской северо-западной национальной лаборатории</a> <a class="mw-redirect" href="/wiki/%D0%94%D0%B5%D0%BF%D0%B0%D1%80%D1%82%D0%B0%D0%BC%D0%B5%D0%BD%D1%82_%D1%8D%D0%BD%D0%B5%D1%80%D0%B3%D0%B5%D1%82%D0%B8%D0%BA%D0%B8_%D0%A1%D0%A8%D0%90" title="Департамент энергетики США">Департамента энергетики США</a> (2000 процессоров Itanium Madison 1,5 ГГц, 7 Тб оперативной памяти, производительность 11,8 <a class="mw-redirect" href="/wiki/%D0%A2%D0%B5%D1%80%D0%B0%D1%84%D0%BB%D0%BE%D0%BF%D1%81" title="Терафлопс">терафлопса</a>) стоимостью 24,5 млн долларов США, используемый для проведения исследований в области химии и биологии, а также — для <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80%D0%BD%D0%BE%D0%B5_%D0%BC%D0%BE%D0%B4%D0%B5%D0%BB%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Компьютерное моделирование">моделирования</a> распространения в почве радиоактивных веществ, в 2003 году обошёл прежнего рекордсмена, Linux-кластер из <a href="/wiki/%D0%9B%D0%B8%D0%B2%D0%B5%D1%80%D0%BC%D0%BE%D1%80%D1%81%D0%BA%D0%B0%D1%8F_%D0%BD%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%B0%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D0%BB%D0%B0%D0%B1%D0%BE%D1%80%D0%B0%D1%82%D0%BE%D1%80%D0%B8%D1%8F" title="Ливерморская национальная лаборатория">Ливерморской национальной лаборатории</a>.<sup class="reference" id="cite_ref-63"><a href="#cite_note-63">[63]</a></sup>
</p><p>Самой производительной из Itanium-систем является суперкомпьютер <a href="/wiki/Columbia_(%D1%81%D1%83%D0%BF%D0%B5%D1%80%D0%BA%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80)" title="Columbia (суперкомпьютер)">«Columbia»</a>, названный в честь экипажа, погибшего при крушении шаттла <a href="/wiki/%D0%9A%D0%BE%D0%BB%D1%83%D0%BC%D0%B1%D0%B8%D1%8F_(%D1%88%D0%B0%D1%82%D1%82%D0%BB)" title="Колумбия (шаттл)">«Колумбия»</a>. Он установлен в <a class="mw-redirect" href="/wiki/NASA" title="NASA">NASA</a>, и по состоянию на май 2012 года содержит 13 824 процессорных ядра и имеет пиковую производительность R<sub>max</sub> = 66,57 <a class="mw-redirect" href="/wiki/%D0%A2%D0%B5%D1%80%D0%B0%D1%84%D0%BB%D0%BE%D0%BF%D1%81" title="Терафлопс">терафлопса</a>.<sup class="reference" id="cite_ref-top500_64-0"><a href="#cite_note-top500-64">[64]</a></sup>
Система «Columbia» была построена компанией <a class="mw-redirect" href="/wiki/SGI" title="SGI">SGI</a> из 4 узлов <a class="mw-redirect" href="/wiki/SGI_Altix" title="SGI Altix">SGI Altix</a> 3700 и 4700 и работает под управлением SUSE Linux Enterprise Server 9.<sup class="reference" id="cite_ref-65"><a href="#cite_note-65">[65]</a></sup><sup class="reference" id="cite_ref-66"><a href="#cite_note-66">[66]</a></sup>
</p><p>В 39-м списке <a class="mw-redirect" href="/wiki/TOP500" title="TOP500">TOP500</a> самых самых производительных компьютеров (июнь 2012 года) «Columbia» остаётся единственной системой на базе Itanium и занимает 430-ю строку.
</p>
<h2><span id=".D0.90.D0.BF.D0.BF.D0.B0.D1.80.D0.B0.D1.82.D0.BD.D0.B0.D1.8F_.D0.BF.D0.BE.D0.B4.D0.B4.D0.B5.D1.80.D0.B6.D0.BA.D0.B0"></span><span class="mw-headline" id="Аппаратная_поддержка">Аппаратная поддержка</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=20" title="Редактировать раздел «Аппаратная поддержка»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=20" title="Редактировать код раздела «Аппаратная поддержка»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Архитектуру Itanium поддержали мировые лидеры в производстве серверов: <a href="/wiki/Bull" title="Bull">Bull</a>, <a href="/wiki/Fujitsu" title="Fujitsu">Fujitsu</a>, <a href="/wiki/Fujitsu_Siemens_Computers" title="Fujitsu Siemens Computers">Fujitsu Siemens Computers</a>, <a href="/wiki/Hitachi" title="Hitachi">Hitachi</a>, <a href="/wiki/Hewlett-Packard" title="Hewlett-Packard">HP</a>, <a href="/wiki/NEC" title="NEC">NEC</a>, <a class="mw-redirect" href="/wiki/SGI" title="SGI">SGI</a> и <a href="/wiki/Unisys" title="Unisys">Unisys</a>. Эти производители присоединились к Intel и множеству разработчиков ПО для создания <a class="external text" href="https://web.archive.org/web/20080908020921/http://www.itaniumsolutionsalliance.org/home/" rel="nofollow">Itanium Solutions Alliance</a> с целью продвижения архитектуры и ускорения темпов <a href="/wiki/%D0%9F%D0%BE%D1%80%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B3%D0%BE_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D1%8F" title="Портирование программного обеспечения">портирования</a> ПО. Тем не менее, ведущие производители серверов, кроме Fujitsu и HP, мало или совсем не заинтересованы в развитии рынка серверов на базе Itanium. <a href="/wiki/IBM" title="IBM">IBM</a> и <a href="/wiki/Dell" title="Dell">Dell</a> которые некоторое время продавали системы на основе Itanium, больше этим не занимаются, и не особо поддерживая 64-битные чипы, предпочли продавать более простые системы, а сконцентрированная на платформах <a href="/wiki/SPARC" title="SPARC">SPARC</a> и <a class="mw-redirect" href="/wiki/AMD64" title="AMD64">AMD64</a> <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun</a> никогда не продавала системы на основе Itanium.
</p><p>В данный момент основные продавцы Itanium сосредоточены на совершенствовании многопроцессорных серверов для работы с часто обновляемыми данными в критически важных коммерческих и технических приложениях. Системы на основе Itanium часто продвигаются как открытая, стандартная для промышленности альтернатива <a href="/wiki/RISC" title="RISC">RISC</a> и <a href="/wiki/%D0%9C%D0%B5%D0%B9%D0%BD%D1%84%D1%80%D0%B5%D0%B9%D0%BC" title="Мейнфрейм">мейнфрейм</a>-системам, производимым IBM и Sun, хотя процессор Itanium разработан, произведён и продаётся исключительно Intel. Сегодня системы на основе Itanium начинаются с 2-процессорных серверов и <a class="mw-redirect" href="/wiki/Blade-%D1%81%D0%B5%D1%80%D0%B2%D0%B5%D1%80" title="Blade-сервер">blade-серверов</a> и кончаются 512-процессорными серверами со 128 Тб общей разделяемой памяти.
</p><p>Компания Platform Solutions, Inc. предлагает системы на основе Itanium, совместимые с <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Операционная система">ОС</a> IBM <a href="/wiki/Z/OS" title="Z/OS">z/OS</a> (а также с <a class="mw-redirect" href="/wiki/GNU/Linux" title="GNU/Linux">GNU/Linux</a> и <a class="mw-redirect" href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>). Таким образом, это первое за многие годы, произведённое не IBM аппаратное обеспечение, появившееся на рынке мейнфреймов. Компания заявляет, что с новым двухъядерным Itanium их системы могут достигнуть производительности, удовлетворяющей требованиям z/OS более, чем на 90 %.
</p><p>В феврале <a href="/wiki/2003_%D0%B3%D0%BE%D0%B4" title="2003 год">2003 года</a> Hewlett-Packard анонсировала масштабируемый чипсет sx1000, который позволяет осуществлять высокоскоростной обмен данными между процессорами, устройствами ввода-вывода и памятью по 128-разрядной системной шине Itanium 2. Кроме Madison/Deerfield и Montecito, чипсет sx1000 совместим с <a href="/wiki/PA-RISC" title="PA-RISC">процессорами PA-8800 и PA-8900 самой</a> Hewlett-Packard. Максимальный размер оперативной памяти, поддерживаемой этим чипсетом, может теоретически равняться 2 Тб, а число поддерживаемых процессоров — 128. Применение — серверы Hewlett-Packard среднего и высшего уровня выпуска второй половины 2003 года. Кроме того, Николай Зезюлинский, менеджер российского представительства Hewlett-Packard по работе с партнёрами, сообщил что согласно плану компании по развитию серверных систем, Itanium станет единственным семейством ЦП и на него планируется постепенно перевести все 64-разрядные серверы<sup class="reference" id="cite_ref-67"><a href="#cite_note-67">[67]</a></sup>.
</p><p><a href="/wiki/Fujitsu" title="Fujitsu">Fujitsu</a>, выпустившая в <a href="/wiki/2005_%D0%B3%D0%BE%D0%B4" title="2005 год">2005 году</a> с опозданием на год обещанные серверы, признаёт, что вопреки её надеждам, рынок ещё не созрел для больших систем Itanium. Они предполагают, что старшей модели (PrimeQuest 480) 32-процессорной системы, оборудованой двухъядерными Montecito, 512 ГБ памяти, 32 <a href="/wiki/%D0%96%D1%91%D1%81%D1%82%D0%BA%D0%B8%D0%B9_%D0%B4%D0%B8%D1%81%D0%BA" title="Жёсткий диск">винчестерами</a> и 16 портами <a href="/wiki/SCSI" title="SCSI">SCSI</a>, будет достаточно для большинства потребителей. Младшая модель той же линейки, PrimeQuest 440, имеет «всего лишь» 16 процессоров, 256 ГБ памяти, 16 винчестеров и 8 портов SCSI. Эти системы стали дополнением к уже существующим в производстве Fujitsu одно-четырёхпроцессорным серверам, которые не очень хорошо расходятся (в 2003 году, в соответствии с данными Gartner, Fujitsu отгрузила всего два сервера Itanium, затем в 2004 году продажи возросли до 233), и до сентября 2005 года, когда Fujitsu запустит поддержку SuSE Linux и <a href="/wiki/Windows_Server_2003" title="Windows Server 2003">Windows Server 2003</a> Datacenter Edition продавались только с <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Операционная система">ОС</a> Red Hat Linux. Аппараты Fujitsu (у которых уже есть 64-процессорный сервер) сейчас идут впереди сложного Itanium-оборудования от HP.<sup class="reference" id="cite_ref-68"><a href="#cite_note-68">[68]</a></sup>.
</p>
<h2><span id=".D0.9F.D1.80.D0.BE.D0.B3.D1.80.D0.B0.D0.BC.D0.BC.D0.BD.D0.B0.D1.8F_.D0.BF.D0.BE.D0.B4.D0.B4.D0.B5.D1.80.D0.B6.D0.BA.D0.B0"></span><span class="mw-headline" id="Программная_поддержка">Программная поддержка</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Itanium&amp;veaction=edit&amp;section=21" title="Редактировать раздел «Программная поддержка»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Itanium&amp;action=edit&amp;section=21" title="Редактировать код раздела «Программная поддержка»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Сейчас Itanium поддерживается 10 операционными системами, включая Windows и различные дистрибутивы GNU/Linux и <a class="mw-redirect" href="/wiki/UNIX" title="UNIX">UNIX</a>, под которыми более 8000 приложений написаны под Itanium и доступны для работы (по состоянию на июнь 2006). Intel утверждает, что в течение последних 12 месяцев доступное количество приложений выросло в 2 раза.
</p><p>Поддержка архитектуры заметно улучшилась по сравнению с предыдущими годами, когда нехватка ПО была серьёзным недостатком архитектуры.
</p><p>Intel предлагает бесплатный набор библиотек для программирования под Itanium.
</p><p>Компанией HP разработан проприетарный программный симулятор работы процессоров архитектуры IA-64 под названием «Ski».<sup class="reference" id="cite_ref-69"><a href="#cite_note-69">[69]</a></sup> 12 августа 2007 года Ski был выпущен под лицензией GNU GPL v2.
</p>
<ul><li><a href="/wiki/%D0%AF%D0%B4%D1%80%D0%BE_Linux" title="Ядро Linux">Ядро Linux</a> и многие дистрибутивы GNU/Linux работают на Itanium. Например, <a href="/wiki/Debian" title="Debian">Debian</a>, <a href="/wiki/Red_Hat_Enterprise_Linux" title="Red Hat Enterprise Linux">Red Hat Enterprise Linux</a>, <a href="/wiki/SUSE_Linux_Enterprise_Server" title="SUSE Linux Enterprise Server">SUSE Linux Enterprise Server</a> и версии от <a class="mw-redirect" href="/wiki/SGI" title="SGI">SGI</a>, предлагаемые с её серверами <a href="/wiki/Altix" title="Altix">Altix</a>. В январе 2002 года было создано сообщество <a class="external text" href="https://web.archive.org/web/20070530053130/http://www.gelato.org/" rel="nofollow">Gelato.org</a> для поддержки и продвижения Linux на Itanium. Intel и другие члены сообщества <a class="new" href="/w/index.php?title=Itanium_Solutions_Alliance&amp;action=edit&amp;redlink=1" title="Itanium Solutions Alliance (страница отсутствует)">Itanium Solutions Alliance</a> работают над оптимизацией компиляторов для приложений под Linux.
<ul><li>В 2010 году <a href="/wiki/Red_Hat" title="Red Hat">Red Hat</a> <a class="external text" href="http://www.pcworld.com/article/185196/red_hat_to_drop_itanium_support_in_enterprise_linux_6.html" rel="nofollow">объявила</a>, что прекращает поддержку архитектуры Itanium. <a href="/wiki/Red_Hat_Enterprise_Linux" title="Red Hat Enterprise Linux">Red Hat Enterprise Linux</a> версии 6 вышел уже без поддержки.</li></ul></li>
<li><a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> портировала <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a>, <a href="/wiki/Windows_2000" title="Windows 2000">Windows 2000</a> и <a href="/wiki/Windows_Server_2003" title="Windows Server 2003">Windows Server 2003</a> на Itanium. Серверные приложения от Microsoft включают <a href="/wiki/Microsoft_SQL_Server" title="Microsoft SQL Server">SQL Server</a>, <a class="mw-redirect" href="/wiki/Microsoft_Operations_Manager" title="Microsoft Operations Manager">Operations Manager</a>, <a class="new" href="/w/index.php?title=CRM_Server&amp;action=edit&amp;redlink=1" title="CRM Server (страница отсутствует)">CRM Server</a>, <a class="mw-redirect" href="/wiki/IIS" title="IIS">IIS</a>, <a href="/wiki/Microsoft_Visual_Studio" title="Microsoft Visual Studio">Visual Studio</a>, и <a href="/wiki/.NET_Framework" title=".NET Framework">.NET Framework</a>. В последние годы было принято решение не включать поддержку Itanium в клиентское ПО или клиентские ОС (например, Windows XP) из-за того, что рынок слишком мал для возмещения затрат на портирование и поддержку. Microsoft объявила о поддержке Itanium в грядущей ОС <a class="mw-redirect" href="/wiki/Windows_Longhorn_Server" title="Windows Longhorn Server">Windows Longhorn Server</a> (Windows Server 2008). Он планируется как процессор для таких приложений, как серверы баз данных, <a class="new" href="/w/index.php?title=Operations_management&amp;action=edit&amp;redlink=1" title="Operations management (страница отсутствует)">operations management</a> и <a class="mw-redirect" href="/wiki/CRM" title="CRM">CRM</a><sup class="reference" id="cite_ref-70"><a href="#cite_note-70">[70]</a></sup>. Microsoft активно поддержала технологию Itanium в Longhorn<sup class="reference" id="cite_ref-71"><a href="#cite_note-71">[71]</a></sup><sup>,</sup><sup class="reference" id="cite_ref-72"><a href="#cite_note-72">[72]</a></sup> и своих планах, связанных с 64-битными вычислениями, указав на то, что делит 64-битные вычисления на два направления, ориентировав решения, базирующиеся на архитектуре x64 для большинства применений, и отнеся к базирующимся на архитектуре Itanium решениям задачи сверхбольших баз данных и заказных пользовательских приложений<sup class="reference" id="cite_ref-73"><a href="#cite_note-73">[73]</a></sup><sup>,</sup><sup class="reference" id="cite_ref-74"><a href="#cite_note-74">[74]</a></sup>
<ul><li>В 2010 году Microsoft объявила<sup class="reference" id="cite_ref-75"><a href="#cite_note-75">[75]</a></sup>, что Windows Server 2008 R2 и SQL Server 2008 R2 стали последними версиям данного ПО, поддерживающими архитектуру Itanium.</li></ul></li>
<li>ОС <a href="/wiki/HP-UX" title="HP-UX">HP-UX</a>, коммерческий UNIX от HP, работает на Itanium с момента появления процессора в 2001 году.</li>
<li>Поддержка <a href="/wiki/OpenVMS" title="OpenVMS">OpenVMS</a> добавлена в 2003 году.</li>
<li>ОС <a class="new" href="/w/index.php?title=NonStop&amp;action=edit&amp;redlink=1" title="NonStop (страница отсутствует)">NonStop</a> от HP также работает на Itanium.</li>
<li><a href="/wiki/Compaq" title="Compaq">Compaq</a> портировала <a href="/wiki/Tru64_UNIX" title="Tru64 UNIX">Tru64 UNIX</a> на Itanium, но работы над этой ОС были прекращены после слияния с HP.</li>
<li><a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a> работает на Itanium, начиная с версии 5.0.</li>
<li><a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun</a> разрабатывала версию ОС <a href="/wiki/Solaris" title="Solaris">Solaris</a> для Itanium, но работа над ней была прекращена в 2000 году<sup class="reference" id="cite_ref-sun_solaris_76-0"><a href="#cite_note-sun_solaris-76">[76]</a></sup>. Тем не менее, компания <a class="new" href="/w/index.php?title=Transitive_Corp&amp;action=edit&amp;redlink=1" title="Transitive Corp (страница отсутствует)">Transitive</a> представила разработанный ею программный двоичный транслятор, который позволяет запускать приложения, написанные под Solaris, на серверах с Itanium под управлением Linux без каких-либо изменений исходного или объектного кода. При этом, как утверждает компания, производительность приложений, написанных под Solaris, на Itanium-системах выше, чем на самых быстрых SPARC системах.</li>
<li><a href="/wiki/IBM" title="IBM">IBM</a> и <a href="/wiki/SCO" title="SCO">SCO</a> работали над портированием <a href="/wiki/AIX" title="AIX">AIX</a> на Itanium в рамках <a class="new" href="/w/index.php?title=%D0%9C%D0%BE%D0%BD%D1%82%D0%B5%D1%80%D0%B5%D0%B9_(%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82)&amp;action=edit&amp;redlink=1" title="Монтерей (проект) (страница отсутствует)">проекта Монтерей</a>, но он так никогда официально и не был представлен. Тем не менее, IBM портировала <a href="/wiki/DB2" title="DB2">DB2</a> и многое другое ПО на Itanium для работы под управлением <a href="/wiki/HP-UX" title="HP-UX">HP-UX</a>, и, судя по заявлениям HP, работы по портированию продолжаются.</li>
<li>8 апреля 2003 года на совместной пресс-конференции компании Oracle, Hewlett-Packard, Intel и <a class="new" href="/w/index.php?title=I-Teco&amp;action=edit&amp;redlink=1" title="I-Teco (страница отсутствует)">i-Teco</a> объявили о выходе первой коммерческой версии СУБД Oracle9i, оптимизированной для работы на платформе Itanium 2. Если до этого основными направлениями применения Itanium 2 были ресурсоёмкие технические и научные расчёты, <a href="/wiki/%D0%A1%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0_%D0%B0%D0%B2%D1%82%D0%BE%D0%BC%D0%B0%D1%82%D0%B8%D0%B7%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D0%BE%D0%B3%D0%BE_%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F" title="Система автоматизированного проектирования">системы автоматизированного проектирования</a>, обработка графической информации, то теперь эту серверную платформу можно использовать со множеством приложений баз данных Oracle9i, а если учесть, что вскоре выйдет коммерческая версия MS SQL Server и для Itanium 2, то речь идёт о выходе на рынок корпоративных систем.</li>
<li>Запущена Oracle9i Database (версий Enterprise Edition и Standard Edition) и Oracle9i Real Application Clusters (ПО для кластеров баз данных) для моделей серверов rx2600 и rx5670 Hewlett-Packard, в распоряжении которой после <a href="/wiki/%D0%A1%D0%BB%D0%B8%D1%8F%D0%BD%D0%B8%D1%8F_%D0%B8_%D0%BF%D0%BE%D0%B3%D0%BB%D0%BE%D1%89%D0%B5%D0%BD%D0%B8%D1%8F" title="Слияния и поглощения">рыночных слияний</a> оказались и 64-разрядные серверы <a href="/wiki/DEC_Alpha" title="DEC Alpha">Alpha</a>, под ОС HP-UX. Кроме того, для 64-разрядной операционной системы Microsoft Windows на серверах с Itanium 2, корпорация Oracle выпустила сервер приложений Oracle9i Application Server Release 2 (Developers Release, версия для разработчиков).</li>
<li>В 2003 году закончено тестирование предварительной версии СУБД Oracle10i на серверах HP Server rx5670 оснащённых четырьмя процессорами Itanium 2 и операционными системами Red Hat Linux Advanced Server IA64 и HP-UX 11i, v.1.6, Performance Pack 64-bit.</li>
<li>В 2011 году <a href="/wiki/Oracle" title="Oracle">Oracle</a> объявила<sup class="reference" id="cite_ref-77"><a href="#cite_note-77">[77]</a></sup>, что прекращает поддержку архитектуры Itanium.</li></ul>
<p>Сразу после выхода Itanium началось портирование некоторых приложений для работы с трёхмерной анимацией (таких, как <a href="/wiki/Autodesk_Maya" title="Autodesk Maya">Maya</a>), но выпуск 64-битных систем с набором инструкций x86 оказался более привлекательным для независимых разработчиков ПО благодаря обратной совместимости с уже существовавшими x86 версиями приложений. Вследствие этого большинство начинаний по портированию на IA-64 были прекращены. Областью, в которой Itanium всё-таки доказал свою привлекательность, стали приложения для трёхмерного рендеринга. Одним из таких приложений является <a class="new" href="/w/index.php?title=VG_Studio_Max&amp;action=edit&amp;redlink=1" title="VG Studio Max (страница отсутствует)">VG Studio Max</a> от <a class="new" href="/w/index.php?title=Volume_Graphics&amp;action=edit&amp;redlink=1" title="Volume Graphics (страница отсутствует)">Volume Graphics</a>.
</p><p>Начало поставок Itanium 2 вызвало широкий отклик в отрасли. О готовности своих приложений для работы на серверах с Itanium 2 заявило около ста софтверных фирм, более 20 OEM-производителей предложили системы<sup class="reference" id="cite_ref-78"><a href="#cite_note-78">[78]</a></sup>.
</p>
