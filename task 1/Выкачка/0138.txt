<div id="contentSub"><div id="mw-content-subtitle"><div class="plainlinks flaggedrevs_preview" id="mw-fr-reviewnotice">Текущая версия страницы пока <a href="/wiki/%D0%92%D0%B8%D0%BA%D0%B8%D0%BF%D0%B5%D0%B4%D0%B8%D1%8F:%D0%9F%D1%80%D0%BE%D0%B2%D0%B5%D1%80%D0%BA%D0%B0_%D1%81%D1%82%D0%B0%D1%82%D0%B5%D0%B9/%D0%9F%D0%BE%D1%8F%D1%81%D0%BD%D0%B5%D0%BD%D0%B8%D0%B5_%D0%B4%D0%BB%D1%8F_%D1%87%D0%B8%D1%82%D0%B0%D1%82%D0%B5%D0%BB%D0%B5%D0%B9" title="Википедия:Проверка статей/Пояснение для читателей">не проверялась</a> опытными участниками и может значительно отличаться от <a class="external text" href="https://ru.wikipedia.org/w/index.php?title=BitTorrent&amp;stable=1">версии</a>, проверенной 8 декабря 2022 года; проверки требуют <a class="external text" href="https://ru.wikipedia.org/w/index.php?title=BitTorrent&amp;oldid=126726906&amp;diff=cur&amp;diffonly=0">19 правок</a>.</div><div class="flaggedrevs_short flaggedrevs_draft_notsynced plainlinks noprint" id="mw-fr-revisiontag"><div class="flaggedrevs_short_basic"><span class="flaggedrevs-icon oo-ui-widget oo-ui-widget-enabled oo-ui-iconElement-icon oo-ui-icon-block oo-ui-iconElement oo-ui-labelElement-invisible oo-ui-iconWidget" title="Текущая версия"></span><span class="fr-toggle-arrow oo-ui-widget oo-ui-widget-enabled oo-ui-indicatorElement-indicator oo-ui-indicator-down oo-ui-indicatorElement oo-ui-labelElement-invisible oo-ui-indicatorWidget" id="mw-fr-revisiontoggle" title="показать/скрыть подробности"></span></div>
<div id="mw-fr-revisiondetails-wrapper" style="position:relative;"><div class="flaggedrevs_short_details" id="mw-fr-revisiondetails" style="display:none">Текущая версия страницы пока <a href="/wiki/%D0%92%D0%B8%D0%BA%D0%B8%D0%BF%D0%B5%D0%B4%D0%B8%D1%8F:%D0%9F%D1%80%D0%BE%D0%B2%D0%B5%D1%80%D0%BA%D0%B0_%D1%81%D1%82%D0%B0%D1%82%D0%B5%D0%B9/%D0%9F%D0%BE%D1%8F%D1%81%D0%BD%D0%B5%D0%BD%D0%B8%D0%B5_%D0%B4%D0%BB%D1%8F_%D1%87%D0%B8%D1%82%D0%B0%D1%82%D0%B5%D0%BB%D0%B5%D0%B9" title="Википедия:Проверка статей/Пояснение для читателей">не проверялась</a> опытными участниками и может значительно отличаться от <a class="external text" href="https://ru.wikipedia.org/w/index.php?title=BitTorrent&amp;stable=1">версии</a>, проверенной 8 декабря 2022 года; проверки требуют <a class="external text" href="https://ru.wikipedia.org/w/index.php?title=BitTorrent&amp;oldid=126726906&amp;diff=cur&amp;diffonly=0">19 правок</a>.</div>

<div class="mw-body-content" id="mw-content-text"><div class="mw-content-ltr mw-parser-output" dir="ltr" lang="ru"><div class="hatnote navigation-not-searchable noprint dabhide">У этого термина существуют и другие значения, см. <a class="mw-disambig" href="/wiki/BitTorrent_(%D0%B7%D0%BD%D0%B0%D1%87%D0%B5%D0%BD%D0%B8%D1%8F)" title="BitTorrent (значения)">BitTorrent (значения)</a>.</div>
<p><b>BitTorrent</b> (букв. <a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;"></span>«<a href="/wiki/%D0%91%D0%B8%D1%82%D0%BE%D0%B2%D1%8B%D0%B9_%D0%BF%D0%BE%D1%82%D0%BE%D0%BA" title="Битовый поток">битовый поток</a>») — <a href="/wiki/%D0%9E%D0%B4%D0%BD%D0%BE%D1%80%D0%B0%D0%BD%D0%B3%D0%BE%D0%B2%D0%B0%D1%8F_%D1%81%D0%B5%D1%82%D1%8C" title="Одноранговая сеть">пиринговый</a> (P2P) <a href="/wiki/%D0%9F%D1%80%D0%BE%D1%82%D0%BE%D0%BA%D0%BE%D0%BB_%D0%BF%D0%B5%D1%80%D0%B5%D0%B4%D0%B0%D1%87%D0%B8_%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85" title="Протокол передачи данных">сетевой протокол</a> для кооперативного обмена <a href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB" title="Файл">файлами</a> через <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D1%80%D0%BD%D0%B5%D1%82" title="Интернет">Интернет</a>.
</p><p>Файлы передаются частями, каждый <a href="/wiki/.torrent" title=".torrent">torrent</a>-<a href="/wiki/%D0%9A%D0%BB%D0%B8%D0%B5%D0%BD%D1%82_%E2%80%94_%D1%81%D0%B5%D1%80%D0%B2%D0%B5%D1%80" title="Клиент — сервер">клиент</a>, получая (скачивая) эти части, в то же время отдаёт (закачивает) их другим клиентам, что снижает нагрузку и зависимость от каждого клиента-источника и обеспечивает <a href="/wiki/%D0%98%D0%B7%D0%B1%D1%8B%D1%82%D0%BE%D1%87%D0%BD%D0%BE%D1%81%D1%82%D1%8C_%D0%B8%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%86%D0%B8%D0%B8" title="Избыточность информации">избыточность данных</a>.
</p><p>Протокол был создан <a href="/wiki/%D0%9A%D0%BE%D1%8D%D0%BD,_%D0%91%D1%80%D1%8D%D0%BC" title="Коэн, Брэм">Брэмом Коэном</a>, написавшим первый torrent-клиент <a href="/wiki/BitTorrent_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B0)" title="BitTorrent (программа)">BitTorrent</a> на языке <a href="/wiki/Python" title="Python">Python</a>, <a href="/wiki/4_%D0%B0%D0%BF%D1%80%D0%B5%D0%BB%D1%8F" title="4 апреля">4 апреля</a> <a href="/wiki/2001_%D0%B3%D0%BE%D0%B4" title="2001 год">2001 года</a>. Запуск первой версии состоялся <a href="/wiki/2_%D0%B8%D1%8E%D0%BB%D1%8F" title="2 июля">2 июля</a> <a href="/wiki/2001_%D0%B3%D0%BE%D0%B4" title="2001 год">2001 года</a>.
</p><p>Существует множество программ-клиентов для обмена файлами по протоколу BitTorrent.
</p>

<h2><span id=".D0.A4.D0.B0.D0.B9.D0.BB_.D0.BC.D0.B5.D1.82.D0.B0.D0.B4.D0.B0.D0.BD.D0.BD.D1.8B.D1.85"></span><span class="mw-headline" id="Файл_метаданных">Файл метаданных</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=1" title="Редактировать раздел «Файл метаданных»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=1" title="Редактировать код раздела «Файл метаданных»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote navigation-not-searchable">Основная статья: <b><a href="/wiki/.torrent" title=".torrent">.torrent</a></b></div>
<p>Файл <a href="/wiki/%D0%9C%D0%B5%D1%82%D0%B0%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D0%B5" title="Метаданные">метаданных</a> является словарём в <a href="/wiki/Bencode" title="Bencode">bencode</a> формате с расширением .torrent — содержит информацию о раздаче (файлах, <a href="/wiki/BitTorrent-%D1%82%D1%80%D0%B5%D0%BA%D0%B5%D1%80" title="BitTorrent-трекер">трекерах</a> и др.)
</p>
<h2><span id=".D0.9F.D1.80.D0.B8.D0.BD.D1.86.D0.B8.D0.BF_.D1.80.D0.B0.D0.B1.D0.BE.D1.82.D1.8B_.D0.BF.D1.80.D0.BE.D1.82.D0.BE.D0.BA.D0.BE.D0.BB.D0.B0"></span><span class="mw-headline" id="Принцип_работы_протокола">Принцип работы протокола</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=2" title="Редактировать раздел «Принцип работы протокола»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=2" title="Редактировать код раздела «Принцип работы протокола»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<figure typeof="mw:File/Thumb"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:Torrentcomp_small.gif"><img class="mw-file-element" data-file-height="334" data-file-width="357" decoding="async" height="334" src="//upload.wikimedia.org/wikipedia/commons/3/3d/Torrentcomp_small.gif" width="357"/></a><figcaption>Принцип работы BitTorrent: нагрузка на распространителя файла уменьшается благодаря тому, что клиенты начинают обмениваться данными сразу же, даже если файл не докачан ими до конца</figcaption></figure>
<p>Перед началом скачивания клиент подсоединяется к <a href="/wiki/BitTorrent-%D1%82%D1%80%D0%B5%D0%BA%D0%B5%D1%80" title="BitTorrent-трекер">трекеру</a> по адресу, указанному в торрент-файле, сообщает ему свой адрес и хеш-сумму торрент-файла, на что в ответ клиент получает адреса других клиентов, скачивающих или раздающих этот же файл. Далее клиент периодически информирует трекер о ходе процесса и получает обновлённый список адресов. Этот процесс называется объявлением (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">announce</span>).
</p><p>Клиенты соединяются друг с другом и обмениваются сегментами файлов без непосредственного участия трекера, который лишь хранит информацию, полученную от подключённых к обмену клиентов, список самих клиентов и другую статистическую информацию. Для эффективной работы сети BitTorrent необходимо, чтобы как можно больше клиентов были способны принимать входящие соединения. Неправильная настройка <a href="/wiki/NAT" title="NAT">NAT</a> или <a href="/wiki/%D0%9C%D0%B5%D0%B6%D1%81%D0%B5%D1%82%D0%B5%D0%B2%D0%BE%D0%B9_%D1%8D%D0%BA%D1%80%D0%B0%D0%BD" title="Межсетевой экран">брандмауэра</a> могут этому помешать.
</p><p>При соединении клиенты сразу обмениваются информацией об имеющихся у них сегментах. Клиент, желающий скачать сегмент (<a class="mw-redirect" href="/wiki/%D0%9B%D0%B8%D1%87%D0%B5%D1%80" title="Личер">личер</a>), посылает запрос и, если второй клиент готов отдавать, получает этот сегмент. После этого клиент проверяет контрольную сумму сегмента. Если она совпала с той, что записана в торрент-файле, то сегмент считается успешно скачанным, и клиент оповещает всех присоединённых пиров о наличии у него этого сегмента. Если же контрольные суммы различаются, то сегмент начинает скачиваться заново. Некоторые клиенты <a href="/wiki/%D0%91%D0%B0%D0%BD_(%D0%98%D0%BD%D1%82%D0%B5%D1%80%D0%BD%D0%B5%D1%82)" title="Бан (Интернет)">банят</a> тех пиров, которые слишком часто отдают некорректные сегменты.
</p><p>Таким образом, объём служебной информации (размер торрент-файла и размер сообщений со списком сегментов) напрямую зависит от количества, а значит, и размера сегментов. Поэтому при выборе сегмента необходимо соблюдать баланс: с одной стороны, при большом размере сегмента объём служебной информации будет меньше, но в случае ошибки проверки контрольной суммы придётся заново скачивать больше информации. С другой стороны, при малом размере ошибки не так критичны, так как необходимо заново скачать меньший объём, но зато размер торрент-файла и сообщений об имеющихся сегментах становится больше.
</p>
<h3><span id=".D0.90.D0.BB.D0.B3.D0.BE.D1.80.D0.B8.D1.82.D0.BC_.D0.BE.D0.B1.D0.BC.D0.B5.D0.BD.D0.B0_.D0.B4.D0.B0.D0.BD.D0.BD.D1.8B.D0.BC.D0.B8"></span><span class="mw-headline" id="Алгоритм_обмена_данными">Алгоритм обмена данными</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=3" title="Редактировать раздел «Алгоритм обмена данными»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=3" title="Редактировать код раздела «Алгоритм обмена данными»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Каждый клиент имеет возможность временно блокировать отдачу другому клиенту (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">choke</span>). Это делается для более эффективного использования канала отдачи. Кроме того, при выборе — кого разблокировать, предпочтение отдаётся пирам, которые сами передали этому клиенту много сегментов. Таким образом, пиры с хорошими скоростями отдачи поощряют друг друга по принципу «ты — мне, я — тебе».
</p><p>Обмен сегментами ведётся по принципу «ты — мне, я — тебе» симметрично в двух направлениях. Клиенты сообщают друг другу об имеющихся у них сегментах при подключении и затем — при получении новых сегментов, и поэтому каждый клиент может хранить информацию о том, какие сегменты есть у других подключённых пиров. Порядок обмена выбирается таким образом, чтобы сначала клиенты обменивались наиболее редкими сегментами: таким образом повышается доступность файлов в раздаче. В то же время выбор сегмента среди наиболее редких — случаен, и поэтому можно избежать ситуации, когда все клиенты начинают скачивать один и тот же самый редкий сегмент, что негативно бы отразилось на производительности.
</p><p>Обмен данными начинается, когда в нём заинтересованы обе стороны, то есть, каждая из сторон имеет сегменты, которых нет у другой. Количество переданных сегментов подсчитывается, и если одна из сторон обнаруживает, что передаёт в среднем больше, чем принимает, она блокирует (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">choke</span>) на некоторое время отдачу другой стороне. Таким образом в протокол заложена защита от <a class="mw-redirect" href="/wiki/%D0%9B%D0%B8%D1%87%D0%B5%D1%80" title="Личер">личеров</a>.
</p><p>Сегменты делятся на блоки, и каждый клиент запрашивает именно эти блоки. Одновременно могут запрашиваться блоки из разных сегментов. Более того, некоторые клиенты поддерживают скачивание блоков одного сегмента у разных пиров. В этом случае описанные выше алгоритмы и механизмы обмена применимы и к уровню блоков.
</p>
<h3><span id=".D0.A0.D0.B5.D0.B6.D0.B8.D0.BC_End_game"></span><span class="mw-headline" id="Режим_End_game">Режим End game</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=4" title="Редактировать раздел «Режим End game»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=4" title="Редактировать код раздела «Режим End game»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Когда скачивание почти завершено, клиент входит в особый режим, называемый end game. В этом режиме он запрашивает все оставшиеся сегменты у всех подключённых пиров, что позволяет избежать замедления или полного «зависания» почти завершённой закачки из-за нескольких медленных клиентов.
</p><p>Спецификация протокола не определяет, когда именно клиент должен войти в режим «end game», однако существует набор общепринятых практик. Некоторые клиенты входят в этот режим, когда не осталось незапрошенных блоков, другие — пока количество оставшихся блоков меньше количества передающихся и не больше 20. Существует негласное мнение, что лучше поддерживать количество ожидаемых блоков низким (1 или 2) для минимизации избыточности, и что при случайном запрашивании меньший шанс получить дубликаты одного и того же блока<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>.
</p>
<h3><span id=".D0.A1.D0.B8.D0.B4.D0.B8.D1.80.D0.BE.D0.B2.D0.B0.D0.BD.D0.B8.D0.B5"></span><span class="mw-headline" id="Сидирование">Сидирование</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=5" title="Редактировать раздел «Сидирование»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=5" title="Редактировать код раздела «Сидирование»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>При получении полного файла клиент переходит в специальный режим работы, в котором он только отдаёт данные (становится сидом). Далее сид периодически информирует трекер об изменениях в состоянии торрентов (закачек) и обновляет списки IP-адресов.
</p>
<h3><span id=".D0.9E.D0.B1.D1.89.D0.B8.D0.B5_.D0.BE.D1.81.D0.BE.D0.B1.D0.B5.D0.BD.D0.BD.D0.BE.D1.81.D1.82.D0.B8"></span><span class="mw-headline" id="Общие_особенности">Общие особенности</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=6" title="Редактировать раздел «Общие особенности»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=6" title="Редактировать код раздела «Общие особенности»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<ul><li>Отсутствие очередей на скачивание.</li>
<li><a href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB" title="Файл">Файлы</a> закачиваются небольшими <a href="/wiki/%D0%A4%D1%80%D0%B0%D0%B3%D0%BC%D0%B5%D0%BD%D1%82" title="Фрагмент">фрагментами</a>; чем менее доступен фрагмент, тем чаще он будет передаваться. Таким образом, присутствие в сети «<a href="#Терминология">сидера</a>» с полным файлом для загрузки необязательно — система распределяет сегменты между «<a href="#Терминология">пирами</a>», чтобы в последующем они могли обмениваться недостающими сегментами.</li>
<li><a class="mw-redirect" href="/wiki/%D0%9A%D0%BB%D0%B8%D0%B5%D0%BD%D1%82_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D1%8B%D0%B9)" title="Клиент (программный)">Клиенты</a> (пиры) обмениваются сегментами непосредственно между собой, по принципу «ты — мне, я — тебе».</li>
<li>Скачанные фрагменты становятся немедленно доступны другим клиентам.</li>
<li>Контролируется <a href="/wiki/%D0%A6%D0%B5%D0%BB%D0%BE%D1%81%D1%82%D0%BD%D0%BE%D1%81%D1%82%D1%8C_%D0%B8%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%86%D0%B8%D0%B8" title="Целостность информации">целостность</a> каждого фрагмента.</li>
<li>На фрагменты разбиваются не отдельные файлы, а вся раздача целиком, поэтому у «<a href="#Терминология">личера</a>», пожелавшего скачать лишь некоторые файлы из раздачи, для поддержания целостности фрагментов нередко будет храниться также небольшой объём избыточной (для него) информации.</li>
<li>В качестве объекта раздачи могут выступать несколько файлов (например, содержимое <a href="/wiki/%D0%9A%D0%B0%D1%82%D0%B0%D0%BB%D0%BE%D0%B3_(%D1%84%D0%B0%D0%B9%D0%BB%D0%BE%D0%B2%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0)" title="Каталог (файловая система)">каталога</a>).</li></ul>
<h2><span id=".D0.9F.D1.80.D0.BE.D1.82.D0.BE.D0.BA.D0.BE.D0.BB.D1.8B_.D0.B8_.D0.BF.D0.BE.D1.80.D1.82.D1.8B"></span><span class="mw-headline" id="Протоколы_и_порты">Протоколы и порты</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=7" title="Редактировать раздел «Протоколы и порты»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=7" title="Редактировать код раздела «Протоколы и порты»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Клиенты соединяются с трекером по протоколу <a href="/wiki/TCP" title="TCP">TCP</a>. Наиболее часто используемый входящий <a class="mw-redirect" href="/wiki/%D0%9F%D0%BE%D1%80%D1%82_(TCP/IP)" title="Порт (TCP/IP)">порт</a> трекера: 6969. Наиболее часто используемый диапазон входящих портов клиентов: 6881—6889.
</p><p>Номера портов не фиксированы в спецификации протокола и могут изменяться при необходимости. В данный момент большинство трекеров использует <a href="/wiki/HTTP" title="HTTP">HTTP</a>-порт 80, а для клиентов рекомендуется выбрать случайный входящий порт. Более того, некоторые трекеры не допускают использования портов клиентов из стандартного диапазона 6881—6889, так как некоторые провайдеры запрещают использование этого диапазона портов.
</p><p><a class="mw-redirect" href="/wiki/DHT" title="DHT">DHT</a>-сеть в BitTorrent-клиентах использует протокол <a href="/wiki/UDP" title="UDP">UDP</a>.
</p><p>Кроме того, протокол <a href="/wiki/UDP" title="UDP">UDP</a> используется <span data-interwiki-article="UDP tracker" data-interwiki-lang="en"><a class="new" href="/w/index.php?title=UDP-%D1%82%D1%80%D0%B5%D0%BA%D0%B5%D1%80&amp;action=edit&amp;redlink=1" title="UDP-трекер (страница отсутствует)">UDP-трекерами</a></span><span class="noprint" style="font-style:normal; font-weight:normal;"><sup><a class="extiw" href="https://en.wikipedia.org/wiki/UDP_tracker" title="en:UDP tracker"><span title="UDP tracker — версия статьи «UDP-трекер» на английском языке">[en]</span></a></sup></span> (поддерживается не всеми клиентами и не является официальной частью протокола) и для соединения клиентов друг с другом через UDP <a class="mw-redirect" href="/wiki/NAT-T" title="NAT-T">NAT Traversal</a> (используется только в клиенте BitComet и не является официальной частью протокола).
</p>
<h2><span id=".D0.A2.D1.80.D0.B5.D0.BA.D0.B5.D1.80"></span><span class="mw-headline" id="Трекер">Трекер</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=8" title="Редактировать раздел «Трекер»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=8" title="Редактировать код раздела «Трекер»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote navigation-not-searchable">Основная статья: <b><a href="/wiki/BitTorrent-%D1%82%D1%80%D0%B5%D0%BA%D0%B5%D1%80" title="BitTorrent-трекер">BitTorrent-трекер</a></b></div>
<p><b>Трекер</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">tracker</span>; <span class="IPA"><a href="/wiki/%D0%9C%D0%B5%D0%B6%D0%B4%D1%83%D0%BD%D0%B0%D1%80%D0%BE%D0%B4%D0%BD%D1%8B%D0%B9_%D1%84%D0%BE%D0%BD%D0%B5%D1%82%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%B0%D0%BB%D1%84%D0%B0%D0%B2%D0%B8%D1%82" title="Международный фонетический алфавит">/ˈtɹækə(ɹ)/</a></span>) — специализированный <a href="/wiki/%D0%A1%D0%B5%D1%80%D0%B2%D0%B5%D1%80_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5)" title="Сервер (программное обеспечение)">сервер</a>, работающий по протоколу <a href="/wiki/HTTP" title="HTTP">HTTP</a>. Трекер нужен для того, чтобы клиенты могли найти друг друга. Фактически, на трекере хранятся <a href="/wiki/IP-%D0%B0%D0%B4%D1%80%D0%B5%D1%81" title="IP-адрес">IP-адреса</a>, входящие порты клиентов и <a class="mw-redirect" href="/wiki/%D0%A5%D0%B5%D1%88%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Хеширование">хеш-суммы</a>, уникальным образом идентифицирующие объекты, участвующие в закачках. По стандарту, имена файлов на трекере не хранятся, и узнать их по хеш-суммам нельзя. Однако на практике трекер часто помимо своей основной функции выполняет и функцию небольшого <a href="/wiki/%D0%92%D0%B5%D0%B1-%D1%81%D0%B5%D1%80%D0%B2%D0%B5%D1%80" title="Веб-сервер">веб-сервера</a>. Такой сервер хранит файлы метаданных и описания распространяемых файлов, предоставляет статистику закачек по разным файлам, показывает текущее количество подключённых пиров и пр.
</p>
<h3><span id=".D0.A0.D0.B0.D0.B1.D0.BE.D1.82.D0.B0_.D0.B1.D0.B5.D0.B7_.D1.82.D1.80.D0.B5.D0.BA.D0.B5.D1.80.D0.B0"></span><span class="mw-headline" id="Работа_без_трекера">Работа без трекера</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=9" title="Редактировать раздел «Работа без трекера»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=9" title="Редактировать код раздела «Работа без трекера»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>В новых версиях протокола были разработаны бестрекерные (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">trackerless</span>) системы, которые решают некоторые из предыдущих проблем. Отказ трекера в таких системах не приводит к автоматическому отказу всей сети.
</p><p>Начиная с версии 4.2.0 официального клиента, выпущенной в конце 2015 года, реализована функция бестрекерной работы, базирующаяся на DHT <a href="/wiki/Kademlia" title="Kademlia">Kademlia</a>. В такой реализации <a href="/wiki/BitTorrent-%D1%82%D1%80%D0%B5%D0%BA%D0%B5%D1%80" title="BitTorrent-трекер">трекер</a> доступен децентрализовано на клиентах в форме <a href="/wiki/%D0%A0%D0%B0%D1%81%D0%BF%D1%80%D0%B5%D0%B4%D0%B5%D0%BB%D1%91%D0%BD%D0%BD%D0%B0%D1%8F_%D1%85%D0%B5%D1%88-%D1%82%D0%B0%D0%B1%D0%BB%D0%B8%D1%86%D0%B0" title="Распределённая хеш-таблица">распределённой хеш-таблицы</a>.
</p><p>На данный момент не все клиенты используют совместимый друг с другом протокол. Совместимы между собой <a href="/wiki/BitComet" title="BitComet">BitComet</a>, <a href="/wiki/%CE%9CTorrent" title="ΜTorrent">µTorrent</a>, <a href="/wiki/Deluge" title="Deluge">Deluge</a>, <a href="/wiki/KTorrent" title="KTorrent">KTorrent</a>, <a href="/wiki/Transmission" title="Transmission">Transmission</a>, <a href="/wiki/QBittorrent" title="QBittorrent">qBittorrent</a> и <a href="/wiki/BitTorrent_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B0)" title="BitTorrent (программа)">официальный клиент BitTorrent</a>. <a href="/wiki/Vuze" title="Vuze">Vuze</a> (Azureus) также имеет режим бестрекерной работы, но его реализация отличается от официальной, вследствие чего он не может работать через DHT с вышеперечисленными клиентами<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup>. Однако, для Vuze существует поддержка стандартного DHT через плагин Mainline DHT.
</p><p>Работа без трекера также возможна при использовании мультипротокольных клиентов, поддерживающих BitTorrent. <a href="/wiki/Shareaza" title="Shareaza">Shareaza</a> через сеть <a href="/wiki/Gnutella2" title="Gnutella2">Gnutella2</a> обменивается хешами и адресами пиров других поддерживаемых сетей, в том числе BitTorrent. В <a class="new" href="/w/index.php?title=GreyLink&amp;action=edit&amp;redlink=1" title="GreyLink (страница отсутствует)">GreyLink</a> 6.0 планируется поддержка BitTorrent, при этом сеть <a href="/wiki/Direct_Connect" title="Direct Connect">Direct Connect</a> может использоваться не только для преобразования в <a class="mw-redirect" href="/wiki/TTH" title="TTH">TTH</a>, но и для поиска пиров.
</p>
<h2><span id=".D0.A0.D0.B0.D0.B1.D0.BE.D1.82.D0.B0_.D0.B1.D0.B5.D0.B7_.D1.82.D0.BE.D1.80.D1.80.D0.B5.D0.BD.D1.82-.D0.BA.D0.BB.D0.B8.D0.B5.D0.BD.D1.82.D0.B0"></span><span class="mw-headline" id="Работа_без_торрент-клиента">Работа без торрент-клиента</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=10" title="Редактировать раздел «Работа без торрент-клиента»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=10" title="Редактировать код раздела «Работа без торрент-клиента»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Для того чтобы брать и раздавать файлы в торрент-сетях, не обязательно пользоваться специальными программами. Существует несколько сервисов, которые позволяют скачивать файлы, используя только браузер<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup>.
</p><p>Наличие в файлах метаданных дополнительной информации, такой, как дополнительные источники и опциональные хеши, позволяет использовать файл метаданных .torrent аналогично форматам <a href="/wiki/Metalink" title="Metalink">Metalink</a>, <a href="/wiki/MAGMA" title="MAGMA">MAGMA</a>, <a href="/wiki/%D0%A1%D0%BF%D0%B8%D1%81%D0%BE%D0%BA_%D1%84%D0%B0%D0%B9%D0%BB%D0%BE%D0%B2_(Direct_Connect)" title="Список файлов (Direct Connect)">Список файлов (Direct Connect)</a>. Клиент <a href="/wiki/Shareaza" title="Shareaza">Shareaza</a> использует опциональные хеши для поиска альтернативных источников в других сетях.
</p>
<h2><span id="Web-.D1.81.D0.B8.D0.B4.D1.8B"></span><span class="mw-headline" id="Web-сиды">Web-сиды</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=11" title="Редактировать раздел «Web-сиды»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=11" title="Редактировать код раздела «Web-сиды»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Одним из вариантов использования является так называемое web-сидирование. Иногда на сервере по разным причинам нельзя запустить полноценный торрент-клиент. В этом случае в качестве источника раздачи выступает сервер, работающий по протоколу HTTP. Как правило, клиенты отдают предпочтение другим BitTorrent клиентам и обращаются к web-сиду только по необходимости. Следует знать, что реализован этот вариант использования как минимум тремя способами: <a class="external text" href="http://www.bittorrent.org/beps/bep_0017.html" rel="nofollow">BEP0017 BitTornado style webseeding</a>, <a class="external text" href="http://www.bittorrent.org/beps/bep_0019.html" rel="nofollow">BEP0019 GetRight style webseeding</a> и <a class="external text" href="http://wiki.depthstrike.com/index.php/P2P:Protocol:Specifications:BitTorrent:External_Sourcing" rel="nofollow">External Sourcing</a>, каждый из которых отличается в деталях реализации.
</p><p>Впервые был создан Джоном «TheSHAD0W» Хоффманом, который создал BitTornado<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup>. Начиная с версии 5.0 клиент BitTorrent поддерживает веб-сиды и загрузки с веб-сайтов, был создан простой инструмент, который создаёт публикации веб-сидов торрентов. В μTorrent добавлена поддержка для получения веб-сидов в версии 1.7. В BitComet добавлена поддержка для получения веб-сидов в версии 1.14.
</p>
<h2><span id="BTIH_.28BitTorrent_Info_Hash.29"></span><span class="mw-headline" id="BTIH_(BitTorrent_Info_Hash)">BTIH (BitTorrent Info Hash)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=12" title="Редактировать раздел «BTIH (BitTorrent Info Hash)»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=12" title="Редактировать код раздела «BTIH (BitTorrent Info Hash)»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Это <a href="/wiki/SHA-1" title="SHA-1">SHA-1</a>-<a href="/wiki/%D0%A5%D0%B5%D1%88-%D1%81%D1%83%D0%BC%D0%BC%D0%B0" title="Хеш-сумма">хеш</a> поля Info из <a href="#Файл_метаданных">файла метаданных</a>. Данный <a href="/wiki/%D0%A5%D0%B5%D1%88-%D1%81%D1%83%D0%BC%D0%BC%D0%B0" title="Хеш-сумма">хеш</a> используется в <a href="/wiki/Magnet-%D1%81%D1%81%D1%8B%D0%BB%D0%BA%D0%B0" title="Magnet-ссылка">магнет-ссылках</a>, а также для идентификации на <a href="/wiki/BitTorrent-%D1%82%D1%80%D0%B5%D0%BA%D0%B5%D1%80" title="BitTorrent-трекер">трекере</a> и между клиентами. При загрузке на <a href="/wiki/BitTorrent-%D1%82%D1%80%D0%B5%D0%BA%D0%B5%D1%80" title="BitTorrent-трекер">трекер</a> <a href="#Файл_метаданных">файла метаданных</a> его Info Hash может измениться, так как <a href="/wiki/BitTorrent-%D1%82%D1%80%D0%B5%D0%BA%D0%B5%D1%80" title="BitTorrent-трекер">трекер</a> может изменить поле info, установив флаг закрытой раздачи private или изменив/добавив поля внутри info. Поэтому необходимо снова скачать <a href="#Файл_метаданных">файл метаданных</a> (файл .torrent) с трекера и добавить его в клиент<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup>.
</p>
<h2><span id="BTC-.D1.81.D1.81.D1.8B.D0.BB.D0.BA.D0.B0"></span><span class="mw-headline" id="BTC-ссылка">BTC-ссылка</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=13" title="Редактировать раздел «BTC-ссылка»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=13" title="Редактировать код раздела «BTC-ссылка»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Указывается в виде:
</p><p><code>btc://[Адрес]: [Порт]/[Peer ID]/[ <a class="mw-selflink-fragment" href="#BTIH_(BitTorrent_Info_Hash)">BTIH</a> ]</code>
</p><p>Ссылка такого вида ссылается на раздачу и на её источник. Поддерживается в <a href="/wiki/Shareaza" title="Shareaza">Shareaza</a>.
</p>
<h2><span id=".D0.9D.D0.B5.D0.B4.D0.BE.D1.81.D1.82.D0.B0.D1.82.D0.BA.D0.B8_.D0.B8_.D0.BE.D0.B3.D1.80.D0.B0.D0.BD.D0.B8.D1.87.D0.B5.D0.BD.D0.B8.D1.8F"></span><span class="mw-headline" id="Недостатки_и_ограничения">Недостатки и ограничения</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=14" title="Редактировать раздел «Недостатки и ограничения»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=14" title="Редактировать код раздела «Недостатки и ограничения»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span id=".D0.9D.D0.B5.D0.B4.D0.BE.D1.81.D1.82.D1.83.D0.BF.D0.BD.D0.BE.D1.81.D1.82.D1.8C_.D1.80.D0.B0.D0.B7.D0.B4.D0.B0.D1.87.D0.B8"></span><span class="mw-headline" id="Недоступность_раздачи">Недоступность раздачи</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=15" title="Редактировать раздел «Недоступность раздачи»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=15" title="Редактировать код раздела «Недоступность раздачи»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>В случае, если раздача непопулярна, то может возникнуть ситуация, когда нет ни одного сида, а данных у присутствующих пиров не хватает, чтобы завершить скачивание. В таком случае необходимо ждать появления либо сида, либо пира, имеющего сегменты, отсутствующие у остальных. Можно также использовать копии файлов, полученные другим путём. Раздача, которая не имеет ни одного сида долгое время, называется «мёртвой».
</p><p>Для поощрения раздач даже был создан BitTorrent-токен<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup>.
</p>
<h3><span id=".D0.9E.D1.82.D1.81.D1.83.D1.82.D1.81.D1.82.D0.B2.D0.B8.D0.B5_.D0.B0.D0.BD.D0.BE.D0.BD.D0.B8.D0.BC.D0.BD.D0.BE.D1.81.D1.82.D0.B8_.D0.B8_.D0.BF.D0.B5.D1.80.D1.81.D0.BE.D0.BD.D0.B0.D0.BB.D0.B8.D0.B7.D0.B0.D1.86.D0.B8.D0.B8"></span><span class="mw-headline" id="Отсутствие_анонимности_и_персонализации">Отсутствие анонимности и персонализации</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=16" title="Редактировать раздел «Отсутствие анонимности и персонализации»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=16" title="Редактировать код раздела «Отсутствие анонимности и персонализации»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Принцип работы BitTorrent-протокола подразумевает, что каждому клиенту известны IP-адреса как минимум других клиентов, полученные от сервера. Использование разнообразных расширений протокола в некоторых случаях позволяет узнать также и адреса других пиров из роя. Поэтому:
</p>
<ul><li>Пользователи незащищённых систем и клиентов с известными уязвимостями могут быть подвергнуты атаке.</li>
<li>Возможно узнать адреса пользователей, передающих или принимающих определённый файл.</li></ul>
<p>Проблема с <a href="/wiki/%D0%90%D0%BD%D0%BE%D0%BD%D0%B8%D0%BC" title="Аноним">анонимностью</a> может быть решена при помощи использования <a href="/wiki/Tor" title="Tor">Tor</a><sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup>. BitTorrent-клиент <a href="/wiki/Vuze" title="Vuze">Vuze</a> имеет встроенную программную поддержку этой <a href="/wiki/%D0%90%D0%BD%D0%BE%D0%BD%D0%B8%D0%BC%D0%BD%D1%8B%D0%B5_%D1%81%D0%B5%D1%82%D0%B8" title="Анонимные сети">анонимной сети</a>. Но данный метод не является стопроцентно эффективным<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup>.
</p><p>С другой стороны, протокол не предполагает использование ников. Отсутствует чат между пирами. Невозможно просмотреть список файлов пира (в поисках других файлов, которые могли бы заинтересовать). Большинство этих функций реализовано в других протоколах (например, <a href="/wiki/Direct_Connect" title="Direct Connect">Direct Connect</a>).
</p>
<h3><span id=".D0.9F.D1.80.D0.BE.D0.B1.D0.BB.D0.B5.D0.BC.D0.B0_.D0.BB.D0.B8.D1.87.D0.B5.D1.80.D0.BE.D0.B2"></span><span class="mw-headline" id="Проблема_личеров">Проблема личеров</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=17" title="Редактировать раздел «Проблема личеров»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=17" title="Редактировать код раздела «Проблема личеров»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Некоторые пользователи, особенно на трекерах, не требующих регистрации, не поддерживают раздачу после завершения скачивания, что ведёт к уменьшению общей производительности, поэтому некоторые <a href="/wiki/BitTorrent-%D1%82%D1%80%D0%B5%D0%BA%D0%B5%D1%80" title="BitTorrent-трекер">торрент-трекеры</a> учитывают и количество скачанного/отданного, и выдают разрешение на скачивание в зависимости от размера отданных клиентом данных.
</p>
<h3><span id=".D0.9E.D1.82.D1.81.D1.83.D1.82.D1.81.D1.82.D0.B2.D0.B8.D0.B5_.D1.82.D0.BE.D1.87.D0.BD.D0.BE.D0.B3.D0.BE_.D1.83.D1.87.D1.91.D1.82.D0.B0_.D1.82.D1.80.D0.B0.D1.84.D0.B8.D0.BA.D0.B0"></span><span class="mw-headline" id="Отсутствие_точного_учёта_трафика">Отсутствие точного учёта трафика</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=18" title="Редактировать раздел «Отсутствие точного учёта трафика»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=18" title="Редактировать код раздела «Отсутствие точного учёта трафика»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>В отличие от многих коммерческих протоколов дистрибуции медиаконтента, архитектура протокола не предусматривает точного механизма учёта и контроля трафика между точками сети. Всё, что есть — поля downloaded и uploaded, в которых клиенты передают при анонсе трекеру количество байт, учтённых при скачивании/загрузке данных с момента предыдущего анонса. Однако не контролируясь никем, кроме как клиентом, они могут быть легко подменены. Для этого пользователи статично прописывают значения этих полей в <a href="/wiki/URI" title="URI">URI</a> трекера, пользуются патчами для клиентов или же отдельными программами (RatioMaster, GiveMeTorrent, GreedyTorrent и т. д.), либо просто удаляют из клиента запись о трекере сразу же после получения с трекера списка точек сети. Всё это позволяет обходить искусственные ограничения, создаваемые администрацией многих частных и публичных трекеров.
</p><p><br/>
</p>
<h2><span id=".D0.A2.D0.B5.D1.80.D0.BC.D0.B8.D0.BD.D0.BE.D0.BB.D0.BE.D0.B3.D0.B8.D1.8F"></span><span class="mw-headline" id="Терминология">Терминология</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=19" title="Редактировать раздел «Терминология»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=19" title="Редактировать код раздела «Терминология»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<ul><li><b>Анонс</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">announce</span>) — обращение клиента к <a href="/wiki/BitTorrent-%D1%82%D1%80%D0%B5%D0%BA%D0%B5%D1%80" title="BitTorrent-трекер">трекеру</a> посредством <a href="/wiki/HTTP#GET" title="HTTP">HTTP-GET</a>-запроса. При каждом анонсе клиент передаёт на трекер информацию об объёмах им скачанного и отданного, a трекер передаёт клиенту список адресов других клиентов. Обращение клиента к трекеру происходит через определённые интервалы времени, которые определяются настройками клиента и трекера.</li>
<li><b>Веб-сид</b> — <a href="/wiki/HTTP" title="HTTP">HTTP</a> или <a href="/wiki/FTP" title="FTP">FTP</a>-сервер, используемый в качестве источника данных наравне с обычными сидами</li>
<li><b>Доступность</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">availability</span>, <a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">distributed copies</span> — распространённые копии) — количество полных копий файла, доступных клиенту. Каждый сид добавляет 1,0 к этому числу; личеры увеличивают доступность в зависимости от количества скачанного, которого нет у других личеров. К примеру, если на раздаче есть один сид и два личера, скачавшие по 50 % файла (скачанные части равны между собой), то доступность равна 1,50.</li>
<li><b>Заглохший</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">choked</span> — заглохший, придушенный) — клиент, обмен данными с которым заглох. Либо его канал на выход забит полностью и он не может ничего передать (достиг max_uploads), либо он является <i>сидом</i> и ему ничего не нужно получать.</li>
<li><b>Заинтересованный</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">interested</span>) — участник, желающий получить куски файла, имеющиеся у другого участника. Например, если у клиента А нет каких-то частей, которые есть у клиента Б, считается, что клиент А заинтересован в обмене с клиентом Б.</li>
<li><b>Излишки</b> — данные, которые были посланы пиром или сидом, но получатель в них не нуждается. К излишкам также относятся ошибки хеша.</li>
<li><b>Индекс</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">index</span>) — это список .torrent-файлов (обычно включающий описания и другую информацию), управляемый веб-сайтом (<i>индексатором</i>) и доступный для поиска. Индексирующий сайт часто ошибочно называют трекером.</li></ul>
<figure typeof="mw:File/Thumb"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:Leech_bittorrent.png"><img class="mw-file-element" data-file-height="604" data-file-width="1016" decoding="async" height="119" src="//upload.wikimedia.org/wikipedia/commons/thumb/5/5d/Leech_bittorrent.png/200px-Leech_bittorrent.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/5/5d/Leech_bittorrent.png/300px-Leech_bittorrent.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/5/5d/Leech_bittorrent.png/400px-Leech_bittorrent.png 2x" width="200"/></a><figcaption>Личер и его рой</figcaption></figure>
<ul><li><b>Лич</b>, иногда <b>личер</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">leech</span> — пиявка) — пир, не имеющий пока всех сегментов, то есть продолжающий скачивание. Термин часто употребляется и в негативном смысле, который он имеет в других файлообменных сетях: пользователь, который отдаёт гораздо меньше, чем скачивает.</li>
<li><b><a href="/wiki/%D0%9E%D1%82%D1%80%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D0%B5_%D1%82%D0%BE%D1%80%D1%80%D0%B5%D0%BD%D1%82-%D1%84%D0%B0%D0%B9%D0%BB%D0%B0" title="Отравление торрент-файла">Отравление торрент-файла</a></b> — ситуация, когда часть пиров раздаёт повреждённые либо специально сфальсифицированные сегменты.</li>
<li><b><a class="mw-redirect" href="/wiki/Peer" title="Peer">Пир</a></b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">peer</span> — соучастник) — клиент, участвующий в раздаче в любой файлообменной сети.</li>
<li><b>Пренебрегающий</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">snubbed</span>) — клиент, подключённый к получателю, но не посылавший ему данные уже более 60 секунд.</li>
<li><b>Раздача</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">seeding</span>) — процесс распространения файла по протоколу BitTorrent.</li>
<li><b>Рейтинг</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">share ratio</span>) — отношение отданного к скачанному.</li>
<li><b>Рой</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">swarm</span>) — совокупность всех пиров, участвующих в раздаче.</li>
<li><b>Сегмент</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">part</span> — часть) — все файлы для передачи делятся на небольшие куски — сегменты, которые затем передаются по сети в произвольном порядке для оптимизации обмена.</li>
<li><b>Сид</b>, иногда <i>сидер</i> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">seeder</span> — сеятель) — пир, имеющий все сегменты распространяемого файла, то есть либо начальный распространитель файла, либо уже скачавший весь файл и оставшийся на раздаче.</li>
<li><span id="Scrape"></span><b>Scrape</b>-запрос (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <i>scrape</i> — скрести, царапать) — дополнительный <a class="mw-redirect" href="/wiki/%D0%A1%D0%B5%D1%82%D0%B5%D0%B2%D0%BE%D0%B9_%D0%BF%D1%80%D0%BE%D1%82%D0%BE%D0%BA%D0%BE%D0%BB" title="Сетевой протокол">протокол</a> запроса клиента к <a href="/wiki/BitTorrent-%D1%82%D1%80%D0%B5%D0%BA%D0%B5%D1%80" title="BitTorrent-трекер">трекеру</a>, при котором трекер сообщает клиенту общее количество сидов и пиров на раздаче. В отличие от анонсирования, запрос scrape не имеет прямого отношения к скачиванию раздачи, является необязательным и отнимает меньше ресурсов у клиента и трекера. Он может запрашиваться и для остановленных в клиенте заданий, а также позволяет одним запросом получить информацию сразу по нескольким торрентам (multi-scrape). Клиент с помощью scrape-запроса может получить точное количество сидов и пиров на каждом задании, включая остановленные. Некоторые клиенты, например <a class="mw-redirect" href="/wiki/Azureus" title="Azureus">Azureus</a>, также могут с помощью scrape-запроса раньше узнать о том, что на раздаче появились дополнительные участники, и сделать внеочередной <a class="mw-redirect" href="/wiki/Announce" title="Announce">announce</a> для получения их адресов, и автоматически останавливать и запускать сидирование заданий в зависимости от числа сидов и пиров, в результате сидируя там, где это нужнее.</li>
<li><b><a class="mw-redirect" href="/wiki/Super_seeding" title="Super seeding">Супер-сидирование</a></b> — специальный режим раздачи в некоторых BitTorrent-клиентах, пытающийся минимизировать количество данных, которое отдаст раздающий до появления первого скачавшего. Суперсид предлагает каждому пиру скачать только один сегмент файла, которого ещё нет у других пиров. Затем сид не даёт этому пиру следующих сегментов, пока не получит от других пиров подтверждения, что они тоже получили этот сегмент. Таким образом, суперсид пытается избежать повторной отдачи одних и тех же сегментов, и старается отдавать сегменты только тем пирам, которые активно передают их другим.</li>
<li><b>Хеш</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">hash</span>) — <a class="mw-redirect" href="/wiki/SHA1" title="SHA1">SHA1</a> отдельных сегментов оригинальных файлов, перечисленных в словаре «info» .torrent-файла. Каждая часть после получения сначала проверяется на совпадение хеша. Если проверка не удалась, данные отбрасываются и запрашиваются ещё раз. Также в протоколе используется хеш самого словаря «info»(«инфохеш»), выступающий в роли идентификатора конкретной раздачи при обращении к трекеру, к другим точкам сети и при составлении <a href="/wiki/Magnet-%D1%81%D1%81%D1%8B%D0%BB%D0%BA%D0%B0" title="Magnet-ссылка">magnet-ссылок</a> (он содержат <a class="mw-redirect" href="/wiki/Base32" title="Base32">Base32</a>-представление инфохеша).</li>
<li><b>Passkey</b> — <a href="/wiki/%D0%90%D1%83%D1%82%D0%B5%D0%BD%D1%82%D0%B8%D1%84%D0%B8%D0%BA%D0%B0%D1%86%D0%B8%D1%8F" title="Аутентификация">аутентификатор</a> пользователя на неанонимных трекерах. Содержится в скачиваемом torrent-файле. Таким образом, если кто-то получит доступ к torrent-файлу (например, пользователь по неосторожности <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80%D0%BD%D1%8B%D0%B9_%D1%81%D0%BB%D0%B5%D0%BD%D0%B3#Р" title="Компьютерный сленг">расшарил</a> его), он сможет работать с трекером от имени этого пользователя. Трекер может изменить passkey по запросу пользователя, но при этом необходимо будет перескачать все прошлые torrent-файлы (или вручную отредактировать их), чтобы иметь возможность и дальше раздавать скачанные файлы.</li>
<li><b><a href="/wiki/URL" title="URL">URL</a> анонса</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">announce URL</span>) — адрес трекера, к которому клиент делает анонс. Во многих клиентах называется «Tracker URL». Может включать «passkey» — уникальный код, назначаемый трекером для аккаунта пользователя, помогающий идентифицировать его на трекере (добавляется к URL анонса в самом *.torrent-файле при скачивании).</li></ul>
<h2><span class="mw-headline" id="BitTorrent_v2">BitTorrent v2</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=BitTorrent&amp;veaction=edit&amp;section=20" title="Редактировать раздел «BitTorrent v2»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=BitTorrent&amp;action=edit&amp;section=20" title="Редактировать код раздела «BitTorrent v2»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Работа над BitTorrent протоколом второй версии ведётся с 2008 года. В протоколе осуществлён уход от использования алгоритма SHA-1, имеющего проблемы с подбором коллизий, в пользу SHA2-256. Инфо-хеш второй версии применяется как для контроля целостности блоков данных, так и для записей в индексах (info-dictionary), что нарушает совместимость в коммуникации по сети DHT новых и необновившихся клиентов, и трекеров (длина v2 хеша 32 байта, а v1 — 20 байт). Для магнитных ссылок на торренты с хэшами SHA2-256 предложен новый префикс «urn:btmh:» (для SHA-1 и гибридных торрентов используется «urn:btih:»).
</p><p>Разработка спецификации BitTorrent v2 изначально велась без оглядки на обратную совместимость и в состав были приняты другие существенные улучшения, такие как использование <a class="mw-redirect" href="/wiki/%D0%94%D0%B5%D1%80%D0%B5%D0%B2%D0%BE_%D0%9C%D0%B5%D1%80%D0%BA%D0%BB%D0%B5" title="Дерево Меркле">хеш-дерева Меркла</a> в индексах для сокращения размера torrent-файлов и проверки загруженных данных на уровне блоков.
</p><p>Из изменений в BitTorrent v2 также выделяется переход к привязке отдельных хэш-деревьев к каждому файлу и применение выравнивания файлов по частям (без добавления добавочного заполнения после каждого файла), что позволяет избавиться от дублирования данных при наличии одинаковых файлов и упрощает выявление разных источников для файлов. Тем самым решается потребность скачивания полного размера части, для проверки целостности отдельных файлов, экономя пространство и трафик. Торрент-файл содержит значения коренных хешей файлов, которые отличаются от стандартного SHA2-256 файла, эти хеши могут также быть показаны на трекерах. Это даёт возможность поиска торрентов по значению хешей файлов, что упрощает нахождение торрентов с нужными файлами, восстанавливания умирающих раздач. Они также могут быть извлечены из .torrent файлов, с помощью пользовательских программ<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup>. Повышена эффективность кодирования структуры каталогов (уходом от повторения полного пути для каждого файла), уменьшая размер .torrent файла.
</p><p>Для сглаживания сосуществования BitTorrent v1 и BitTorrent v2 реализована возможность создания гибридных torrent-файлов, которые включают, помимо структур с хэшами SHA-1, также индексы с SHA2-256. Гибридные торренты могут использоваться клиентами, без поддержки BitTorrent v2. Так как данные обоих протоколов вшиты в info-раздел гибридного .torrent файла, то клиенты с дополнительной поддержкой BitTorrent v2, автоматически подцепляют поддержку v2 протокола через DHT пиров (клиенты которых могут даже его не поддерживать), при этом руководствуясь только v1 хешом в магнитной ссылке. Протокол BEP-47 (выравнивание файлов по частям файлами заполнения во мнимой папке .pad) обязателен для гибридных файлов. Для решения несовместимости пользователям на первое время рекомендуется пользоваться гибридами.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup>. Также ведётся разработка для поддержки протокола WebTorrent, работающего в браузерах<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup>.
</p>
