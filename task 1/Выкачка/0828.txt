<div class="vector-body" id="bodyContent">
<div class="noprint" id="siteSub">Материал из Википедии — свободной энциклопедии</div>

<div class="mw-body-content" id="mw-content-text"><div class="mw-content-ltr mw-parser-output" dir="ltr" lang="ru"><p><b>AUTOEXEC.BAT</b> (от <a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">automatic execution</span> — автоматическое исполнение и <a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">batch</span> — пакет, группа) — системный <a href="/wiki/%D0%9F%D0%B0%D0%BA%D0%B5%D1%82%D0%BD%D1%8B%D0%B9_%D1%84%D0%B0%D0%B9%D0%BB" title="Пакетный файл">пакетный файл</a> (файл, содержащий последовательность команд на языке <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D1%80%D0%BF%D1%80%D0%B5%D1%82%D0%B0%D1%82%D0%BE%D1%80_%D0%BA%D0%BE%D0%BC%D0%B0%D0%BD%D0%B4%D0%BD%D0%BE%D0%B9_%D1%81%D1%82%D1%80%D0%BE%D0%BA%D0%B8" title="Интерпретатор командной строки">интерпретатора командной строки</a> — поставляемого в составе MS-DOS <a href="/wiki/COMMAND.COM" title="COMMAND.COM">COMMAND.COM</a> или его клонов вроде <a href="/wiki/4DOS" title="4DOS">4DOS</a>), расположенный в <a href="/wiki/%D0%9A%D0%B0%D1%82%D0%B0%D0%BB%D0%BE%D0%B3_(%D1%84%D0%B0%D0%B9%D0%BB%D0%BE%D0%B2%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0)" title="Каталог (файловая система)">корневом каталоге</a> <span data-interwiki-article="Boot disk" data-interwiki-lang="en"><a class="new" href="/w/index.php?title=%D0%97%D0%B0%D0%B3%D1%80%D1%83%D0%B7%D0%BE%D1%87%D0%BD%D0%BE%D0%B5_%D1%83%D1%81%D1%82%D1%80%D0%BE%D0%B9%D1%81%D1%82%D0%B2%D0%BE&amp;action=edit&amp;redlink=1" title="Загрузочное устройство (страница отсутствует)">загрузочного устройства</a></span> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;"><a class="extiw" href="https://en.wikipedia.org/wiki/Boot_disk" title="en:Boot disk"><span title="Boot disk — версия статьи «Загрузочное устройство» на английском языке">Boot disk</span></a></span>) (<a href="/wiki/%D0%94%D0%B8%D1%81%D0%BA%D0%B5%D1%82%D0%B0" title="Дискета">дискеты</a> или <a href="/wiki/%D0%96%D1%91%D1%81%D1%82%D0%BA%D0%B8%D0%B9_%D0%B4%D0%B8%D1%81%D0%BA" title="Жёсткий диск">диска</a>). Впервые этот файл появился в операционной системе <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>, а его имя является аббревиатурой и описывает его функцию — автоматическое исполнение команд при <a href="/wiki/%D0%9D%D0%B0%D1%87%D0%B0%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D0%B7%D0%B0%D0%B3%D1%80%D1%83%D0%B7%D0%BA%D0%B0_%D0%BA%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80%D0%B0" title="Начальная загрузка компьютера">загрузке системы</a>. Аббревиатура была необходима из-за ограничения <a href="/wiki/8.3" title="8.3">8.3</a> на длину имени файла (8 знаков собственно имени и 3 знака <a href="/wiki/%D0%A0%D0%B0%D1%81%D1%88%D0%B8%D1%80%D0%B5%D0%BD%D0%B8%D0%B5_%D0%B8%D0%BC%D0%B5%D0%BD%D0%B8_%D1%84%D0%B0%D0%B9%D0%BB%D0%B0" title="Расширение имени файла">расширения</a>) в файловой системе <a href="/wiki/FAT" title="FAT">FAT</a><sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup>.
</p><p>AUTOEXEC.BAT ничем не отличается от прочих пакетных файлов (в отличие от другого файла конфигурирования DOS, Windows 9x и OS/2 — <a href="/wiki/CONFIG.SYS" title="CONFIG.SYS">CONFIG.SYS</a>). Как правило, он используется в <a href="/wiki/DOS" title="DOS">DOS</a> для установки ключевых <a href="/wiki/%D0%A1%D1%80%D0%B5%D0%B4%D0%B0_%D0%BE%D0%BA%D1%80%D1%83%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F" title="Среда окружения">переменных окружения</a> (таких как <tt>PATH</tt>), загрузки <a href="/wiki/%D0%94%D1%80%D0%B0%D0%B9%D0%B2%D0%B5%D1%80" title="Драйвер">драйверов</a> и <a href="/wiki/%D0%A0%D0%B5%D0%B7%D0%B8%D0%B4%D0%B5%D0%BD%D1%82%D0%BD%D0%B0%D1%8F_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B0" title="Резидентная программа">резидентных программ</a> (<a class="mw-redirect" href="/wiki/SMARTDRV.EXE" title="SMARTDRV.EXE">SMARTDRV.EXE</a>, <a href="/wiki/MSCDEX" title="MSCDEX">MSCDEX</a>, дра́йвера <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80%D0%BD%D0%B0%D1%8F_%D0%BC%D1%8B%D1%88%D1%8C" title="Компьютерная мышь">мыши</a>, программ русификации клавиатуры и экрана и т. п.), а также запуска утилит конфигурации (например, для настройки параметров звуковых карт) и проверки системы (например, <a href="/wiki/%D0%90%D0%BD%D1%82%D0%B8%D0%B2%D0%B8%D1%80%D1%83%D1%81%D0%BD%D0%B0%D1%8F_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B0" title="Антивирусная программа">антивирусных сканеров</a>).
</p>

<h2><span id=".D0.98.D1.81.D0.BF.D0.BE.D0.BB.D1.8C.D0.B7.D0.BE.D0.B2.D0.B0.D0.BD.D0.B8.D0.B5"></span><span class="mw-headline" id="Использование">Использование</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=1" title="Редактировать раздел «Использование»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=1" title="Редактировать код раздела «Использование»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="MS-DOS">MS-DOS</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=2" title="Редактировать раздел «MS-DOS»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=2" title="Редактировать код раздела «MS-DOS»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>В <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a> AUTOEXEC.BAT исполняется после старта командного интерпретатора, который загружается после обработки команд из файла конфигурации <a href="/wiki/CONFIG.SYS" title="CONFIG.SYS">CONFIG.SYS</a>. С помощью директивы <code>shell=</code> в CONFIG.SYS можно задействовать командный интерпретатор, отличный от COMMAND.COM, и/или указать имя для стартового пакетного файла, отличное от AUTOEXEC.BAT. При отсутствии файла AUTOEXEC.BAT (или при отмене его исполнения) в большинстве версий DOS командный интерпретатор запрашивает у пользователя текущие дату и время.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>
</p><p>Начиная с MS-DOS 6.0,<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> появилась возможность пропустить исполнение AUTOEXEC.BAT с помощью функциональных клавиш <kbd style="border:1px solid #a2a9b1; border-radius:.2em; box-shadow:.1em .2em .2em #c8ccd1; background-color:#f8f9fa; font-family:inherit; font-size:0.85em; background-image:linear-gradient(top, #eaecf0, #f8f9fa, #eaecf0); padding:.1em .3em; white-space:nowrap">F5</kbd> и <kbd style="border:1px solid #a2a9b1; border-radius:.2em; box-shadow:.1em .2em .2em #c8ccd1; background-color:#f8f9fa; font-family:inherit; font-size:0.85em; background-image:linear-gradient(top, #eaecf0, #f8f9fa, #eaecf0); padding:.1em .3em; white-space:nowrap">F8</kbd> (для этого в COMMAND.COM ввели поддержку опций <tt>/Y</tt> и <tt>/D</tt>), но с помощью директивы <code>switches=</code> в файле CONFIG.SYS эти клавиши можно заблокировать.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup>
</p>
<h3><span id=".D0.9A.D0.BB.D0.BE.D0.BD.D1.8B_MS-DOS"></span><span class="mw-headline" id="Клоны_MS-DOS">Клоны MS-DOS</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=3" title="Редактировать раздел «Клоны MS-DOS»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=3" title="Редактировать код раздела «Клоны MS-DOS»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Клоны MS-DOS кроме файла AUTOEXEC.BAT могут использовать файлы и с другими именами, что облегчает сосуществование разных версий DOS на одном диске. Например, в <a class="mw-redirect" href="/wiki/Caldera" title="Caldera">Caldera</a> <a href="/wiki/DR-DOS" title="DR-DOS">DR-DOS</a> 7 вместо AUTOEXEC.BAT исполняется файл AUTODOS7.BAT<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup>, если такой имеется на диске.
</p>
<h3><span id="Windows_95_.D0.B8_Windows_98_.28MS-DOS_7.x.29"></span><span class="mw-headline" id="Windows_95_и_Windows_98_(MS-DOS_7.x)">Windows 95 и Windows 98 (MS-DOS 7.x)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=4" title="Редактировать раздел «Windows 95 и Windows 98 (MS-DOS 7.x)»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=4" title="Редактировать код раздела «Windows 95 и Windows 98 (MS-DOS 7.x)»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Windows_95" title="Windows 95">Windows 95</a> и <a href="/wiki/Windows_98" title="Windows 98">Windows 98</a> основаны на MS-DOS 7.x и работа с AUTOEXEC.BAT в этих операционных системах происходит так же, как и в предыдущих версиях MS-DOS. Отличие же MS-DOS 7.x от предыдущих версий заключается в том, что по окончании исполнения AUTOEXEC.BAT автоматически запускается графическая оболочка, а не выводится командная строка. Однако, с помощью настройки <code>BootGUI</code> в файле <a href="/wiki/MSDOS.SYS" title="MSDOS.SYS">MSDOS.SYS</a> это можно отменить, и тогда пользователь, как и ранее, будет оставаться в командной строке, из которой графическую оболочку можно запустить вручную командой <tt><a href="/wiki/WIN.COM" title="WIN.COM">WIN</a></tt>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup>
</p>
<h4><span id=".D0.94.D0.B2.D0.BE.D0.B9.D0.BD.D0.B0.D1.8F_.D0.B7.D0.B0.D0.B3.D1.80.D1.83.D0.B7.D0.BA.D0.B0_DOS_.D0.B8_Windows_9x"></span><span class="mw-headline" id="Двойная_загрузка_DOS_и_Windows_9x">Двойная загрузка DOS и Windows 9x</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=5" title="Редактировать раздел «Двойная загрузка DOS и Windows 9x»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=5" title="Редактировать код раздела «Двойная загрузка DOS и Windows 9x»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h4>
<p>При установке <a href="/wiki/Windows_9x" title="Windows 9x">Windows 9x</a> поверх уже установленной DOS или <a href="/wiki/Windows" title="Windows">Windows</a> файл AUTOEXEC.BAT (так же, как <a href="/wiki/CONFIG.SYS" title="CONFIG.SYS">CONFIG.SYS</a> и системные исполняемые файлы <a href="/wiki/IO.SYS" title="IO.SYS">IO.SYS</a>, <a href="/wiki/MSDOS.SYS" title="MSDOS.SYS">MSDOS.SYS</a> и <a href="/wiki/COMMAND.COM" title="COMMAND.COM">COMMAND.COM</a>) переименовывается в <b>AUTOEXEC.DOS</b>.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> Это делается с целью облегчения двойной загрузки между Windows 9x и DOS: при загрузке в предыдущую версию системы (через выбор соответствующего пункта из стартового меню Windows, вызываемого клавишей <kbd style="border:1px solid #a2a9b1; border-radius:.2em; box-shadow:.1em .2em .2em #c8ccd1; background-color:#f8f9fa; font-family:inherit; font-size:0.85em; background-image:linear-gradient(top, #eaecf0, #f8f9fa, #eaecf0); padding:.1em .3em; white-space:nowrap">F8</kbd>, или через нажатие клавиши <kbd style="border:1px solid #a2a9b1; border-radius:.2em; box-shadow:.1em .2em .2em #c8ccd1; background-color:#f8f9fa; font-family:inherit; font-size:0.85em; background-image:linear-gradient(top, #eaecf0, #f8f9fa, #eaecf0); padding:.1em .3em; white-space:nowrap">F4</kbd> при старте системы), этот файл временно, до следующей перезагрузки, переименовывается обратно, а файл AUTOEXEC.BAT, созданный для Windows 9x, переименовывается в AUTOEXEC.W40.<sup class="reference" id="cite_ref-mdgx-reng_8-0"><a href="#cite_note-mdgx-reng-8">[8]</a></sup> (При этом файл IO.SYS в Windows 95 переименовывается в WINBOOT.SYS, а в <a href="/wiki/Windows_98" title="Windows 98">Windows 98</a>/<a class="mw-redirect" href="/wiki/Windows_ME" title="Windows ME">ME</a> — в JO.SYS).<sup class="reference" id="cite_ref-mdgx-reng_8-1"><a href="#cite_note-mdgx-reng-8">[8]</a></sup><sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup>
</p>
<h4><span id=".D0.9F.D0.B5.D1.80.D0.B5.D0.B7.D0.B0.D0.B3.D1.80.D1.83.D0.B7.D0.BA.D0.B0_.D0.B2_.D1.80.D0.B5.D0.B6.D0.B8.D0.BC.D0.B5_MS-DOS_.D0.B8.D0.B7-.D0.BF.D0.BE.D0.B4_Windows"></span><span class="mw-headline" id="Перезагрузка_в_режиме_MS-DOS_из-под_Windows">Перезагрузка в режиме MS-DOS из-под Windows</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=6" title="Редактировать раздел «Перезагрузка в режиме MS-DOS из-под Windows»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=6" title="Редактировать код раздела «Перезагрузка в режиме MS-DOS из-под Windows»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h4>
<p>При выборе пункта «перезагрузить компьютер в режиме MS-DOS» (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Restart the computer in MS-DOS mode</span>) в диалоге «Завершение работы Windows» (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Shut Down Windows</span>) или при запуске DOS-приложения через ярлык в Windows 9x имеется возможность задать конфигурацию (CONFIG.SYS и AUTOEXEC.BAT), отличную от основной конфигурации.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup><sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup><sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup><sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup>
</p><p>При перезагрузке компьютера через выбор пункта «перезагрузить компьютер в режиме MS-DOS» в диалоге «Завершение работы Windows», Windows закрывает все запущенные программы и выгружает себя из памяти, после чего выходит в командную строку в <a href="/wiki/%D0%A0%D0%B5%D0%B0%D0%BB%D1%8C%D0%BD%D1%8B%D0%B9_%D1%80%D0%B5%D0%B6%D0%B8%D0%BC" title="Реальный режим">реальном режиме</a>. Если в свойствах (закладка «Программа», кнопка «Дополнительно») файла <tt>Exit to DOS.pif</tt> (<tt>Выход в Dos.PIF</tt> в русской версии Windows), который находится в каталоге Windows и управляет перезагрузкой в режим MS-DOS, выбрана опция «Использовать текущую конфигурацию MS-DOS» (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Use Current MS-DOS Configuration</span>), то исполняется файл DOSSTART.BAT. В противном случае, если выбрана опция «Выбрать новую конфигурацию MS-DOS» (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Specify A New MS-DOS Configuration</span>), вместо основных CONFIG.SYS и AUTOEXEC.BAT используются файлы с содержанием, указанным в тех же настройках. Для возврата к Windows после перезапуска компьютера в режиме MS-DOS нужно выполнить команду <code>exit</code> в командной строке.
</p><p>При запуске через ярлык программы, настроенной на запуск в режиме MS-DOS со своей конфигурацией (закладка «Программ», кнопка «Дополнительно», опция «Выбрать новую конфигурацию MS-DOS»), файлы CONFIG.SYS и AUTOEXEC.BAT переименовываются в файлы CONFIG.WOS и AUTOEXEC.WOS (в случае отсутствия этих файлов создаются файлы нулевой длины). После чего создаются новые файлы CONFIG.SYS и AUTOEXEC.BAT и туда копируются строки, указанные в настройках ярлыка. При этом первой строкой нового CONFIG.SYS будет строка <code>dos=single</code>, а последней строкой AUTOEXEC.BAT — <code><i>&lt;windir&gt;</i>\win.com /wx</code>" (где вместо <i>&lt;windir&gt;</i> записывается путь к каталогу Windows). Также, файл AUTOEXEC.BAT будет содержать команды <code>CD</code> для перехода в каталог с программой и <code>CALL</code> для запуска программы. Далее Windows перезапускает компьютер и, обнаружив файлы CONFIG.WOS и AUTOEXEC.WOS, IO.SYS при загрузке системы выдаёт сообщение «Windows 95/98 is now starting your MS-DOS-based program».
</p><p>Опция <tt>/wx</tt> в команде запуска win.com заставляет Windows переименовать CONFIG.SYS и AUTOEXEC.BAT в CONFIG.APP и AUTOEXEC.APP, вернуть исходные файлы CONFIG.SYS и AUTOEXEC.BAT из файлов CONFIG.WOS и AUTOEXEC.WOS и перезагрузиться. После перезагрузки, при старте Windows, содержимое файлов CONFIG.APP и AUTOEXEC.APP копируется обратно в свойства DOS-программы, которая и была запущена в режиме MS-DOS.
</p>
<h3><span id="Windows_ME_.28MS-DOS_8.0.29"></span><span class="mw-headline" id="Windows_ME_(MS-DOS_8.0)">Windows ME (MS-DOS 8.0)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=7" title="Редактировать раздел «Windows ME (MS-DOS 8.0)»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=7" title="Редактировать код раздела «Windows ME (MS-DOS 8.0)»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>В <a class="mw-redirect" href="/wiki/Windows_ME" title="Windows ME">Windows ME</a> в файле AUTOEXEC.BAT игнорируются все команды, кроме команд установки <a href="/wiki/%D0%A1%D1%80%D0%B5%D0%B4%D0%B0_%D0%BE%D0%BA%D1%80%D1%83%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F" title="Среда окружения">переменных окружения</a><sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup>, однако известны способы обхода этого ограничения<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup>.
</p>
<h3><span id=".D0.A1.D0.B5.D0.BC.D0.B5.D0.B9.D1.81.D1.82.D0.B2.D0.BE_Windows_NT"></span><span class="mw-headline" id="Семейство_Windows_NT">Семейство Windows NT</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=8" title="Редактировать раздел «Семейство Windows NT»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=8" title="Редактировать код раздела «Семейство Windows NT»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>В операционных системах семейства <a href="/wiki/Windows_NT" title="Windows NT">Windows NT</a> AUTOEXEC.BAT обрабатывается при входе пользователя в систему, и, как и в <a class="mw-redirect" href="/wiki/Windows_ME" title="Windows ME">Windows ME</a>, в нём игнорируются все команды, кроме команд установки переменных окружения (<code>PATH</code>, <code>PROMPT</code> и <code>SET</code>).<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> После обработки переменные из AUTOEXEC.BAT добавляются к переменным, заданным в <a href="/wiki/%D0%A0%D0%B5%D0%B5%D1%81%D1%82%D1%80_Windows" title="Реестр Windows">реестре</a> (в том числе, содержимое переменной <code>PATH</code> дописывается к содержимому, сформированному Windows). Обработку AUTOEXEC.BAT можно отменить, установив в 0 значение ключа реестра <tt>HKCU\Software\Microsoft\Windows NT\CurrentVersion\Winlogon\ParseAutoexec</tt>.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup>
</p>
<h2><span id="OS.2F2"></span><span class="mw-headline" id="OS/2">OS/2</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=9" title="Редактировать раздел «OS/2»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=9" title="Редактировать код раздела «OS/2»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>В операционных системах семейства <a href="/wiki/OS/2" title="OS/2">OS/2</a> файл AUTOEXEC.BAT отрабатывается при запуске сессии <a href="/wiki/DOS" title="DOS">DOS</a> в Virtual DOS Machine.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup>
</p>
<h2><span id=".D0.94.D1.80.D1.83.D0.B3.D0.B8.D0.B5_.D0.B0.D0.B2.D1.82.D0.BE.D1.81.D1.82.D0.B0.D1.80.D1.82.D1.83.D1.8E.D1.89.D0.B8.D0.B5_.D0.BF.D0.B0.D0.BA.D0.B5.D1.82.D0.BD.D1.8B.D0.B5_.D1.84.D0.B0.D0.B9.D0.BB.D1.8B_.D1.81.D0.B8.D1.81.D1.82.D0.B5.D0.BC.D1.8B"></span><span class="mw-headline" id="Другие_автостартующие_пакетные_файлы_системы">Другие автостартующие пакетные файлы системы</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=10" title="Редактировать раздел «Другие автостартующие пакетные файлы системы»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=10" title="Редактировать код раздела «Другие автостартующие пакетные файлы системы»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span id=".D0.A4.D0.B0.D0.B9.D0.BB_autoexec.nt"></span><span class="mw-headline" id="Файл_autoexec.nt">Файл autoexec.nt</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=11" title="Редактировать раздел «Файл autoexec.nt»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=11" title="Редактировать код раздела «Файл autoexec.nt»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>В операционных системах семейства <a href="/wiki/Windows_NT" title="Windows NT">Windows NT</a> файл AUTOEXEC.BAT используется только для чтения переменных окружения. При старте DOS-сессий (для запуска в режиме <a href="/wiki/%D0%92%D0%B8%D1%80%D1%82%D1%83%D0%B0%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D0%BC%D0%B0%D1%88%D0%B8%D0%BD%D0%B0" title="Виртуальная машина">эмуляции</a> приложений, написанных для <a href="/wiki/DOS" title="DOS">DOS</a>) вместо него исполняется файл <b>autoexec.nt</b>, расположенный в <tt>%systemroot%\System32</tt>. Синтаксис этого файла похож на синтаксис AUTOEXEC.BAT, но исполняется он без вывода сообщений о программах и командах на консоль (если только в файле <a class="mw-redirect" href="/wiki/Config.nt" title="Config.nt">config.nt</a> не дана команда <tt>echoconfig</tt><sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup>). Помимо этого, в свойствах ярлыка (pif-файла) для DOS-приложения можно задать собственные файлы config.nt и autoexec.nt (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Custom MS-DOS initialization files</span>).
</p>
<h3><span id=".D0.A4.D0.B0.D0.B9.D0.BB.D1.8B_WINSTART.BAT_.D0.B8_DOSSTART.BAT"></span><span class="mw-headline" id="Файлы_WINSTART.BAT_и_DOSSTART.BAT">Файлы WINSTART.BAT и DOSSTART.BAT</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=12" title="Редактировать раздел «Файлы WINSTART.BAT и DOSSTART.BAT»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=12" title="Редактировать код раздела «Файлы WINSTART.BAT и DOSSTART.BAT»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>В <a href="/wiki/Windows_3.x" title="Windows 3.x">Windows 3.x</a> и операционных системах <a href="/wiki/Windows_95" title="Windows 95">Windows 95</a> и <a href="/wiki/Windows_98" title="Windows 98">Windows 98</a> файл <b>WINSTART.BAT</b>, располагающийся в каталоге Windows (указывается в переменной окружения <tt>%windir%</tt>), исполняется непосредственно перед стартом <a href="/wiki/%D0%93%D1%80%D0%B0%D1%84%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81_%D0%BF%D0%BE%D0%BB%D1%8C%D0%B7%D0%BE%D0%B2%D0%B0%D1%82%D0%B5%D0%BB%D1%8F" title="Графический интерфейс пользователя">графической оболочки</a>.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup>
</p><p>Файл <b>DOSSTART.BAT</b>, добавленный в Windows 95 и Windows 98 и располагающийся также в каталоге Windows, исполняется при выходе из графической оболочки в режим командной строки.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup>
</p>
<h2><span id=".D0.9F.D1.80.D0.B8.D0.BC.D0.B5.D1.80_.D1.84.D0.B0.D0.B9.D0.BB.D0.B0_AUTOEXEC.BAT"></span><span class="mw-headline" id="Пример_файла_AUTOEXEC.BAT">Пример файла AUTOEXEC.BAT</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=AUTOEXEC.BAT&amp;veaction=edit&amp;section=13" title="Редактировать раздел «Пример файла AUTOEXEC.BAT»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AUTOEXEC.BAT&amp;action=edit&amp;section=13" title="Редактировать код раздела «Пример файла AUTOEXEC.BAT»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="mw-highlight mw-highlight-lang-bat mw-content-ltr" dir="ltr"><pre><span></span><span class="p">@</span><span class="k">ECHO</span> OFF
<span class="c1">REM C:\WINDOWS\SMARTDRV.EXE</span>
C<span class="p">:</span><span class="nl">\WINDOWS\SMARTDRV.EXE</span><span class="c1"> 2038 512</span>
<span class="k">PROMPT</span> $p$g
<span class="k">PATH</span> C:\DOS;C:\WINDOWS;C:\LWORKS;C:\EXPLORER.4LC
<span class="k">SET</span> <span class="nv">TEMP</span><span class="p">=</span>C:\DOS
MODE LPT1:,,P <span class="p">&gt;</span>nul
C<span class="p">:</span><span class="nl">\DOS\SHARE.EXE</span><span class="c1"> /F:150 /L:1500</span>
C<span class="p">:</span><span class="nl">\WINDOWS\mouse.COM</span><span class="c1"> /Y</span>
<span class="k">cd</span> windows
WIN
</pre></div>
<p>В этом примере можно видеть отключение эха (дублирование обрабатываемых строк на экран), запуск драйвера SMARTDRV (одна строка закомментирована, в другой SMARTDRV запускается с аргументами), установку переменных окружения (<code>PROMPT</code>, <code>PATH</code>, <code>TEMP</code>), запуск резидентной программы (MODE) и других драйверов (SHARE и MOUSE), и, наконец, переход в каталог <tt>windows</tt> (<code>CD</code>) и запуск собственно Windows (<tt><a href="/wiki/WIN.COM" title="WIN.COM">WIN</a></tt>).
</p>
