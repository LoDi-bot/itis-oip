<p><b>Kernel panic</b> (с <span style="font-style:normal"><a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a></span><span class="nowrap"> —</span> «тревога, сбой в ядре», дословно <i>паника ядра</i>) — сообщение о критической ошибке <a href="/wiki/%D0%AF%D0%B4%D1%80%D0%BE_%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%BE%D0%B9_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D1%8B" title="Ядро операционной системы">ядра операционной системы</a>, после которой операционная система не может продолжать дальнейшую работу<sup class="reference" id="cite_ref-_0210fefa5a0a51e6_1-0"><a href="#cite_note-_0210fefa5a0a51e6-1">[1]</a></sup>.
</p><p>Обычно этот термин применяется в среде операционных систем типа <a class="mw-redirect" href="/wiki/UNIX" title="UNIX">UNIX</a>. Её имя связано с текстом ошибки вида «<code>Kernel panic: …</code>» и именем функции ядра <code>panic()</code> из оригинальной ОС UNIX<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>.
</p><p>Kernel panic возможен на <a href="/wiki/Android" title="Android">Android</a>, который базируется на ядре <a href="/wiki/Linux" title="Linux">Linux</a>, и в самих Linux-системах. Поскольку <a class="mw-redirect" href="/wiki/Mac_OS_X" title="Mac OS X">Mac OS X</a> и <a href="/wiki/IOS" title="IOS">iOS</a> базируются на <a href="/wiki/Darwin" title="Darwin">Darwin</a>, являющейся подсемейством UNIX-систем, они также подвержены kernel panic.<sup class="reference" id="cite_ref-mac_reasons_3-0"><a href="#cite_note-mac_reasons-3">[3]</a></sup>.
</p>

<h2><span id=".D0.98.D1.81.D1.82.D0.BE.D1.80.D0.B8.D1.8F"></span><span class="mw-headline" id="История">История</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Kernel_panic&amp;veaction=edit&amp;section=1" title="Редактировать раздел «История»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Kernel_panic&amp;action=edit&amp;section=1" title="Редактировать код раздела «История»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<figure typeof="mw:File/Thumb"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:Ken_Thompson_and_Dennis_Ritchie--1973.jpg"><img class="mw-file-element" data-file-height="201" data-file-width="310" decoding="async" height="195" src="//upload.wikimedia.org/wikipedia/commons/thumb/1/1b/Ken_Thompson_and_Dennis_Ritchie--1973.jpg/300px-Ken_Thompson_and_Dennis_Ritchie--1973.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/1/1b/Ken_Thompson_and_Dennis_Ritchie--1973.jpg 1.5x" width="300"/></a><figcaption><a href="/wiki/%D0%A2%D0%BE%D0%BC%D0%BF%D1%81%D0%BE%D0%BD,_%D0%9A%D0%B5%D0%BD" title="Томпсон, Кен">Кен Томпсон</a> и <a href="/wiki/%D0%A0%D0%B8%D1%82%D1%87%D0%B8,_%D0%94%D0%B5%D0%BD%D0%BD%D0%B8%D1%81" title="Ритчи, Деннис">Деннис Ритчи</a>, создатели UNIX и концепции kernel panic.</figcaption></figure>
<p>История Kernel panic тесно связана с историей операционной системы <a class="mw-redirect" href="/wiki/UNIX" title="UNIX">UNIX</a>, которая была разработана в конце <a href="/wiki/1960-%D0%B5_%D0%B3%D0%BE%D0%B4%D1%8B" title="1960-е годы">1960-х годов</a> сотрудниками <a class="mw-redirect" href="/wiki/Bell_Labs" title="Bell Labs">Bell Labs</a>, — в первую очередь <a href="/wiki/%D0%A2%D0%BE%D0%BC%D0%BF%D1%81%D0%BE%D0%BD,_%D0%9A%D0%B5%D0%BD" title="Томпсон, Кен">Кеном Томпсоном</a>, <a href="/wiki/%D0%A0%D0%B8%D1%82%D1%87%D0%B8,_%D0%94%D0%B5%D0%BD%D0%BD%D0%B8%D1%81" title="Ритчи, Деннис">Деннисом Ритчи</a> и <a href="/wiki/%D0%9C%D0%B0%D0%BA%D0%B8%D0%BB%D1%80%D0%BE%D0%B9,_%D0%94%D1%83%D0%B3%D0%BB%D0%B0%D1%81" title="Макилрой, Дуглас">Дугласом Макилроем</a>.
</p><p>Сообщение Kernel panic было введено в ранних версиях <a class="mw-redirect" href="/wiki/UNIX" title="UNIX">UNIX</a> и представляло собой важное отличие в философии этой <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Операционная система">операционной системы</a> от <a href="/wiki/Multics" title="Multics">Multics</a> — главного конкурента на то время и предшественника UNIX. Multics был разработан для работы на 36-битном <a href="/wiki/%D0%9C%D0%B5%D0%B9%D0%BD%D1%84%D1%80%D0%B5%D0%B9%D0%BC" title="Мейнфрейм">мейнфрейме</a> <a class="new" href="/w/index.php?title=GE-645&amp;action=edit&amp;redlink=1" title="GE-645 (страница отсутствует)">GE-645</a>, в то время как UNIX разрабатывался для гораздо менее мощного 18-битного мини-компьютера <a href="/wiki/PDP-7" title="PDP-7">PDP-7</a> и по этой причине UNIX было доступно меньше ресурсов, что привело к необходимости их экономии, в том числе и при обработке ошибок. Разработчик Multics Том ван Влек так описывает это изменение в дискуссии с разработчиком UNIX Деннисом Ритчи<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup>:

</p>
<blockquote class="ts-Начало_цитаты-quote">
<p>
Я сказал Деннису, что примерно половина кода, который я написал для Multics, была кодом обработки ошибок. Он ответил: «Мы всё это отбросили. Если произошла ошибка, у нас есть процедура под названием <i>panic</i>, и если она вызвана, компьютер зависает и вы кричите: „Эй, перезапустите его!“».
</p>
<div class="mw-collapsible mw-collapsed has-collapsible-header ts-Скрытый_блок ts-Скрытый_блок-gray ts-Скрытый_блок-noneHideLink ts-oq has-collapsible-header" style=""><div class="ts-Скрытый_блок-title collapsible-header ts-Скрытый_блок-title-leftTitle ts-oq-header collapsible-header" style="text-align:left;">Оригинальный текст <small class="ref-info" style="cursor:help;" title="на английском языке">(англ.)</small></div><div class="mw-collapsible-content ts-oq-content" style="">
<div dir="auto" lang="en">I remarked to Dennis that easily half the code I was writing in Multics was error recovery code. He said, «We left all that stuff out. If there’s an error, we have this routine called panic, and when it is called, the machine crashes, and you holler down the hall, ‘Hey, reboot it.’</div>
</div></div>
</blockquote>
<p>Изначальная функция <code>panic()</code> принципиально не менялась от UNIX V5 до базирующихся на <a href="/wiki/VAX" title="VAX">VAX</a> систем 32V и выводила только сообщение об ошибке без дополнительной информации, после чего система переводилась в <a href="/wiki/%D0%91%D0%B5%D1%81%D0%BA%D0%BE%D0%BD%D0%B5%D1%87%D0%BD%D1%8B%D0%B9_%D1%86%D0%B8%D0%BA%D0%BB" title="Бесконечный цикл">бесконечный пустой цикл</a>. Позже — в процессе развития UNIX — функция <code>panic()</code> была доработана и стала выводить на <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80%D0%BD%D1%8B%D0%B9_%D1%82%D0%B5%D1%80%D0%BC%D0%B8%D0%BD%D0%B0%D0%BB" title="Компьютерный терминал">терминал</a> разнообразную информацию, необходимую для отладки.
</p><p>Подобный принцип обработки критических ошибок был перенят большинством более поздних операционных систем, например <a class="mw-redirect" href="/wiki/Mac_OS" title="Mac OS">Mac OS</a><sup class="reference" id="cite_ref-mac_reasons_3-1"><a href="#cite_note-mac_reasons-3">[3]</a></sup> или <a class="mw-redirect" href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a><sup class="reference" id="cite_ref-Windows_5-0"><a href="#cite_note-Windows-5">[5]</a></sup>.
</p>
<h2><span id=".D0.9F.D1.80.D0.B8.D1.87.D0.B8.D0.BD.D1.8B_Kernel_panic"></span><span class="mw-headline" id="Причины_Kernel_panic">Причины Kernel panic</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Kernel_panic&amp;veaction=edit&amp;section=2" title="Редактировать раздел «Причины Kernel panic»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Kernel_panic&amp;action=edit&amp;section=2" title="Редактировать код раздела «Причины Kernel panic»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<figure typeof="mw:File/Thumb"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:Kernel-panic.jpg"><img class="mw-file-element" data-file-height="1706" data-file-width="2560" decoding="async" height="200" src="//upload.wikimedia.org/wikipedia/commons/thumb/7/71/Kernel-panic.jpg/300px-Kernel-panic.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/7/71/Kernel-panic.jpg/450px-Kernel-panic.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/7/71/Kernel-panic.jpg/600px-Kernel-panic.jpg 2x" width="300"/></a><figcaption>Сообщение об аппаратной ошибке и Kernel panic.</figcaption></figure>
<p>Одной из самых распространённых причин kernel panic является невозможность найти и смонтировать корневую файловую систему. Часто это ошибка конфигурации, которая может быть исправлена при перезагрузке ядра вручную<sup class="reference" id="cite_ref-_35fd3e5c1da233de_6-0"><a href="#cite_note-_35fd3e5c1da233de-6">[6]</a></sup>.
</p><p>В <a href="/wiki/Linux" title="Linux">Linux</a> возникновению паники ядра зачастую предшествует состояние под названием «<a href="/wiki/Linux_oops" title="Linux oops">oops</a>». В ряде случаев «oops» может приводить к такому же неработоспособному состоянию системы, как и паника ядра<sup class="reference" id="cite_ref-_0210fefa5a0a51e6_1-1"><a href="#cite_note-_0210fefa5a0a51e6-1">[1]</a></sup>.
</p><p>В большинстве остальных случаев причиной Kernel panic является критическая аппаратная ошибка (отказ <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%B8%D0%B2%D0%BD%D0%B0%D1%8F_%D0%BF%D0%B0%D0%BC%D1%8F%D1%82%D1%8C" title="Оперативная память">оперативной памяти</a>, ошибка <a class="mw-redirect" href="/wiki/%D0%9F%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BE%D1%80" title="Процессор">процессора</a>, <a href="/wiki/%D0%9C%D0%B0%D1%82%D0%B5%D1%80%D0%B8%D0%BD%D1%81%D0%BA%D0%B0%D1%8F_%D0%BF%D0%BB%D0%B0%D1%82%D0%B0" title="Материнская плата">материнской платы</a>, <a href="/wiki/%D0%92%D0%B8%D0%B4%D0%B5%D0%BE%D0%BA%D0%B0%D1%80%D1%82%D0%B0" title="Видеокарта">видеокарты</a> или иного критически важного устройства) или ошибка в самом <a href="/wiki/%D0%AF%D0%B4%D1%80%D0%BE_%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%BE%D0%B9_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D1%8B" title="Ядро операционной системы">ядре ОС</a> (например, попытка обращения к неверному или запрещённому адресу в памяти). Также причиной для Kernel panic могут быть ошибки в <a href="/wiki/%D0%94%D1%80%D0%B0%D0%B9%D0%B2%D0%B5%D1%80" title="Драйвер">драйверах</a> аппаратных (в частности периферийных) устройств или ошибки в файловой системе<sup class="reference" id="cite_ref-mac_reasons_3-2"><a href="#cite_note-mac_reasons-3">[3]</a></sup><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup>. Во время финальной стадии инициализации <a class="mw-redirect" href="/wiki/%D0%9F%D1%80%D0%BE%D1%81%D1%82%D1%80%D0%B0%D0%BD%D1%81%D1%82%D0%B2%D0%BE_%D0%BF%D0%BE%D0%BB%D1%8C%D0%B7%D0%BE%D0%B2%D0%B0%D1%82%D0%B5%D0%BB%D1%8F" title="Пространство пользователя">пространства пользователя</a> kernel panic обычно возникает тогда, когда не удаётся запустить <a href="/wiki/Init" title="Init">init</a>, так как, несмотря на запущенное и работоспособное ядро, сама система все ещё остаётся в нерабочем состоянии<sup class="reference" id="cite_ref-Mauerer2008_8-0"><a href="#cite_note-Mauerer2008-8">[8]</a></sup>.
</p><p>Kernel panic может быть вызван и прикладной программой, если та некорректно работает с ядром. Так в 2012 году ошибка в <a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a> вызывала Kernel panic в Mac OS X<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup>.
</p>
<h2><span id=".D0.98.D1.81.D1.85.D0.BE.D0.B4.D0.BD.D1.8B.D0.B9_.D0.BA.D0.BE.D0.B4_.D1.84.D1.83.D0.BD.D0.BA.D1.86.D0.B8.D0.B8_panic.28.29"></span><span class="mw-headline" id="Исходный_код_функции_panic()">Исходный код функции panic()</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Kernel_panic&amp;veaction=edit&amp;section=3" title="Редактировать раздел «Исходный код функции panic()»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Kernel_panic&amp;action=edit&amp;section=3" title="Редактировать код раздела «Исходный код функции panic()»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Исходный код функции panic() в <a class="mw-redirect" href="/wiki/UNIX" title="UNIX">UNIX</a> V6<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup>:
</p>
<div class="mw-highlight mw-highlight-lang-c mw-content-ltr" dir="ltr"><pre><span></span><span class="w"> </span><span class="kt">char</span><span class="w">    </span><span class="o">*</span><span class="n">panicstr</span><span class="p">;</span>
<span class="w"> </span><span class="cm">/*</span>
<span class="cm">  * Panic is called on unresolvable</span>
<span class="cm">  * fatal errors.</span>
<span class="cm">  * It syncs, prints "panic: mesg" and</span>
<span class="cm">  * then loops.</span>
<span class="cm"> */</span>
<span class="w"> </span><span class="n">panic</span><span class="p">(</span><span class="n">s</span><span class="p">)</span>
<span class="w"> </span><span class="kt">char</span><span class="w"> </span><span class="o">*</span><span class="n">s</span><span class="p">;</span>
<span class="w"> </span><span class="p">{</span>
<span class="w"> 	</span><span class="n">panicstr</span><span class="w"> </span><span class="o">=</span><span class="w"> </span><span class="n">s</span><span class="p">;</span>
<span class="w"> 	</span><span class="n">update</span><span class="p">();</span>
<span class="w"> 	</span><span class="n">printf</span><span class="p">(</span><span class="s">"panic:%s</span><span class="se">\n</span><span class="s">"</span><span class="p">,</span><span class="w"> </span><span class="n">s</span><span class="p">);</span>
<span class="w"> 	</span><span class="k">for</span><span class="p">(;;)</span>
<span class="w"> 		</span><span class="n">idle</span><span class="p">();</span>
<span class="w"> </span><span class="p">}</span>
</pre></div>
<h2><span id=".D0.9E.D0.B1.D1.80.D0.B0.D0.B1.D0.BE.D1.82.D0.BA.D0.B0_Kernel_panic"></span><span class="mw-headline" id="Обработка_Kernel_panic">Обработка Kernel panic</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Kernel_panic&amp;veaction=edit&amp;section=4" title="Редактировать раздел «Обработка Kernel panic»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Kernel_panic&amp;action=edit&amp;section=4" title="Редактировать код раздела «Обработка Kernel panic»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>В нормальном случае при возникновении Kernel panic происходит аварийная остановка (авост) работы <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Операционная система">операционной системы</a> с выдачей сообщений об ошибках на экран, после чего система ожидает выключения или <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%B7%D0%B0%D0%B3%D1%80%D1%83%D0%B7%D0%BA%D0%B0" title="Перезагрузка">перезагрузки</a> <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80" title="Компьютер">компьютера</a>. Однако такой механизм обработки этого события неприемлем тогда, когда простой компьютера крайне нежелателен или человека нет рядом (например, на удалённых <a href="/wiki/%D0%A1%D0%B5%D1%80%D0%B2%D0%B5%D1%80_(%D0%B0%D0%BF%D0%BF%D0%B0%D1%80%D0%B0%D1%82%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5)" title="Сервер (аппаратное обеспечение)">серверах</a> или в нерабочее время)<sup class="reference" id="cite_ref-_e91a844b7fb8b9b1_11-0"><a href="#cite_note-_e91a844b7fb8b9b1-11">[11]</a></sup>.
</p><p>В современных ОС (в частности в <a class="mw-redirect" href="/wiki/GNU/Linux" title="GNU/Linux">GNU/Linux</a>, <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a> или <a href="/wiki/Solaris" title="Solaris">Solaris</a>) существует возможность изменить стандартное поведение функции panic() и производить перезагрузку компьютера автоматически. В <a class="mw-redirect" href="/wiki/GNU/Linux" title="GNU/Linux">GNU/Linux</a> данная настройка осуществляется при помощи <a href="/wiki/Procfs" title="Procfs">procfs</a><sup class="reference" id="cite_ref-_e91a844b7fb8b9b1_11-1"><a href="#cite_note-_e91a844b7fb8b9b1-11">[11]</a></sup>:
</p>
<div class="mw-highlight mw-highlight-lang-bash mw-content-ltr" dir="ltr"><pre><span></span><span class="nb">echo</span><span class="w"> </span><span class="m">5</span><span class="w"> </span>&gt;<span class="w"> </span>/proc/sys/kernel/panic
</pre></div>
<p>Чтобы изменения действовали в <a class="mw-redirect" href="/wiki/GNU/Linux" title="GNU/Linux">GNU/Linux</a> и после перезагрузки, необходимо добавить в файл <code>/etc/sysctl.d/99-sysctl.conf</code> строку:
</p>
<div class="mw-highlight mw-highlight-lang-ini mw-content-ltr" dir="ltr"><pre><span></span><span class="na">kernel.panic</span><span class="o">=</span><span class="s">5</span>
</pre></div>
<p>Значение параметра kernel.panic — количество секунд, после которых произойдёт перезагрузка. При установке отрицательного или равного 0 значения этого параметра автоматической перезагрузка не произойдёт<sup class="reference" id="cite_ref-_e91a844b7fb8b9b1_11-2"><a href="#cite_note-_e91a844b7fb8b9b1-11">[11]</a></sup>.
</p><p>Также в <a href="/wiki/BSD" title="BSD">BSD</a>-системах есть специальная опция в ядре. Цитата из файла <code>/usr/src/sys/conf/NOTES</code><sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup>:
</p>
<div class="mw-highlight mw-highlight-lang-c mw-content-ltr" dir="ltr"><pre><span></span><span class="cp"># Set the amount of time (in seconds) the system will wait before</span>
<span class="cp"># rebooting automatically when a kernel panic occurs. If set to (-1),</span>
<span class="cp"># the system will wait indefinitely until a key is pressed on the</span>
<span class="cp"># console.</span>
<span class="n">options</span><span class="w"> </span><span class="n">PANIC_REBOOT_WAIT_TIME</span><span class="o">=</span><span class="mi">16</span>
</pre></div>
<p>В <a href="/wiki/Solaris" title="Solaris">Solaris</a> автоматическая перезагрузка после Kernel panic является стандартным поведением системы<sup class="reference" id="cite_ref-_3e14f6ba5882955e_13-0"><a href="#cite_note-_3e14f6ba5882955e-13">[13]</a></sup>.
</p><p>Перезагрузка после Kernel panic имеет и очень серьёзный недостаток, особенно если это изменение не пропадает после первой <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%B7%D0%B0%D0%B3%D1%80%D1%83%D0%B7%D0%BA%D0%B0" title="Перезагрузка">перезагрузки</a>. В случае, если перезагрузка не устраняет ошибку, вызывавшую Kernel panic, система будет останавливаться и перезапускаться вновь и вновь, что может привести к аппаратным ошибкам или потерям данных<sup class="reference" id="cite_ref-_35fd3e5c1da233de_6-1"><a href="#cite_note-_35fd3e5c1da233de-6">[6]</a></sup>. В случае если такая ситуация возникла после сборки нового ядра, решением проблемы может стать загрузка сохранённой копии старого работающего ядра. Как правило, для этого достаточно вручную указать при загрузке путь к работоспособной копии ядра<sup class="reference" id="cite_ref-_92103225696d220c_14-0"><a href="#cite_note-_92103225696d220c-14">[14]</a></sup>.
</p><p>Для изучения причины паники ядра Linux может пригодиться файл <a href="/wiki/System.map" title="System.map">System.map</a><sup class="reference" id="cite_ref-_fbf645480376e823_15-0"><a href="#cite_note-_fbf645480376e823-15">[15]</a></sup>.
</p>
<h2><span id="Kernel_panic_.D0.B2_.D1.80.D0.B0.D0.B7.D0.BB.D0.B8.D1.87.D0.BD.D1.8B.D1.85_.D0.BE.D0.BF.D0.B5.D1.80.D0.B0.D1.86.D0.B8.D0.BE.D0.BD.D0.BD.D1.8B.D1.85_.D1.81.D0.B8.D1.81.D1.82.D0.B5.D0.BC.D0.B0.D1.85"></span><span class="mw-headline" id="Kernel_panic_в_различных_операционных_системах">Kernel panic в различных операционных системах</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Kernel_panic&amp;veaction=edit&amp;section=5" title="Редактировать раздел «Kernel panic в различных операционных системах»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Kernel_panic&amp;action=edit&amp;section=5" title="Редактировать код раздела «Kernel panic в различных операционных системах»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<figure typeof="mw:File/Thumb"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:MacOSX_kernel_panic.png"><img class="mw-file-element" data-file-height="255" data-file-width="472" decoding="async" height="162" src="//upload.wikimedia.org/wikipedia/commons/thumb/0/04/MacOSX_kernel_panic.png/300px-MacOSX_kernel_panic.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/0/04/MacOSX_kernel_panic.png/450px-MacOSX_kernel_panic.png 1.5x, //upload.wikimedia.org/wikipedia/commons/0/04/MacOSX_kernel_panic.png 2x" width="300"/></a><figcaption>Kernel panic в <a class="mw-redirect" href="/wiki/Mac_OS_X" title="Mac OS X">Mac OS X</a> версии 10.6 и ниже.</figcaption></figure>
<p>Изначально сообщение о Kernel panic ограничивалось коротким текстом о необходимости перезагрузки системы. В современных системах обычно выдается больше дополнительной информации.
</p>
<ul><li><a class="mw-redirect" href="/wiki/GNU/Linux" title="GNU/Linux">GNU/Linux</a> и большинство других <a class="mw-redirect" href="/wiki/UNIX" title="UNIX">UNIX</a>-совместимых операционных систем создают лог с описанием ошибки и выводят на экран сообщение об ошибке, содержащее информацию, необходимую для <a class="mw-redirect" href="/wiki/%D0%9E%D1%82%D0%BB%D0%B0%D0%B4%D0%BA%D0%B0" title="Отладка">отладки</a> и поиска причин этой ошибки. Этот механизм носит название <a href="/wiki/Linux_oops" title="Linux oops">Linux oops</a>. В современных дистрибутивах Linux используется графический сервер <a class="mw-redirect" href="/wiki/X_Window" title="X Window">X Window</a>, и Kernel panic не приводит к переключению на физическую консоль, на которую выводятся диагностические сообщения. Распознать Kernel panic можно по мигающим светодиодам Caps Lock и Scroll Lock на клавиатуре<sup class="reference" id="cite_ref-_0210fefa5a0a51e5_16-0"><a href="#cite_note-_0210fefa5a0a51e5-16">[16]</a></sup>.</li>
<li>В изначальных версиях <a class="mw-redirect" href="/wiki/Mac_OS_X" title="Mac OS X">Mac OS X</a> (от 10.0 до 10.0.1.5) по аналогии с операционными системами, базирующимися на ядре Linux, на экран выводилась информация о произошедшей ошибке, после чего система останавливалась. Начиная с версии Mac OS X 10.2 это сообщение было упрощено и указывает лишь на необходимость перезапустить компьютер на четырёх языках (английском, немецком, французском и японском) вне зависимости от языковой версии операционной системы<sup class="reference" id="cite_ref-mac_reasons_3-3"><a href="#cite_note-mac_reasons-3">[3]</a></sup><sup class="reference" id="cite_ref-mac_screens_17-0"><a href="#cite_note-mac_screens-17">[17]</a></sup>. Однако OS X позволяет<sup class="reference" id="cite_ref-mac_screens_17-1"><a href="#cite_note-mac_screens-17">[17]</a></sup> заменить изображение на любое другое, что дает возможность разработчикам показывать изменённые сообщения об ошибках в различных ситуациях<sup class="reference" id="cite_ref-mac_screens_17-2"><a href="#cite_note-mac_screens-17">[17]</a></sup>. Благодаря этой возможности на OS X возможно даже симулировать <a href="/wiki/%D0%A1%D0%B8%D0%BD%D0%B8%D0%B9_%D1%8D%D0%BA%D1%80%D0%B0%D0%BD_%D1%81%D0%BC%D0%B5%D1%80%D1%82%D0%B8" title="Синий экран смерти">синий экран смерти</a> операционной системы <a href="/wiki/Windows" title="Windows">Windows</a>, заменив стандартное изображение <a class="mw-redirect" href="/wiki/%D0%A1%D0%BA%D1%80%D0%B8%D0%BD%D1%88%D0%BE%D1%82" title="Скриншот">скриншотом</a> соответствующего изображения <a href="/wiki/Windows" title="Windows">Windows</a><sup class="reference" id="cite_ref-mac_screens_17-3"><a href="#cite_note-mac_screens-17">[17]</a></sup>.</li></ul>
<h2><span id=".D0.92_.D0.BD.D0.B5-UNIX_.D0.BE.D0.BF.D0.B5.D1.80.D0.B0.D1.86.D0.B8.D0.BE.D0.BD.D0.BD.D1.8B.D1.85_.D1.81.D0.B8.D1.81.D1.82.D0.B5.D0.BC.D0.B0.D1.85"></span><span class="mw-headline" id="В_не-UNIX_операционных_системах">В не-UNIX операционных системах</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Kernel_panic&amp;veaction=edit&amp;section=6" title="Редактировать раздел «В не-UNIX операционных системах»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Kernel_panic&amp;action=edit&amp;section=6" title="Редактировать код раздела «В не-UNIX операционных системах»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>В то время как термин Kernel panic употребляется в основном для <a class="mw-redirect" href="/wiki/UNIX" title="UNIX">UNIX</a>-совместимых операционных систем, в других <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Операционная система">операционных системах</a> обработка критических ошибок методом остановки системы тоже прижилась и получила следующие названия:
</p>
<ul><li>В большинстве версий <a class="mw-redirect" href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> система останавливается с выдачей голубого экрана с кратким описанием ошибки<sup class="reference" id="cite_ref-Windows_5-1"><a href="#cite_note-Windows-5">[5]</a></sup>, который получил название <a href="/wiki/%D0%A1%D0%B8%D0%BD%D0%B8%D0%B9_%D1%8D%D0%BA%D1%80%D0%B0%D0%BD_%D1%81%D0%BC%D0%B5%D1%80%D1%82%D0%B8" title="Синий экран смерти">синий экран смерти</a>. В операционной системе <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a> при возникновении ошибки компьютер перезагружается автоматически. Это поведение системы управляется через панель управления <a href="/wiki/Windows" title="Windows">Windows</a>. Если ошибка происходит при загрузке ОС, изменить поведение системы можно через меню кнопки F8<sup class="reference" id="cite_ref-Windows_5-2"><a href="#cite_note-Windows-5">[5]</a></sup>.</li>
<li>В старых компьютерах <a href="/wiki/Macintosh" title="Macintosh">Macintosh</a>: <a class="mw-redirect" href="/wiki/Sad_Mac" title="Sad Mac">Sad Mac</a> (аппаратная ошибка при запуске системы)<sup class="reference" id="cite_ref-_a6314c9fe7d7b616_18-0"><a href="#cite_note-_a6314c9fe7d7b616-18">[18]</a></sup>, <span data-interwiki-article="Bomb (symbol)" data-interwiki-lang="en"><a class="new" href="/w/index.php?title=Bomb_(%D1%81%D0%B8%D0%BC%D0%B2%D0%BE%D0%BB)&amp;action=edit&amp;redlink=1" title="Bomb (символ) (страница отсутствует)">Bomb</a></span><span class="noprint" style="font-style:normal; font-weight:normal;"><sup><a class="extiw" href="https://en.wikipedia.org/wiki/Bomb_(symbol)" title="en:Bomb (symbol)"><span title="Bomb (symbol) — версия статьи «Bomb (символ)» на английском языке">[en]</span></a></sup></span> (для ошибок программ или операционной системы)<sup class="reference" id="cite_ref-_2f5691244436013c_19-0"><a href="#cite_note-_2f5691244436013c-19">[19]</a></sup>.</li>
<li>На компьютерах <a href="/wiki/Amiga" title="Amiga">Amiga</a> в <a href="/wiki/AmigaOS" title="AmigaOS">AmigaOS</a> до 2.04 этот механизм назывался <a href="/wiki/Guru_Meditation" title="Guru Meditation">Guru Meditation</a> и работал аналогично Kernel panic в Unix<sup class="reference" id="cite_ref-_dcd478e714613e69_20-0"><a href="#cite_note-_dcd478e714613e69-20">[20]</a></sup>. В последующих версиях текст «Guru meditation» был удалён из сообщения об ошибке<sup class="reference" id="cite_ref-_dcd478e714613e69_20-1"><a href="#cite_note-_dcd478e714613e69-20">[20]</a></sup>.</li></ul>
