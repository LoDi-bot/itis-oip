<p><b>Tcl</b> (от <a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <span lang="en" style="font-style:italic;">Tool Command Language</span> — «командный язык инструментов», читается «тикль» или «ти-си-эль») — <a class="mw-redirect" href="/wiki/%D0%A1%D0%BA%D1%80%D0%B8%D0%BF%D1%82%D0%BE%D0%B2%D1%8B%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Скриптовый язык">скриптовый язык</a> высокого уровня.
</p><p>Tcl часто применяется совместно с <a href="/wiki/%D0%93%D1%80%D0%B0%D1%84%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81_%D0%BF%D0%BE%D0%BB%D1%8C%D0%B7%D0%BE%D0%B2%D0%B0%D1%82%D0%B5%D0%BB%D1%8F" title="Графический интерфейс пользователя">графической</a> <a href="/wiki/%D0%91%D0%B8%D0%B1%D0%BB%D0%B8%D0%BE%D1%82%D0%B5%D0%BA%D0%B0_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Библиотека (программирование)">библиотекой</a> <a href="/wiki/Tk_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B0)" title="Tk (программа)">Tk</a> (Tool Kit). Связку Tcl/Tk по-русски иногда называют «Так-тикль» (английский вариант — «тикль-ток»).
</p><p>Области применения языка — <a href="/wiki/%D0%9F%D1%80%D0%BE%D1%82%D0%BE%D1%82%D0%B8%D0%BF%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B3%D0%BE_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D1%8F#Быстрое_прототипирование" title="Прототипирование программного обеспечения">быстрое прототипирование</a>, создание графических интерфейсов для <a class="mw-redirect" href="/wiki/%D0%9A%D0%BE%D0%BD%D1%81%D0%BE%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B0" title="Консольная программа">консольных программ</a> (пакетов программ), <a class="mw-redirect" href="/wiki/%D0%A1%D0%BA%D1%80%D0%B8%D0%BF%D1%82%D0%BE%D0%B2%D1%8B%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA#Встраиваемые" title="Скриптовый язык">встраивание</a> в прикладные программы, <a href="/wiki/%D0%A2%D0%B5%D1%81%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B3%D0%BE_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D1%8F" title="Тестирование программного обеспечения">тестирование</a>. Также Tcl применяется в <a href="/wiki/%D0%92%D0%B5%D0%B1-%D1%80%D0%B0%D0%B7%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D0%BA%D0%B0" title="Веб-разработка">веб-разработке</a>.
</p>

<h2><span id=".D0.9E.D1.81.D0.BE.D0.B1.D0.B5.D0.BD.D0.BD.D0.BE.D1.81.D1.82.D0.B8"></span><span class="mw-headline" id="Особенности">Особенности</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=1" title="Редактировать раздел «Особенности»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=1" title="Редактировать код раздела «Особенности»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>В Tcl данными всех типов, включая код программы, можно манипулировать как строками. Это делает его языком с естественной поддержкой <a href="/wiki/%D0%9C%D0%B5%D1%82%D0%B0%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Метапрограммирование">метапрограммирования</a>. Эта <a href="/wiki/%D0%9F%D0%B0%D1%80%D0%B0%D0%B4%D0%B8%D0%B3%D0%BC%D0%B0_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F" title="Парадигма программирования">парадигма программирования</a> учитывалась в ходе разработки и эволюции языка.
</p><p>Программа на Tcl состоит из команд, разделённых символами новой строки или точками с запятой. Каждая команда состоит из набора полей, разделённых пробелами. Первое поле должно быть именем команды, а необязательные остальные поля — передаваемые этой команде аргументы. Команда возвращает значение, иногда пустое. То есть, как и в <a class="mw-redirect" href="/wiki/Lisp" title="Lisp">Лиспе</a>, в Tcl используется префиксная нотация.
</p><p><a href="/wiki/%D0%9A%D0%BB%D1%8E%D1%87%D0%B5%D0%B2%D0%BE%D0%B5_%D1%81%D0%BB%D0%BE%D0%B2%D0%BE" title="Ключевое слово">Ключевых слов</a> как таковых нет — понятие команды в Tcl аналогично понятию <a class="mw-redirect" href="/wiki/%D0%9F%D1%80%D0%BE%D1%86%D0%B5%D0%B4%D1%83%D1%80%D0%B0_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Процедура (программирование)">процедуры</a> или <a href="/wiki/%D0%A4%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D1%8F_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Функция (программирование)">функции</a> распространённых языков программирования. Это относится и к управляющим конструкциям языка. В сочетании с элементарным синтаксисом это обеспечивает хорошую расширяемость языка, в том числе и библиотеками, написанными на других языках, таких как <a href="/wiki/%D0%A1%D0%B8_(%D1%8F%D0%B7%D1%8B%D0%BA_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F)" title="Си (язык программирования)">C</a>/<a href="/wiki/C%2B%2B" title="C++">C++</a> или <a href="/wiki/Java" title="Java">Java</a>.
</p><p>В Tcl также качественно реализована <a href="/wiki/%D0%A1%D0%BE%D0%B1%D1%8B%D1%82%D0%B8%D0%B9%D0%BD%D0%BE-%D0%BE%D1%80%D0%B8%D0%B5%D0%BD%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Событийно-ориентированное программирование">модель управления программой на основе событий</a>. События могут генерироваться таймером, при появлении данных в канале, изменении значения переменной, при завершении какой-либо внешней программы или просто при работе пользователя с интерфейсом Tk. Можно задавать свои события и управлять ими.
</p><p>Как и большинство современных <a class="mw-redirect" href="/wiki/%D0%A1%D0%BA%D1%80%D0%B8%D0%BF%D1%82%D0%BE%D0%B2%D1%8B%D0%B5_%D1%8F%D0%B7%D1%8B%D0%BA%D0%B8" title="Скриптовые языки">скриптовых языков</a>, Tcl содержит развитые средства работы с регулярными выражениями, работает с <a href="/wiki/%D0%90%D1%81%D1%81%D0%BE%D1%86%D0%B8%D0%B0%D1%82%D0%B8%D0%B2%D0%BD%D1%8B%D0%B9_%D0%BC%D0%B0%D1%81%D1%81%D0%B8%D0%B2" title="Ассоциативный массив">ассоциативными массивами</a> и другими динамическими структурами данных.
</p><p>В базовом Tcl поддержка <a href="/wiki/%D0%9E%D0%B1%D1%8A%D0%B5%D0%BA%D1%82%D0%BD%D0%BE-%D0%BE%D1%80%D0%B8%D0%B5%D0%BD%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Объектно-ориентированное программирование">ООП</a> отсутствовала до версии 8.6, однако, благодаря простоте расширения синтаксиса языка, существует множество расширений Tcl объектно-ориентированными механизмами, реализованных в виде подключаемых библиотек на <a href="/wiki/%D0%A1%D0%B8_(%D1%8F%D0%B7%D1%8B%D0%BA_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F)" title="Си (язык программирования)">Си</a> или на самом Tcl. До версии 8.5 самыми распространёнными из них были: <a href="/wiki/Incr_Tcl" title="Incr Tcl">incr Tcl</a>, <a href="/wiki/XOTcl" title="XOTcl">XOTcl</a> и <a href="/wiki/Snit" title="Snit">Snit</a><sup class="reference" id="cite_ref-_484a6b82aa792e65_2-0"><a href="#cite_note-_484a6b82aa792e65-2">[2]</a></sup>. Ещё одна реализация механизмов ООП средствами самого Tcl, включённая в стандартную библиотеку Tcl<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup>, — библиотека <a href="/wiki/STOOOP" title="STOOOP">STOOOP</a><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup>. Начиная с версии 8.5 появляется новая реализация ООП для Tcl, <a class="new" href="/w/index.php?title=TclOO&amp;action=edit&amp;redlink=1" title="TclOO (страница отсутствует)">TclOO</a>, включённая, начиная с версии 8.6, в ядро языка<sup class="reference" id="cite_ref-_c2fac8db714287e6_5-0"><a href="#cite_note-_c2fac8db714287e6-5">[5]</a></sup><sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[примечание 1]</a></sup>:

<td>реализация объектной модели на основе <a href="/wiki/%D0%94%D0%B5%D0%BB%D0%B5%D0%B3%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Делегирование (программирование)">делегирования</a>, а не <a href="/wiki/%D0%9D%D0%B0%D1%81%D0%BB%D0%B5%D0%B4%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Наследование (программирование)">наследования</a>
</td></tr>
<tr>
<td><a href="/wiki/STOOOP" title="STOOOP">STOOOP</a>
</td>
<td>Tcl
</td>
<td>ООП в стиле, максимально близком к C++, также введён механизм <i>переключения классов</i>
</td></tr>
<tr>
<td><a class="new" href="/w/index.php?title=TclOO&amp;action=edit&amp;redlink=1" title="TclOO (страница отсутствует)">TclOO</a><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[6]</a></sup>
</td>
<td>C
</td>
<td>Компактная, но мощная объектно-ориентированная система, которую можно использовать как непосредственно, так и как фреймвок для построения расширенных моделей ООП.
</td></tr></tbody></table>
<p>Пространства имён поддерживаются на уровне ядра языка. При реализации механизмов ООП на Tcl используется поддержка иерархии вложенности <a href="/wiki/%D0%9F%D1%80%D0%BE%D1%81%D1%82%D1%80%D0%B0%D0%BD%D1%81%D1%82%D0%B2%D0%BE_%D0%B8%D0%BC%D1%91%D0%BD_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Пространство имён (программирование)">пространств имён</a> в этом языке<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[7]</a></sup>.
</p><p>На Tcl легко программировать в <a href="/wiki/%D0%A4%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%BE%D0%BD%D0%B0%D0%BB%D1%8C%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Функциональное программирование">функциональном стиле</a>: он обеспечивает такие механизмы, как функции высшего порядка и абстракции функций<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[8]</a></sup>, хотя Tcl редко используют таким образом. Вот пример того, как можно осуществить <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D0%BE%D0%B7%D0%B8%D1%86%D0%B8%D1%8F_%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%B9" title="Композиция функций">композицию двух функций</a>:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">proc</span><span class="w"> </span>o<span class="w"> </span><span class="k">{</span><span class="nv">f</span><span class="w"> </span>g<span class="w"> </span>x<span class="k">}</span><span class="w"> </span><span class="k">{</span><span class="nv">$f</span><span class="w"> </span><span class="k">[</span><span class="nv">$g</span><span class="w"> </span><span class="nv">$x</span><span class="k">]}</span>
</pre></div>
<p>В версии 8.5 в Tcl также введена команда <i>apply</i>, значительно облегчающая программирование в этом стиле.
</p><p>Вот пример её использования из официальной документации:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">proc</span><span class="w"> </span>map<span class="w"> </span><span class="k">{</span><span class="nv">lambda</span><span class="w"> </span>list<span class="k">}</span><span class="w"> </span><span class="k">{</span>
<span class="w">   </span><span class="k">set</span><span class="w"> </span>result<span class="w"> </span><span class="k">{}</span>
<span class="w">   </span><span class="k">foreach</span><span class="w"> </span>item<span class="w"> </span><span class="nv">$list</span><span class="w"> </span><span class="k">{</span>
<span class="w">      </span><span class="nb">lappend</span><span class="w"> </span>result<span class="w"> </span><span class="k">[apply</span><span class="w"> </span><span class="nv">$lambda</span><span class="w"> </span><span class="nv">$item</span><span class="k">]</span>
<span class="w">   </span><span class="k">}</span>
<span class="w">   </span><span class="k">return</span><span class="w"> </span><span class="nv">$result</span>
<span class="k">}</span>

<span class="nv">map</span><span class="w"> </span><span class="k">{</span><span class="nv">x</span><span class="w"> </span><span class="k">{return</span><span class="w"> </span><span class="k">[</span><span class="nb">string</span><span class="w"> </span>length<span class="w"> </span><span class="nv">$x</span><span class="k">]</span><span class="o">:</span><span class="nv">$x</span><span class="k">}}</span><span class="w"> </span><span class="k">{</span><span class="nv">a</span><span class="w"> </span>bb<span class="w"> </span>ccc<span class="w"> </span>dddd<span class="k">}</span>
</pre></div>
<p>результатом работы этого примера будет
</p>
<pre>1:a 2:bb 3:ccc 4:dddd</pre>
<h2><span id=".D0.98.D1.81.D1.82.D0.BE.D1.80.D0.B8.D1.8F"></span><span class="mw-headline" id="История">История</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=2" title="Редактировать раздел «История»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=2" title="Редактировать код раздела «История»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Язык разработан <a href="/wiki/%D0%9E%D1%83%D1%81%D1%82%D0%B5%D1%80%D1%85%D0%B0%D1%83%D1%82,_%D0%94%D0%B6%D0%BE%D0%BD" title="Оустерхаут, Джон">Джоном Оустерхаутом</a> (John Ousterhout) в то время, когда он работал в университете Калифорнии в Беркли. Первоначальной целью было создание универсального языка для встраивания в <a class="mw-redirect" href="/wiki/%D0%9A%D0%BE%D0%BD%D1%81%D0%BE%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B0" title="Консольная программа">консольные приложения</a>. Вслед за этим Оустерхаут расширил свой язык средствами создания <a href="/wiki/%D0%93%D1%80%D0%B0%D1%84%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81_%D0%BF%D0%BE%D0%BB%D1%8C%D0%B7%D0%BE%D0%B2%D0%B0%D1%82%D0%B5%D0%BB%D1%8F" title="Графический интерфейс пользователя">графических интерфейсов</a> — так появился Tcl/Tk.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[9]</a></sup>
</p><p>За 4 года развития в университетских стенах Тикль-сообщество выросло до нескольких сотен тысяч программистов. В 1993 году было разработано первое объектно-ориентированное расширение — <a href="/wiki/Incr_Tcl" title="Incr Tcl">incr Tcl</a>.
</p><p>В 1994 г. Оустерхаут становится ведущим инженером проекта SunScript. В течение следующих 4 лет, став одним из проектов корпорации <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun</a>, Тикль не только остался <a href="/wiki/%D0%A1%D0%B2%D0%BE%D0%B1%D0%BE%D0%B4%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Свободное программное обеспечение">свободно распространяемой</a> системой, но и значительно усовершенствовался, стал <a class="mw-redirect" href="/wiki/%D0%9A%D1%80%D0%BE%D1%81%D1%81%D0%BF%D0%BB%D0%B0%D1%82%D1%84%D0%BE%D1%80%D0%BC%D0%B5%D0%BD%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Кроссплатформенное программное обеспечение">кроссплатформенным</a>.
</p><p>Заметный удар по популярности языка нанёс <a href="/wiki/%D0%A1%D1%82%D0%BE%D0%BB%D0%BB%D0%BC%D0%B0%D0%BD,_%D0%A0%D0%B8%D1%87%D0%B0%D1%80%D0%B4_%D0%9C%D1%8D%D1%82%D1%82%D1%8C%D1%8E" title="Столлман, Ричард Мэттью">Ричард Столлман</a>, опубликовав в сентябре 1994 г. в ряде новостных групп рекомендацию «Почему вы не должны использовать Tcl»<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[10]</a></sup>. В ней он на основании утверждений, что <i>«Tcl разработан не как серьёзный язык программирования»</i> и <i>«Tcl имеет особый синтаксис, притягательный для <a href="/wiki/%D0%A5%D0%B0%D0%BA%D0%B5%D1%80" title="Хакер">хакеров</a> из-за своей простоты. Но синтаксис Tcl кажется странным для большинства пользователей.»</i>, а также в связи с продвижением <a href="/wiki/Emacs_Lisp" title="Emacs Lisp">Emacs Lisp</a>, вынес вердикт: <i>«проект <a href="/wiki/GNU" title="GNU">GNU</a> не собирается использовать Tcl в программном обеспечении GNU»</i>. Помимо субъективных претензий к дизайну языка, Столлман указал на объективный на тот момент недостаток — низкое быстродействие.
</p><p>Тем не менее, рост количества разработчиков, активно использующих тикль, составил за этот период 600 %.
</p><p>В 1998 г. в связи с <a href="/wiki/Java" title="Java">Java</a>-бумом и связанной с ним острой конкурентной борьбой корпорация уже не могла уделять тиклю прежнего внимания. Джон Оустерхаут покидает Sun и создаёт (при поддержке Sun) компанию Scriptics, продолжая развитие технологии Tcl/Tk<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[11]</a></sup><sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[12]</a></sup>. Это была одна из первых коммерческих программных фирм, чья деятельность строилась вокруг бесплатного <a href="/wiki/%D0%A1%D0%B2%D0%BE%D0%B1%D0%BE%D0%B4%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Свободное программное обеспечение">свободного ПО</a>.
</p>
<figure typeof="mw:File/Frame"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:Tcl_logo.gif"><img class="mw-file-element" data-file-height="100" data-file-width="64" decoding="async" height="100" src="//upload.wikimedia.org/wikipedia/ru/4/4d/Tcl_logo.gif" width="64"/></a><figcaption>Цветовое решение логотипа, принятое в 90-х гг. XX века</figcaption></figure>
<p>В марте 1998 года Оустерхаут пишет ставшую классической статью «Сценарии: высокоуровневое программирование для XXI века»<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[13]</a></sup>, в которой назвал скриптовые языки языками системной интеграции, потому что они ориентированы прежде всего на работу с объектами операционной среды, а не с элементарными данными. Это позволяет без написания тысяч <a href="/wiki/%D0%9A%D0%BE%D0%BB%D0%B8%D1%87%D0%B5%D1%81%D1%82%D0%B2%D0%BE_%D1%81%D1%82%D1%80%D0%BE%D0%BA_%D0%BA%D0%BE%D0%B4%D0%B0" title="Количество строк кода">строк кода</a> двумя-тремя командами «склеивать» такие объекты в одно целое.
</p><p>В том же году он выиграл за разработку языка Tcl две крупных награды. Первая — это <a class="mw-redirect" href="/wiki/ACM" title="ACM">ACM</a> Software System Award, вручаемая за «наиболее значимое программное обеспечение». Этой наградой до этого были отмечены такие основополагающие системы, как стек протоколов <a href="/wiki/TCP/IP" title="TCP/IP">TCP/IP</a>, первая <a href="/wiki/%D0%AD%D0%BB%D0%B5%D0%BA%D1%82%D1%80%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%82%D0%B0%D0%B1%D0%BB%D0%B8%D1%86%D0%B0" title="Электронная таблица">электронная таблица</a>, первая <a href="/wiki/%D0%A0%D0%B5%D0%BB%D1%8F%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D0%B1%D0%B0%D0%B7%D0%B0_%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85" title="Реляционная база данных">реляционная база данных</a>, <a class="mw-redirect" href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>, <a href="/wiki/Unix" title="Unix">Unix</a>, <a href="/wiki/PostScript" title="PostScript">PostScript</a> и <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a>. Вторая — это ежегодная USENIX Software Tools User Group (STUG) Award в знак признания выдающегося программного обеспечения.
</p><p>В 1999 году выходит Tcl 8.0 — в этой реализация впервые появилась компиляция в <a class="mw-redirect" href="/wiki/%D0%91%D0%B0%D0%B9%D1%82%D0%BA%D0%BE%D0%B4" title="Байткод">байткод</a>, что позволило в 6 раз поднять быстродействие<sup class="reference" id="cite_ref-bench_15-0"><a href="#cite_note-bench-15">[14]</a></sup>. В следующую модификацию Tcl 8.1 введена полноценная поддержка <a href="/wiki/%D0%AE%D0%BD%D0%B8%D0%BA%D0%BE%D0%B4" title="Юникод">Юникода</a>, в ней впервые реализована <a href="/wiki/%D0%9C%D0%BD%D0%BE%D0%B3%D0%BE%D0%BF%D0%BE%D1%82%D0%BE%D1%87%D0%BD%D0%BE%D1%81%D1%82%D1%8C" title="Многопоточность">многопоточность</a>. В связи c этим релизом Оустерхаут заявил:
</p>
<div class="ts-Цитата"><blockquote><p>«В целом 8.1 переводит Tcl в новую категорию, поскольку теперь его можно использовать для серверных приложений. Си, Си++ и Java можно по-прежнему использовать для создания частей приложений, критических с точки зрения производительности, а Tcl — для интеграции и создания интерфейсных компонентов».<i><sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[15]</a></sup></i></p></blockquote></div>
<p>В 2000 году фирма Scriptics была переименована в Ajuba Solutions, а затем — куплена фирмой Interwoven, не заинтересованной в работе со <a href="/wiki/%D0%A1%D0%B2%D0%BE%D0%B1%D0%BE%D0%B4%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Свободное программное обеспечение">свободным ПО</a>. В связи с этим разработка Tcl была передана сообществу программистов. Так была сформирована <a class="external text" href="http://www.tcl.tk/community/coreteam/" rel="nofollow">Tcl Core Team</a>.
</p><p>На 9-й конференции по Tcl/Tk, проходившей в <a href="/wiki/%D0%92%D0%B0%D0%BD%D0%BA%D1%83%D0%B2%D0%B5%D1%80" title="Ванкувер">Ванкувере</a>, была представлена виртуальная файловая система <a href="/wiki/MetaKit#StarKit" title="MetaKit">StarKit</a>, обеспечившая новый, удобный для обычного пользователя способ распространения Tcl-программ.
</p>
<h2><span id=".D0.A1.D0.B8.D0.BD.D1.82.D0.B0.D0.BA.D1.81.D0.B8.D1.81_Tcl"></span><span class="mw-headline" id="Синтаксис_Tcl">Синтаксис Tcl</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=3" title="Редактировать раздел «Синтаксис Tcl»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=3" title="Редактировать код раздела «Синтаксис Tcl»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<dl><dt>Скрипт</dt>
<dd>команды, разделённые переводом строки или точкой с запятой (;).</dd></dl>
<dl><dt>Команда</dt>
<dd>слова, разделённые пробелами.</dd></dl>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="w"> </span><span class="nv">commandName</span><span class="w"> </span>argument1<span class="w"> </span>argument2<span class="w"> </span><span class="err">…</span><span class="w"> </span>argumentN
</pre></div>
<p>Первое слово — имя команды, остальные — её аргументы. Любой аргумент может быть заменён другой командой, помещённой в квадратные скобки. Любые аргументы в фигурных скобках передаются команде «как есть», в виде единого аргумента.
</p>
<h3><span id=".D0.A1.D0.B8.D0.BC.D0.B2.D0.BE.D0.BB.D1.8B.2C_.D0.B8.D0.BC.D0.B5.D1.8E.D1.89.D0.B8.D0.B5_.D0.BE.D1.81.D0.BE.D0.B1.D0.BE.D0.B5_.D0.B7.D0.BD.D0.B0.D1.87.D0.B5.D0.BD.D0.B8.D0.B5"></span><span class="mw-headline" id="Символы,_имеющие_особое_значение">Символы, имеющие особое значение</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=4" title="Редактировать раздел «Символы, имеющие особое значение»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=4" title="Редактировать код раздела «Символы, имеющие особое значение»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<ul><li><tt>$ </tt> — вызывает подстановку значения переменной.</li>
<li><tt>[]</tt> — вызывает подстановку результата выполнения команды внутри скобок.</li>
<li><tt>""</tt> — группирует аргументы в один с подстановкой значений переменных.</li>
<li><tt>{}</tt> — группирует аргументы в один без подстановки значений переменных.</li>
<li><tt>\ </tt> — экранирует следующий символ или вызывает подстановку управляющего символа.</li>
<li><tt># </tt> — начало однострочного комментария, должен быть после неэкранированного символа перевода строки или «;» (точки с запятой).</li></ul>
<h3><span id=".D0.97.D0.B0.D0.BC.D0.B5.D1.87.D0.B0.D0.BD.D0.B8.D1.8F_.D0.BE_.D0.BF.D1.80.D0.BE.D1.86.D0.B5.D1.81.D1.81.D0.B5_.D1.80.D0.B0.D0.B7.D0.B1.D0.BE.D1.80.D0.B0_.D1.81.D0.BA.D1.80.D0.B8.D0.BF.D1.82.D0.B0"></span><span class="mw-headline" id="Замечания_о_процессе_разбора_скрипта">Замечания о процессе разбора скрипта</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=5" title="Редактировать раздел «Замечания о процессе разбора скрипта»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=5" title="Редактировать код раздела «Замечания о процессе разбора скрипта»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<ul><li>Разбивка на слова, подстановка значений переменных и результатов выполнения команд производятся за один проход по команде.</li>
<li>Все символы, находящееся внутри скобок <tt>{}</tt> и кавычек <tt>""</tt>, рассматривается как одно слово.</li>
<li>Первое слово команды рассматривается как встроенная или определённая процедура.</li></ul>
<h3><span id=".D0.9F.D1.80.D0.B8.D0.BC.D0.B5.D1.80.D1.8B_.D0.BA.D0.BE.D0.BC.D0.B0.D0.BD.D0.B4"></span><span class="mw-headline" id="Примеры_команд">Примеры команд</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=6" title="Редактировать раздел «Примеры команд»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=6" title="Редактировать код раздела «Примеры команд»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<ul><li><b>Присваивание</b> обозначается словом <i>set</i>, а не знаком равенства.</li></ul>
<p>Синтаксис:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">set</span><span class="w"> </span>переменная<span class="w"> </span>значение
</pre></div>
<p>Пример:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">set</span><span class="w"> </span>a<span class="w"> </span><span class="mi">2</span>
</pre></div>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">set</span><span class="w"> </span>b<span class="w"> </span><span class="mi">3</span>
</pre></div>
<ul><li><b>Комментарий</b> обозначается символом <i>#</i></li></ul>
<p>Синтаксис:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="c"># текст комментария</span>
</pre></div>
<p>Пример:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="c"># это комментарий</span>
</pre></div>
<ul><li><b>Вывод (печать)</b> реализуется командой <i>puts</i></li></ul>
<p>Синтаксис:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="nb">puts</span><span class="w"> </span>выражение
</pre></div>
<p>Пример:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="nb">puts</span><span class="w"> </span><span class="s2">"Привет! Это я."</span>
<span class="nb">puts</span><span class="w"> </span><span class="mi">123</span>
<span class="nb">puts</span><span class="w"> </span><span class="nv">$a</span>
<span class="nb">puts</span><span class="w"> </span><span class="s2">"b = $b"</span>
</pre></div>
<ul><li><b>Цикл</b> <i>пока</i> реализуется командой <i>while</i>, имеющей два аргумента. Первый — Tcl-выражение (выражения фактически пишутся на миниатюрном языке, состоящем из математических операций и операций сравнения в стиле языка <a href="/wiki/%D0%A1%D0%B8_(%D1%8F%D0%B7%D1%8B%D0%BA_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F)" title="Си (язык программирования)">C</a>). Второй аргумент — это скрипт, выполняемый при каждой <a href="/wiki/%D0%98%D1%82%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D1%8F_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Итерация (программирование)">итерации</a>. Его, как правило, помещают в фигурные скобки, чтобы избежать немедленного выполнения.</li></ul>
<p>Синтаксис:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">while</span><span class="w"> </span><span class="k">{</span><span class="w"> </span><span class="nv">Выражение_Tcl_С_Логическим_Значением</span><span class="w"> </span><span class="k">}</span><span class="w"> </span><span class="k">{</span>
<span class="w">    </span><span class="nv">TclКоманда</span>
<span class="w">    </span><span class="nv">другаяTclКоманда</span>
<span class="w">    </span><span class="err">…</span><span class="w"> </span><span class="err">…</span>
<span class="k">}</span>
</pre></div>
<p>Пример:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">while</span><span class="w"> </span><span class="k">{</span><span class="w"> </span><span class="nv">$x</span><span class="w"> </span><span class="o">&lt;</span><span class="w"> </span><span class="nv">10</span><span class="w"> </span><span class="k">}</span><span class="w"> </span><span class="k">{</span>
<span class="w">    </span><span class="nb">puts</span><span class="w"> </span><span class="nv">$x</span>
<span class="w">    </span><span class="nb">incr</span><span class="w"> </span>x
<span class="k">}</span>
</pre></div>
<ul><li><b>Условная</b> команда</li></ul>
<p>Синтаксис:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">if</span><span class="w"> </span><span class="k">{</span><span class="w"> </span><span class="nv">Выражение_Tcl_С_Логическим_Значением</span><span class="w"> </span><span class="k">}</span><span class="w"> </span><span class="k">{</span>
<span class="w">    </span><span class="nv">КомандаTcl</span>
<span class="k">}</span>
</pre></div>
<p>Пример:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">if</span><span class="w"> </span><span class="k">{</span><span class="nv">$x</span><span class="w"> </span><span class="o">&lt;</span><span class="w"> </span><span class="nv">0</span><span class="k">}</span><span class="w"> </span><span class="k">{</span>
<span class="w">    </span><span class="k">set</span><span class="w"> </span>x<span class="w"> </span><span class="mi">0</span>
<span class="k">}</span>
</pre></div>
<ul><li>Команда ветвления</li></ul>
<p>Пример:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">switch</span><span class="w"> </span><span class="o">-</span>glob<span class="w"> </span><span class="o">--</span><span class="w"> </span><span class="nv">$var</span><span class="w"> </span><span class="k">{</span>
<span class="w"> </span><span class="k">{</span><span class="nv">see</span><span class="k">}</span><span class="w"> </span><span class="nv">-</span>
<span class="w"> </span><span class="k">{</span><span class="nv">saw</span><span class="k">}</span><span class="w"> </span><span class="nv">-</span>
<span class="w"> </span><span class="k">{</span><span class="nv">seen</span><span class="k">}</span><span class="w"> </span><span class="k">{</span><span class="w"> </span><span class="nb">puts</span><span class="w"> </span><span class="s2">"Все это значит одно и то же!"</span><span class="k">}</span>
<span class="k">}</span>
</pre></div>
<ul><li>Команда может не иметь аргументов, например</li></ul>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="nb">pwd</span>
</pre></div>
<p>просто возвращает текущий рабочий каталог. А командой
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">set</span><span class="w"> </span>wdir<span class="w"> </span><span class="k">[</span><span class="nb">pwd</span><span class="k">]</span>
</pre></div>
<p>вы сохраните имя текущего каталога в переменной <i>wdir</i>.
</p>
<ul><li>Команда может возвращать список, например</li></ul>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="nb">glob</span><span class="w"> </span>Образец
</pre></div>
<p>возвратит список имён файлов в текущем каталоге, удовлетворяющих шаблону <i>Образец</i>.
</p>
<h3><span id=".D0.9F.D1.80.D0.BE.D1.86.D0.B5.D0.B4.D1.83.D1.80.D1.8B"></span><span class="mw-headline" id="Процедуры">Процедуры</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=7" title="Редактировать раздел «Процедуры»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=7" title="Редактировать код раздела «Процедуры»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Процедуры определяются следующим образом
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">proc</span><span class="w"> </span>имяПроцедуры<span class="w"> </span><span class="k">{</span><span class="w"> </span><span class="nv">списокАргументов</span><span class="w"> </span><span class="k">}</span><span class="w"> </span><span class="k">{</span>
<span class="w">  </span><span class="err">…</span><span class="w"> </span><span class="err">…</span>
<span class="k">}</span>
</pre></div>
<h3><span id=".D0.90.D1.81.D1.81.D0.BE.D1.86.D0.B8.D0.B0.D1.82.D0.B8.D0.B2.D0.BD.D1.8B.D0.B5_.D0.BC.D0.B0.D1.81.D1.81.D0.B8.D0.B2.D1.8B"></span><span class="mw-headline" id="Ассоциативные_массивы">Ассоциативные массивы</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=8" title="Редактировать раздел «Ассоциативные массивы»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=8" title="Редактировать код раздела «Ассоциативные массивы»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Следующий фрагмент кода создаст и инициализирует <a href="/wiki/%D0%90%D1%81%D1%81%D0%BE%D1%86%D0%B8%D0%B0%D1%82%D0%B8%D0%B2%D0%BD%D1%8B%D0%B9_%D0%BC%D0%B0%D1%81%D1%81%D0%B8%D0%B2" title="Ассоциативный массив">ассоциативный массив</a> (называемый в других языках также карта, словарь или хеш).
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">set</span><span class="w"> </span>capital<span class="k">(</span><span class="nv">France</span><span class="k">)</span><span class="w"> </span>Paris
<span class="k">set</span><span class="w"> </span>capital<span class="k">(</span><span class="nv">Italy</span><span class="k">)</span><span class="w"> </span>Rome
<span class="k">set</span><span class="w"> </span>capital<span class="k">(</span><span class="nv">Germany</span><span class="k">)</span><span class="w"> </span>Berlin
<span class="k">set</span><span class="w"> </span>capital<span class="k">(</span><span class="nv">Poland</span><span class="k">)</span><span class="w"> </span>Warsaw
<span class="k">set</span><span class="w"> </span>capital<span class="k">(</span><span class="nv">Russia</span><span class="k">)</span><span class="w"> </span>Moscow
<span class="k">set</span><span class="w"> </span>capital<span class="k">(</span><span class="nv">Spain</span><span class="k">)</span><span class="w"> </span>Madrid
</pre></div>
<p>Для запроса и вывода одного из значений ассоциативного массива используется
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="nb">puts</span><span class="w"> </span><span class="nv">$capital</span><span class="k">(</span><span class="nv">Italy</span><span class="k">)</span>
</pre></div>
<p>Список всех стран (ключей массива) можно получить командой
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">array</span><span class="w"> </span>names<span class="w"> </span>capital
</pre></div>
<p>Результат не будет отсортирован, потому что массивы Tcl базируются на <a href="/wiki/%D0%A5%D0%B5%D1%88-%D1%82%D0%B0%D0%B1%D0%BB%D0%B8%D1%86%D0%B0" title="Хеш-таблица">хеш-таблицах</a>.
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="nv">Poland</span><span class="w"> </span>Spain<span class="w"> </span>Russia<span class="w"> </span>Germany<span class="w"> </span>Italy<span class="w"> </span>France
</pre></div>
<p>Сортировку можно выполнить командой
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="nb">lsort</span><span class="w"> </span><span class="k">[array</span><span class="w"> </span>names<span class="w"> </span>capital<span class="k">]</span>
</pre></div>
<p>Чтобы сослаться на массив (передать его как ссылку), используется имя содержащей его переменной:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">proc</span><span class="w"> </span>demo<span class="w"> </span>arrName<span class="w"> </span><span class="k">{</span>
<span class="w">    </span><span class="k">upvar</span><span class="w"> </span><span class="mi">1</span><span class="w"> </span><span class="nv">$arrName</span><span class="w"> </span>arr<span class="w"> </span><span class="k">;</span><span class="c"># tie the reference to a local name</span>
<span class="w">    </span><span class="c">#do something with arr</span>
<span class="k">}</span>
</pre></div>
<p>Для доступа к массиву по значению используется <i>array get</i> или <i>array set</i>. Например, чтобы скопировать один массив в другой:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">array</span><span class="w"> </span>set<span class="w"> </span>Newarr<span class="w"> </span><span class="k">[array</span><span class="w"> </span>get<span class="w"> </span>Oldarr<span class="k">]</span>
</pre></div>
<p>В Tcl 8.5 введены массивы, передаваемые по значению — словари (<i>dict</i>).
</p><p>Для преобразования скалярной переменной в массив и наоборот следует удалить существующую переменную при помощи команды <i>unset</i>.
</p>
<h3><span id=".D0.9E.D0.B1.D0.BB.D0.B0.D1.81.D1.82.D1.8C_.D0.B2.D0.B8.D0.B4.D0.B8.D0.BC.D0.BE.D1.81.D1.82.D0.B8_.D0.BF.D0.B5.D1.80.D0.B5.D0.BC.D0.B5.D0.BD.D0.BD.D1.8B.D1.85"></span><span class="mw-headline" id="Область_видимости_переменных">Область видимости переменных</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=9" title="Редактировать раздел «Область видимости переменных»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=9" title="Редактировать код раздела «Область видимости переменных»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>По умолчанию внутри процедуры видны только <i>определённые в ней</i> переменные. За пределами определения процедуры её переменные недоступны иначе, чем через механизм передачи параметров. Можно иметь переменные в процедуре с таким же именем, как и в основной программе, при этом значения этих переменных будут различаться, по сути это будут разные переменные. Чтобы изменить это поведение, используются команды <i>global</i> или <i>upvar</i>.
</p>

<h4><span id=".D0.A1.D0.BF.D0.B8.D1.81.D0.BE.D0.BA_.D1.81.D0.BE.D0.B4.D0.B5.D1.80.D0.B6.D0.B8.D0.BC.D0.BE.D0.B3.D0.BE_.D0.BC.D0.B0.D1.81.D1.81.D0.B8.D0.B2.D0.B0"></span><span class="mw-headline" id="Список_содержимого_массива">Список содержимого массива</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=15" title="Редактировать раздел «Список содержимого массива»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=15" title="Редактировать код раздела «Список содержимого массива»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h4>
<p>В массиве <i>tcl_platform</i> содержится информация о свойствах платформы, на которой запущен Tcl.
Список имён свойств может быть получен с помощью

<p>Следующий фрагмент выводит их вместе со значениями:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">foreach</span><span class="w"> </span>i<span class="w"> </span><span class="k">[array</span><span class="w"> </span>names<span class="w"> </span>tcl_platform<span class="k">]</span><span class="w"> </span><span class="k">{</span>

<span class="k">}</span>
</pre></div>
<p>Если свойства должны быть отсортированы
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">foreach</span><span class="w"> </span>i<span class="w"> </span><span class="k">[</span><span class="nb">lsort</span><span class="w"> </span><span class="k">[array</span><span class="w"> </span>names<span class="w"> </span>tcl_platform<span class="k">]</span><span class="w"> </span><span class="k">]</span><span class="w"> </span><span class="k">{</span>
<span class="w">    </span><span class="nb">puts</span><span class="w"> </span><span class="s2">"$i = $tcl_platform($i)"</span>
<span class="k">}</span>
</pre></div>
<p>Здесь демонстрируется вложенность команд. Они могут быть вложены на любую глубину. Подобного же результата можно достичь с помощью команды <i>parray</i><sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[16]</a></sup>, появившейся в Tcl 8.5.
</p>
<h3><span id=".D0.9F.D0.B5.D1.80.D0.B5.D1.81.D0.B5.D1.87.D0.B5.D0.BD.D0.B8.D0.B5_.D0.BC.D0.BD.D0.BE.D0.B6.D0.B5.D1.81.D1.82.D0.B2"></span><span class="mw-headline" id="Пересечение_множеств">Пересечение множеств</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=16" title="Редактировать раздел «Пересечение множеств»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=16" title="Редактировать код раздела «Пересечение множеств»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Процедура <i>filter</i> возвращает те элементы списка, для которых <i>script</i> принимает значение TRUE:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">proc</span><span class="w"> </span>filter<span class="w"> </span><span class="k">{</span><span class="nb">list</span><span class="w"> </span>script<span class="k">}</span><span class="w"> </span><span class="k">{</span>

<span class="w">    </span><span class="k">foreach</span><span class="w"> </span>e<span class="w"> </span><span class="nv">$list</span><span class="w"> </span><span class="k">{if</span><span class="w"> </span><span class="k">{[uplevel</span><span class="w"> </span><span class="mi">1</span><span class="w"> </span><span class="nv">$script</span><span class="w"> </span><span class="nv">$e</span><span class="k">]}</span><span class="w"> </span><span class="k">{</span><span class="nb">lappend</span><span class="w"> </span>res<span class="w"> </span><span class="nv">$e</span><span class="k">}}</span>

</p><p>Процедура <i>in</i> — это сокращённая запись для включения множеств:
</p>
<div class="mw-highlight mw-highlight-lang-tcl mw-content-ltr" dir="ltr"><pre><span></span><span class="k">proc</span><span class="w"> </span><span class="ow">in</span><span class="w"> </span><span class="k">{</span><span class="nb">list</span><span class="w"> </span>e<span class="k">}</span><span class="w"> </span><span class="k">{</span>

</p>

<span class="nv">b</span><span class="w"> </span>c
</pre></div>
<h3><span id=".D0.A4.D0.B0.D0.BA.D1.82.D0.BE.D1.80.D0.B8.D0.B0.D0.BB"></span><span class="mw-headline" id="Факториал">Факториал</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=17" title="Редактировать раздел «Факториал»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=17" title="Редактировать код раздела «Факториал»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>

<h2><span id=".D0.A0.D0.B5.D0.B0.D0.BB.D0.B8.D0.B7.D0.B0.D1.86.D0.B8.D1.8F.2C_.D0.B4.D0.B8.D1.81.D1.82.D1.80.D0.B8.D0.B1.D1.83.D1.82.D0.B8.D0.B2.D1.8B.2C_.D0.B8.D0.BD.D1.81.D1.82.D1.80.D1.83.D0.BC.D0.B5.D0.BD.D1.82.D1.8B"></span><span class="mw-headline" id="Реализация,_дистрибутивы,_инструменты">Реализация, дистрибутивы, инструменты</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=18" title="Редактировать раздел «Реализация, дистрибутивы, инструменты»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=18" title="Редактировать код раздела «Реализация, дистрибутивы, инструменты»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Tcl — <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D1%80%D0%BF%D1%80%D0%B5%D1%82%D0%B0%D1%82%D0%BE%D1%80" title="Интерпретатор">интерпретируемый</a> язык, то есть <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80%D0%BD%D0%B0%D1%8F_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B0" title="Компьютерная программа">программы</a> на Tcl готовы к выполнению без <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D0%B8%D0%BB%D1%8F%D1%82%D0%BE%D1%80" title="Компилятор">компиляции</a> и <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D0%BE%D0%BD%D0%BE%D0%B2%D1%89%D0%B8%D0%BA" title="Компоновщик">компоновки</a>.
</p><p>Интерпретатор Tcl <a href="/wiki/%D0%9F%D0%BE%D1%80%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B3%D0%BE_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D1%8F" title="Портирование программного обеспечения">портирован</a> на большинство распространённых платформ<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[17]</a></sup>. Он распространяется под <a class="mw-redirect" href="/wiki/%D0%A1%D0%B2%D0%BE%D0%B1%D0%BE%D0%B4%D0%BD%D1%8B%D0%B9_%D1%81%D0%BE%D1%84%D1%82" title="Свободный софт">свободной</a> <a href="/wiki/%D0%9A%D0%BE%D0%BF%D0%B8%D0%BB%D0%B5%D1%84%D1%82" title="Копилефт">некопилефтной</a> <a class="external text" href="http://www.tcl.tk/software/tcltk/license.html" rel="nofollow">лицензией</a>, позволяющей использовать его без ограничения в разработке <a class="mw-redirect" href="/wiki/%D0%9F%D1%80%D0%BE%D0%BF%D1%80%D0%B8%D0%B5%D1%82%D0%B0%D1%80%D0%BD%D0%BE%D0%B5_%D0%9F%D0%9E" title="Проприетарное ПО">проприетарных приложений</a>, а также создавать проприетарные производные системы. Разработчики относят её к <a href="/wiki/%D0%9B%D0%B8%D1%86%D0%B5%D0%BD%D0%B7%D0%B8%D1%8F_BSD" title="Лицензия BSD">BSD-подобным</a>, хотя, по сравнению с лицензией BSD, лицензия Tcl накладывает меньшее количество запретов.
</p>
<h3><span id=".D0.A0.D0.B5.D0.B0.D0.BB.D0.B8.D0.B7.D0.B0.D1.86.D0.B8.D1.8F"></span><span class="mw-headline" id="Реализация">Реализация</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=19" title="Редактировать раздел «Реализация»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=19" title="Редактировать код раздела «Реализация»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>В настоящее время работа над ядром языка Tcl ведётся группой разработчиков<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[18]</a></sup>, именуемой Tcl Core Team<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[19]</a></sup>. Самые свежие версии Tcl-транслятора и основных библиотек доступны в <a href="/wiki/%D0%A0%D0%B5%D0%BF%D0%BE%D0%B7%D0%B8%D1%82%D0%BE%D1%80%D0%B8%D0%B9" title="Репозиторий">репозитории</a> на <a href="/wiki/SourceForge.net" title="SourceForge.net">SourceForge.net</a>, см. список sourceforge-проектов, включаемых в основную поставку Tcl/Tk<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[20]</a></sup>.
</p><p>Транслятор Tcl компилирует программу в промежуточное представление — <a class="mw-redirect" href="/wiki/%D0%91%D0%B0%D0%B9%D1%82%D0%BA%D0%BE%D0%B4" title="Байткод">байткод</a>, а затем интерпретирует её. В отличие от, например, <a href="/wiki/Python" title="Python">Python</a>, это промежуточное представление не сохраняется на диск, программа может храниться только в исходных текстах, возможно, упакованных в <a href="/wiki/MetaKit#StarKit" title="MetaKit">StarKit</a>-модуль или с помощью какого-нибудь другого <a class="mw-disambig" href="/wiki/%D0%92%D1%80%D0%B0%D0%BF%D0%BF%D0%B5%D1%80" title="Враппер">враппера</a>. В зависимости от параметров компиляции интерпретатор Tcl может поддерживать или не поддерживать <a href="/wiki/%D0%9C%D0%BD%D0%BE%D0%B3%D0%BE%D0%BF%D0%BE%D1%82%D0%BE%D1%87%D0%BD%D0%BE%D1%81%D1%82%D1%8C" title="Многопоточность">многопоточность</a>.
</p><p>Хотя в большинстве учебников по Tcl для начинающих говорится что-то вроде «в Tcl есть всего один тип переменных — строка», на самом деле трансляторы Tcl уже давно как устроены по-другому. Внутри Tcl использует систему динамических типов с автоматическим прозрачным преобразованием к строковому виду.
</p><p>Tcl входит практически во все дистрибутивы <a href="/wiki/Linux" title="Linux">Linux</a>. Для <a class="mw-redirect" href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> требуется отдельный установочный пакет.
</p><p>Версия Tcl 8.4 отличается от предыдущей новыми функциями (оптимизация 64-битного кода, VFS, дополнения в плане <a href="/wiki/%D0%98%D0%BD%D1%82%D1%80%D0%BE%D1%81%D0%BF%D0%B5%D0%BA%D1%86%D0%B8%D1%8F_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Интроспекция (программирование)">интроспекции</a>) и некоторым повышением быстродействия за счёт оптимизации работы с байткодом.
</p><p>Новые возможности, добавленные в версии 8.1 (в частности, поддержка многопотокового выполнения, средства интернационализации и работа с символами Unicode), замедлили работу Tcl примерно на 19 % по сравнению с давшей скачок скорости восьмой версией. Разработчики 8.4 приложили все усилия для того, чтобы максимально повысить скорость выполнения программ, добиться, чтобы они работали не только не медленнее, но даже быстрее, чем это было при появлении версии 8.0. Эта цель была достигнута практически для всех средств Tcl и Tk — версия 8.4 в среднем в 8 раз быстрее, чем версия 7.6<sup class="reference" id="cite_ref-bench_15-1"><a href="#cite_note-bench-15">[14]</a></sup>.
</p><p>Текущая версия TCL — 8.6.
</p>
<h4><span id=".D0.90.D0.BB.D1.8C.D1.82.D0.B5.D1.80.D0.BD.D0.B0.D1.82.D0.B8.D0.B2.D0.BD.D1.8B.D0.B5_.D1.80.D0.B5.D0.B0.D0.BB.D0.B8.D0.B7.D0.B0.D1.86.D0.B8.D0.B8_.D0.B8_.D0.B4.D0.B8.D0.B0.D0.BB.D0.B5.D0.BA.D1.82.D1.8B"></span><span class="mw-headline" id="Альтернативные_реализации_и_диалекты">Альтернативные реализации и диалекты</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=20" title="Редактировать раздел «Альтернативные реализации и диалекты»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=20" title="Редактировать код раздела «Альтернативные реализации и диалекты»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Существуют альтернативные реализации Tcl для популярных <a href="/wiki/%D0%92%D0%B8%D1%80%D1%82%D1%83%D0%B0%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D0%BC%D0%B0%D1%88%D0%B8%D0%BD%D0%B0" title="Виртуальная машина">виртуальных машин</a> общего назначения. Это написанный на <a href="/wiki/Java" title="Java">Java</a> интерпретатор — Jacl<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[21]</a></sup> (последняя версия от 2008 года), и проект Eagle<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[22]</a></sup>, реализующий большинство команд Tcl 8.4 для <a class="mw-redirect" href="/wiki/CLR" title="CLR">CLR</a> (доступны сборки для <a href="/wiki/.NET_Framework" title=".NET Framework">.NET</a> 4.0 и 2.0). Ведутся эксперименты по трансляции Tcl-программ в код виртуальной машины <a href="/wiki/Parrot" title="Parrot">Parrot</a>, разрабатываемой для шестой версии <a href="/wiki/Perl" title="Perl">Perl</a><sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[23]</a></sup>.
</p><p>Для программирования <a href="/wiki/%D0%92%D1%81%D1%82%D1%80%D0%B0%D0%B8%D0%B2%D0%B0%D0%B5%D0%BC%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Встраиваемая система">встраиваемых систем</a> и встраивания в приложения разработан специальный диалект Tcl — Jim<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[24]</a></sup><sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[25]</a></sup>. Это интерпретатор, реализующий большое подмножество Tcl, расширенное дополнительными механизмами <a href="/wiki/%D0%9E%D0%B1%D1%8A%D0%B5%D0%BA%D1%82%D0%BD%D0%BE-%D0%BE%D1%80%D0%B8%D0%B5%D0%BD%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Объектно-ориентированное программирование">объектно-ориентированного</a> и <a href="/wiki/%D0%A4%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%BE%D0%BD%D0%B0%D0%BB%D1%8C%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Функциональное программирование">функционального</a> программирования и занимающий в скомпилированном виде менее 100 килобайт.
</p><p>Ещё один диалект Tcl — скриптовый язык мобильных устройств Hecl<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[26]</a></sup>, разработанный Дэвидом Велтоном и реализованный на <a href="/wiki/Java" title="Java">Java</a> (<a class="mw-redirect" href="/wiki/J2ME" title="J2ME">J2ME</a>)<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[27]</a></sup><sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[28]</a></sup>. Что же касается платформы <a href="/wiki/Android" title="Android">Android</a>, поддержку Tcl на ней (наряду с другими скриптовыми языками) обеспечивает: скриптовая прослойка <a href="/wiki/SL4A" title="SL4A">SL4A</a><sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[29]</a></sup>; существует также нативная реализация Tcl для этой ОС, называемая AndroWish<sup class="reference" id="cite_ref-androwish_31-0"><a href="#cite_note-androwish-31">[30]</a></sup>.
</p>
<h3><span id=".D0.94.D0.B8.D1.81.D1.82.D1.80.D0.B8.D0.B1.D1.83.D1.82.D0.B8.D0.B2.D1.8B"></span><span class="mw-headline" id="Дистрибутивы">Дистрибутивы</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=21" title="Редактировать раздел «Дистрибутивы»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=21" title="Редактировать код раздела «Дистрибутивы»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Все нижеперечисленные дистрибутивы распространяются свободно и собраны из компонентов, имеющих ту или иную <a href="/wiki/%D0%A1%D0%B2%D0%BE%D0%B1%D0%BE%D0%B4%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Свободное программное обеспечение">свободную лицензию</a>. Большинство из них — <a class="mw-redirect" href="/wiki/%D0%9A%D1%80%D0%BE%D1%81%D1%81%D0%BF%D0%BB%D0%B0%D1%82%D1%84%D0%BE%D1%80%D0%BC%D0%B5%D0%BD%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Кроссплатформенное программное обеспечение">кроссплатформенно</a> (кроме <i>WinTclTk</i> и <i>Tcl/Tk Aqua</i>), хотя в <a href="/wiki/Unix" title="Unix">Unix</a>-системах Tcl/Tk обычно является частью дистрибутива <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Операционная система">ОС</a>.
</p>
<ul><li><b>ActiveTcl</b><sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[31]</a></sup>
<dl><dd>Самый распространённый дистрибутив Tcl. Подобно другим продуктам компании <a href="/wiki/ActiveState" title="ActiveState">ActiveState</a> включает в себя большой набор библиотек и расширений. В Windows-версиях включает в себя средства интеграции с <a class="mw-redirect" href="/wiki/WSH" title="WSH">WSH</a>. Какие-либо <a href="/wiki/%D0%98%D0%BD%D1%81%D1%82%D1%80%D1%83%D0%BC%D0%B5%D0%BD%D1%82%D0%B0%D0%BB%D1%8C%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Инструментальное программное обеспечение">инструментальные средства</a>, облегчающие жизнь программиста, отсутствуют. Фирма продаёт пакет TclDevKit, содержащий средства отладки и создания автономных программ (последнее — на базе технологии <a href="/wiki/MetaKit#StarKit" title="MetaKit">StarKit</a>) и <a class="mw-redirect" href="/wiki/Komodo" title="Komodo">Komodo</a> — универсальную <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D0%B3%D1%80%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D1%80%D0%B5%D0%B4%D0%B0_%D1%80%D0%B0%D0%B7%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D0%BA%D0%B8" title="Интегрированная среда разработки">IDE</a> для скриптовых языков, включающее средства визуального программирования <a class="mw-disambig" href="/wiki/Tk" title="Tk">Tk</a>-форм.</dd></dl></li>
<li><b>WinTclTk</b><sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[32]</a></sup>
<dl><dd>Более компактный, чем ActiveTcl, дистрибутив, ориентированный на работу с <a href="/wiki/XOTcl" title="XOTcl">XOTcl</a> — объектно-ориентированным расширением Tcl. Список включённых в него библиотек несколько уже, чем в ActiveTcl, но базовый набор здесь присутствует. В этот дистрибутив входят Tloona IDE<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[33]</a></sup> — <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D0%B3%D1%80%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D1%80%D0%B5%D0%B4%D0%B0_%D1%80%D0%B0%D0%B7%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D0%BA%D0%B8" title="Интегрированная среда разработки">среда разработки</a> традиционного типа и XOTclIde<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[34]</a></sup> — среда объектно-ориентированного программирования в стиле <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a>, обе — с отладчиками. Также на сайте разработчиков свободно доступен пакет для создания единого запускаемого файла TkWrapper, основанный на FreeWrap<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[35]</a></sup> — технологии, альтернативной <a href="/wiki/MetaKit#StarKit" title="MetaKit">StarKit</a>, основанной на простом <a href="/wiki/ZIP" title="ZIP">ZIP</a>-сжатии. Последняя на сегодняшний день версия — 8.5.6 (3 декабря 2009, Tcl 8.5.6).</dd></dl></li>
<li><b>TclKit</b><sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[36]</a></sup>
<dl><dd>Это Tcl/Tk c базовым набором библиотек в одном компактном выполняемом файле. Дистрибутив создан с использованием технологии <a href="/wiki/MetaKit#StarKit" title="MetaKit">StarKit</a>. Содержит библиотеку <a href="/wiki/MetaKit" title="MetaKit">MetaKit</a>, позволяющую управлять содержанием этого файла. Ещё один популярный дистрибутив Tcl в одном файле — это <b>eTcl</b><sup class="reference" id="cite_ref-38"><a href="#cite_note-38">[37]</a></sup>. На аналогичных принципах построен дистрибутив <b>dqkit</b><sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[38]</a></sup> — альтернативная сборка, известная как «TclKit на стероидах», имеется несколько вариантов, отличающихся составом библиотек, поддержкой многопоточности и т. д.</dd></dl></li>
<li><b>MyTcl</b><sup class="reference" id="cite_ref-40"><a href="#cite_note-40">[39]</a></sup>
<dl><dd>Полноценная <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D0%B3%D1%80%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D1%80%D0%B5%D0%B4%D0%B0_%D1%80%D0%B0%D0%B7%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D0%BA%D0%B8" title="Интегрированная среда разработки">интегрированная среда разработки</a> (IDE) на Tcl, включающая в себя дистрибутив языковой среды. Имеет возможность генерации автономных приложений в <a href="/wiki/MetaKit" title="MetaKit">StarKit</a>-формате. Текущая версия — 1.0 Build 8 (18 июня 2012) для Win32 и Win64 с интегрированным в систему Tcl 8.5.11. В настоящее время <b>не</b> является <a href="/wiki/%D0%9E%D1%82%D0%BA%D1%80%D1%8B%D1%82%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Открытое программное обеспечение">открытым ПО</a>.</dd></dl></li>
<li><b>AndroWish</b><sup class="reference" id="cite_ref-androwish_31-1"><a href="#cite_note-androwish-31">[30]</a></sup>
<dl><dd>Среда для запуска Tcl/Tk-скриптов и компоновки tcl/tk-приложения в apk-файл для исполнения под <a href="/wiki/Android" title="Android">Android</a></dd></dl></li></ul>
<h2><span id=".D0.9F.D1.80.D0.B8.D0.BC.D0.B5.D0.BD.D0.B5.D0.BD.D0.B8.D0.B5"></span><span class="mw-headline" id="Применение">Применение</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=22" title="Редактировать раздел «Применение»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=22" title="Редактировать код раздела «Применение»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Tcl, наряду с <a href="/wiki/Perl" title="Perl">Perl</a> и <a href="/wiki/Python" title="Python">Python</a>, стал одним из трёх классических скриптовых языков общего назначения. Эта троица фигурирует не только в качестве списка свободных дистрибутивов, собираемых в <a href="/wiki/ActiveState" title="ActiveState">ActiveState</a>, но и, например, как языки, на которых (помимо диалекта <a href="/wiki/PL/pgSQL" title="PL/pgSQL">PL/pgSQL</a>) можно писать <a href="/wiki/%D0%A2%D1%80%D0%B8%D0%B3%D0%B3%D0%B5%D1%80_(%D0%B1%D0%B0%D0%B7%D1%8B_%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85)" title="Триггер (базы данных)">триггеры</a> и <a href="/wiki/%D0%A5%D1%80%D0%B0%D0%BD%D0%B8%D0%BC%D0%B0%D1%8F_%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D0%B4%D1%83%D1%80%D0%B0" title="Хранимая процедура">хранимые процедуры</a> популярного сервера БД <a href="/wiki/PostgreSQL" title="PostgreSQL">PostgreSQL</a>.
</p><p>Tcl как встраиваемый язык нашёл применение и в сфере <a class="mw-redirect" href="/wiki/%D0%A1%D0%90%D0%9F%D0%A0" title="САПР">САПР</a> (<a class="mw-redirect" href="/wiki/%D0%A1%D0%90%D0%9F%D0%A0" title="САПР">CAD</a>|<a href="/wiki/CAM" title="CAM">CAM</a>|<a href="/wiki/Computer-aided_engineering" title="Computer-aided engineering">CAE</a>)<sup class="reference" id="cite_ref-41"><a href="#cite_note-41">[40]</a></sup><sup class="reference" id="cite_ref-42"><a href="#cite_note-42">[41]</a></sup>. Он, к примеру, используется как средство настройки баз данных, в постпроцессоре <a class="mw-redirect" href="/wiki/Unigraphics" title="Unigraphics">Unigraphics</a>. Кроме того, он де-факто является языком автоматизации и интеграции во всех ведущих программных пакетах разработки микросхем (<a href="/wiki/%D0%9F%D0%9B%D0%98%D0%A1" title="ПЛИС">ПЛИС</a> и <a class="mw-redirect" href="/wiki/ASIC" title="ASIC">ASIC</a>).
</p>
<h3><span id=".D0.98.D1.81.D0.BF.D0.BE.D0.BB.D1.8C.D0.B7.D0.BE.D0.B2.D0.B0.D0.BD.D0.B8.D0.B5_.D0.B2_.D0.B2.D0.B5.D0.B1-.D1.80.D0.B0.D0.B7.D1.80.D0.B0.D0.B1.D0.BE.D1.82.D0.BA.D0.B5"></span><span class="mw-headline" id="Использование_в_веб-разработке">Использование в веб-разработке</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=23" title="Редактировать раздел «Использование в веб-разработке»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=23" title="Редактировать код раздела «Использование в веб-разработке»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>В web-программировании Tcl обычно используется вместе с веб-сервером <a href="/wiki/AOLServer" title="AOLServer">AOLServer</a> или одним из «облегчённых» веб-серверов, вроде <a href="/wiki/Lighttpd" title="Lighttpd">lighttpd</a>, ориентированных на <a href="/wiki/FastCGI" title="FastCGI">FastCGI</a>. Для интеграции Tcl с <a href="/wiki/Apache_HTTP_Server" title="Apache HTTP Server">Apache</a> существует модуль Rivet<sup class="reference" id="cite_ref-43"><a href="#cite_note-43">[42]</a></sup>.
</p><p>Под управлением AOLServer работает, в частности, <a href="/wiki/OpenACS" title="OpenACS">OpenACS</a><sup class="reference" id="cite_ref-44"><a href="#cite_note-44">[43]</a></sup> — <a href="/wiki/%D0%A1%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0_%D1%83%D0%BF%D1%80%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F_%D1%81%D0%BE%D0%B4%D0%B5%D1%80%D0%B6%D0%B8%D0%BC%D1%8B%D0%BC" title="Система управления содержимым">система управления содержимым</a> веб-сайтов с богатыми возможностями расширения на языке Tcl, расширенном <a class="mw-redirect" href="/wiki/XOtcl" title="XOtcl">XOtcl</a> или project-open<sup class="reference" id="cite_ref-45"><a href="#cite_note-45">[44]</a></sup> — <a href="/wiki/%D0%A1%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0_%D1%83%D0%BF%D1%80%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F_%D0%B4%D0%BE%D0%BA%D1%83%D0%BC%D0%B5%D0%BD%D1%82%D0%B0%D0%BC%D0%B8" title="Система управления документами">система групповой работы</a> c <a href="/wiki/%D0%92%D0%B5%D0%B1-%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81" title="Веб-интерфейс">веб-интерфейсом</a>.
</p><p>Существуют также такие проекты, как TclHttpd<sup class="reference" id="cite_ref-46"><a href="#cite_note-46">[45]</a></sup> и его преемник — Wub<sup class="reference" id="cite_ref-47"><a href="#cite_note-47">[46]</a></sup>, являющиеся полноценными Web-<a href="/wiki/%D0%A4%D1%80%D0%B5%D0%B9%D0%BC%D0%B2%D0%BE%D1%80%D0%BA" title="Фреймворк">фреймворками</a>. На базе нового объектного расширения языка TclOO разрабатывается веб-фреймвок Woof (Web Oriented Object Framework)<sup class="reference" id="cite_ref-48"><a href="#cite_note-48">[47]</a></sup>. Примером CMS для TclHttpd является Ucome<sup class="reference" id="cite_ref-49"><a href="#cite_note-49">[48]</a></sup>.
</p><p>Помимо использования в качестве языка реализации веб-приложений, Tcl может использоваться как скриптовый язык для расширения существующих приложений. К примеру, на нём пишут модули для IRC-бота <a href="/wiki/Eggdrop" title="Eggdrop">Eggdrop</a><sup class="reference" id="cite_ref-50"><a href="#cite_note-50">[49]</a></sup><sup class="reference" id="cite_ref-51"><a href="#cite_note-51">[50]</a></sup>.
</p>
<h2><span id=".D0.94.D0.BE.D1.81.D1.82.D0.BE.D0.B8.D0.BD.D1.81.D1.82.D0.B2.D0.B0_.D0.B8_.D0.BD.D0.B5.D0.B4.D0.BE.D1.81.D1.82.D0.B0.D1.82.D0.BA.D0.B8"></span><span class="mw-headline" id="Достоинства_и_недостатки">Достоинства и недостатки</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=24" title="Редактировать раздел «Достоинства и недостатки»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=24" title="Редактировать код раздела «Достоинства и недостатки»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Недостатком ранних версий Tcl/Tk, несомненно, было низкое быстродействие. Версии ниже 8-й использовали прямую интерпретацию исходного текста, однако использование <a class="mw-redirect" href="/wiki/%D0%91%D0%B0%D0%B9%D1%82%D0%BA%D0%BE%D0%B4" title="Байткод">байткода</a> разрешило эту проблему. Сейчас по быстродействию Tcl не сильно отличается от <a href="/wiki/PHP" title="PHP">PHP</a>, хотя заметно уступает таким скриптовым языкам как, например, <a href="/wiki/Python" title="Python">Python</a> или <a href="/wiki/Lua" title="Lua">Lua</a>. Сложные или <a class="mw-redirect" href="/wiki/%D0%9A%D1%80%D0%B8%D1%82%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D1%83%D1%87%D0%B0%D1%81%D1%82%D0%BE%D0%BA_%D0%BA%D0%BE%D0%B4%D0%B0" title="Критический участок кода">критические по времени выполнения участки кода</a> рекомендуется выполнять в виде внешних модулей на <a href="/wiki/%D0%A1%D0%B8_(%D1%8F%D0%B7%D1%8B%D0%BA_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F)" title="Си (язык программирования)">C</a>/<a href="/wiki/C%2B%2B" title="C++">C++</a>.
</p><p>Tcl/Tk отличает непривычность синтаксиса для программистов на «традиционных» языках. Он ближе к стандартному синтаксису <a class="mw-redirect" href="/wiki/%D0%9A%D0%BE%D0%BC%D0%B0%D0%BD%D0%B4%D0%BD%D0%B0%D1%8F_%D0%BE%D0%B1%D0%BE%D0%BB%D0%BE%D1%87%D0%BA%D0%B0_UNIX" title="Командная оболочка UNIX">оболочки Unix</a>. Этот синтаксис может быть произвольно изменён (как и в других «программируемых языках программирования», таких как <a class="mw-redirect" href="/wiki/Forth" title="Forth">Forth</a><sup class="reference" id="cite_ref-52"><a href="#cite_note-52">[примечание 2]</a></sup> или <a href="/wiki/Common_Lisp" title="Common Lisp">Common Lisp</a>). Это может быть преимуществом для программиста-индивидуала или небольшой группы программистов, но делает Tcl/Tk вряд ли пригодным для использования в качестве языка общего назначения при разработке в промышленном масштабе. Хотя наличие стандартизированных расширений снижает остроту этой проблемы. Tcl может использоваться и в крупных проектах в пределах своей «экологической ниши» (встроенные скрипты, «склеивание» компонент, WEB-программирование).
</p><p>Хотя Tcl обладает всеми свойствами <a href="/wiki/%D0%A4%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%BE%D0%BD%D0%B0%D0%BB%D1%8C%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Функциональное программирование">функционального языка</a>, его интерпретатор до версии 8.6 не производил оптимизации <a href="/wiki/%D0%A5%D0%B2%D0%BE%D1%81%D1%82%D0%BE%D0%B2%D0%B0%D1%8F_%D1%80%D0%B5%D0%BA%D1%83%D1%80%D1%81%D0%B8%D1%8F" title="Хвостовая рекурсия">хвостовой рекурсии</a>, что затрудняло использование чисто функционального стиля, уменьшало его эффективность. Впрочем, гибкость языка позволяет обойти эту проблему, реализовав подобие такой оптимизации на высоком уровне<sup class="reference" id="cite_ref-53"><a href="#cite_note-53">[51]</a></sup>. Наконец, появившаяся в тестовой версии 8.6 команда <i>tailcall</i> позволяет без особых проблем использовать хвостовую рекурсию.
</p><p>Наиболее тесная интеграция Tcl с <a href="/wiki/%D0%93%D1%80%D0%B0%D1%84%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81_%D0%BF%D0%BE%D0%BB%D1%8C%D0%B7%D0%BE%D0%B2%D0%B0%D1%82%D0%B5%D0%BB%D1%8F" title="Графический интерфейс пользователя">графической</a> <a href="/wiki/%D0%91%D0%B8%D0%B1%D0%BB%D0%B8%D0%BE%D1%82%D0%B5%D0%BA%D0%B0_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Библиотека (программирование)">библиотекой</a> <a class="mw-disambig" href="/wiki/Tk" title="Tk">Tk</a>, которая зачастую используется и в сочетании с другими языками программирования — несомненное преимущество Tcl. Для создания <a class="mw-redirect" href="/wiki/GUI" title="GUI">GUI</a>-оболочек к <a class="mw-redirect" href="/wiki/%D0%9A%D0%BE%D0%BD%D1%81%D0%BE%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B0" title="Консольная программа">консольным программам</a> или пакетам консольных программ Tcl может оказаться наилучшим выбором.
</p>
<h3><span id=".D0.92.D0.BB.D0.B8.D1.8F.D0.BD.D0.B8.D0.B5"></span><span class="mw-headline" id="Влияние">Влияние</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Tcl&amp;veaction=edit&amp;section=25" title="Редактировать раздел «Влияние»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=25" title="Редактировать код раздела «Влияние»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Хотя Tcl и не относится к числу широко используемых языков программирования, он, наряду с такими (более ранними) языками, как <a href="/wiki/%D0%9B%D0%B8%D1%81%D0%BF" title="Лисп">Lisp</a> и <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a>, оказал заметное влияние на программирование. Он стал первым языком, ориентированным на простоту встраивания в приложения<sup class="reference" id="cite_ref-:0_54-0"><a href="#cite_note-:0-54">[52]</a></sup>.
</p><p>Не было аналогов по простоте использования и у библиотеки графических <a class="mw-redirect" href="/wiki/%D0%92%D0%B8%D0%B4%D0%B6%D0%B5%D1%82" title="Виджет">виджетов</a> <a class="mw-disambig" href="/wiki/Tk" title="Tk">Tk</a>, которая, помимо Tcl, стала популярной и в других универсальных <a class="mw-redirect" href="/wiki/%D0%A1%D0%BA%D1%80%D0%B8%D0%BF%D1%82%D0%BE%D0%B2%D1%8B%D0%B5_%D1%8F%D0%B7%D1%8B%D0%BA%D0%B8" title="Скриптовые языки">скриптовых языках</a>. <a href="/wiki/Tkinter" title="Tkinter">Tkinter</a> остаётся базовым средством создания <a class="mw-redirect" href="/wiki/GUI" title="GUI">GUI</a>-приложений в <a href="/wiki/Python" title="Python">Python</a>'е. Имеются также библиотеки привязки Tk к <a href="/wiki/Perl" title="Perl">Perl</a>'у<sup class="reference" id="cite_ref-55"><a href="#cite_note-55">[53]</a></sup> (Tkx) и <a href="/wiki/Ruby" title="Ruby">Ruby</a><sup class="reference" id="cite_ref-:0_54-1"><a href="#cite_note-:0-54">[52]</a></sup>.
</p><p><a href="/wiki/SQLite" title="SQLite">SQLite</a> — встраиваемая легковесная <a href="/wiki/SQL" title="SQL">SQL</a> <a href="/wiki/%D0%A1%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0_%D1%83%D0%BF%D1%80%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F_%D0%B1%D0%B0%D0%B7%D0%B0%D0%BC%D0%B8_%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85" title="Система управления базами данных">система управления базами данных</a> — изначально разрабатывалась как расширение Tcl<sup class="reference" id="cite_ref-:0_54-2"><a href="#cite_note-:0-54">[52]</a></sup>.
</p>
