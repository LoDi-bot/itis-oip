<p><b>MetaPost</b> — <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D1%80%D0%BF%D1%80%D0%B5%D1%82%D0%B0%D1%82%D0%BE%D1%80" title="Интерпретатор">интерпретатор</a> <a href="/wiki/%D0%AF%D0%B7%D1%8B%D0%BA_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F" title="Язык программирования">языка программирования</a> <a class="mw-redirect" href="/wiki/META_(%D1%8F%D0%B7%D1%8B%D0%BA_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F)" title="META (язык программирования)">META</a>, который можно использовать для создания графических иллюстраций. MetaPost был создан <a class="new" href="/w/index.php?title=%D0%A5%D0%BE%D0%B1%D0%B1%D0%B8,_%D0%94%D0%B6%D0%BE%D0%BD&amp;action=edit&amp;redlink=1" title="Хобби, Джон (страница отсутствует)">Джоном Хобби</a> в то время, когда он был аспирантом у <a href="/wiki/%D0%9A%D0%BD%D1%83%D1%82,_%D0%94%D0%BE%D0%BD%D0%B0%D0%BB%D1%8C%D0%B4_%D0%AD%D1%80%D0%B2%D0%B8%D0%BD" title="Кнут, Дональд Эрвин">Дональда Кнута</a>. В качестве основы была взята система создания шрифтов <a href="/wiki/METAFONT" title="METAFONT">METAFONT</a><sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup>.
</p><p>На входе интерпретатору подаётся текст на META, а на выходе получается графический файл в формате <a href="/wiki/PostScript" title="PostScript">PostScript</a><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>. Начиная с версии 1.200 MetaPost поддерживает в качестве выходного формата <a href="/wiki/SVG" title="SVG">SVG</a>-графику<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup>.
</p><p>Язык META, унаследованный от METAFONT, позволяет оперировать геометрическими объектами, такими как: точка, путь, картинка и выполнять над ними различные алгебраические действия, например, <a href="/wiki/%D0%9F%D0%B0%D1%80%D0%B0%D0%BB%D0%BB%D0%B5%D0%BB%D1%8C%D0%BD%D1%8B%D0%B9_%D0%BF%D0%B5%D1%80%D0%B5%D0%BD%D0%BE%D1%81" title="Параллельный перенос">сдвиг</a>, <a href="/wiki/%D0%9F%D0%BE%D0%B2%D0%BE%D1%80%D0%BE%D1%82" title="Поворот">вращение</a> и другие <a class="mw-redirect" href="/wiki/%D0%9B%D0%B8%D0%BD%D0%B5%D0%B9%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%B5%D0%BE%D0%B1%D1%80%D0%B0%D0%B7%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Линейное преобразование">линейные преобразования</a>.
</p><p>Основными отличиями MetaPost от METAFONT кроме выходного формата является наличие поддержки цвета и возможность делать текстовые вставки. Текстовые вставки создаются с помощью <a href="/wiki/TeX" title="TeX">TeX</a>, таким образом, любая конструкция, которая может быть создана в TeX, также может быть вставлена в картинку MetaPost. Кроме этого изначально автор MetaPost <a class="new" href="/w/index.php?title=%D0%A5%D0%BE%D0%B1%D0%B1%D0%B8,_%D0%94%D0%B6%D0%BE%D0%BD&amp;action=edit&amp;redlink=1" title="Хобби, Джон (страница отсутствует)">Джон Хобби</a> разработал библиотеку METAOBJ (“metapost Objects”) для визуализации двумерных графиков<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup><sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup>.
</p><p>Интерпретатор MetaPost (исполняемый файл mpost) вместе со стандартными макро-библиотеками распространяется как <a href="/wiki/%D0%9E%D1%82%D0%BA%D1%80%D1%8B%D1%82%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Открытое программное обеспечение">открытое программное обеспечение</a>, обычно, в составе дистрибутивов TeX.
</p>

<h2><span id="MetaPost-.D0.BA.D0.BE.D0.BD.D0.B2.D0.B5.D0.B9.D0.B5.D1.80"></span><span class="mw-headline" id="MetaPost-конвейер">MetaPost-конвейер</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=MetaPost&amp;veaction=edit&amp;section=1" title="Редактировать раздел «MetaPost-конвейер»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MetaPost&amp;action=edit&amp;section=1" title="Редактировать код раздела «MetaPost-конвейер»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<figure typeof="mw:File/Thumb"><a class="mw-file-description" href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB:MetaPost_%D0%BA%D0%BE%D0%BD%D0%B2%D0%B5%D0%B9%D0%B5%D1%80.png"><img class="mw-file-element" data-file-height="251" data-file-width="350" decoding="async" height="215" src="//upload.wikimedia.org/wikipedia/ru/thumb/8/8f/MetaPost_%D0%BA%D0%BE%D0%BD%D0%B2%D0%B5%D0%B9%D0%B5%D1%80.png/300px-MetaPost_%D0%BA%D0%BE%D0%BD%D0%B2%D0%B5%D0%B9%D0%B5%D1%80.png" srcset="//upload.wikimedia.org/wikipedia/ru/8/8f/MetaPost_%D0%BA%D0%BE%D0%BD%D0%B2%D0%B5%D0%B9%D0%B5%D1%80.png 1.5x" width="300"/></a><figcaption>MetaPost-конвейер</figcaption></figure>
<p>На вход программы <tt>mpost</tt> подаётся «META-картинка». «META-картинка» — это текстовый файл с расширением <tt>.mp</tt> (далее для краткости mp-файл) с инструкциями на языке META. В одном mp-файле можно хранить несколько описаний картинок. При компиляции с помощью <tt>mpost</tt> создаются файлы с тем же именем, что и у исходного файла, но с расширениями в виде чисел, которые указываются в декларации <code>beginfig</code>. Результирующие файлы сразу можно вставлять в LaTeX-тексты с помощью обычного <code>\includegraphics</code>. Для этого достаточно в заголовок tex-файла добавить команду из LaTeX-пакета graphicx:
</p>
<div class="mw-highlight mw-highlight-lang-latex mw-content-ltr" dir="ltr"><pre><span></span><span class="k">\DeclareGraphicsRule</span><span class="nb">{</span>*<span class="nb">}{</span>eps<span class="nb">}{</span>*<span class="nb">}{}</span>
</pre></div>
<p>От «правильных» eps-файлов они отличаются только тем, что в них не «внедрены» шрифты, поэтому просмотреть их без дополнительной обработки не удастся.
</p><p>Шрифты можно внедрить посредством программ <tt>latex</tt> и <tt>dvips</tt> с результатом в виде eps-файла или скрипта <tt>mptopdf</tt> с результатом в виде pdf-файла. Эти картинки уже можно использовать независимо любой программой, которая поддерживает эти векторные форматы.
</p>
<h3><span id=".D0.9A.D0.B8.D1.80.D0.B8.D0.BB.D0.BB.D0.B8.D1.86.D0.B0_.D0.B8_MetaPost"></span><span class="mw-headline" id="Кириллица_и_MetaPost">Кириллица и MetaPost</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=MetaPost&amp;veaction=edit&amp;section=2" title="Редактировать раздел «Кириллица и MetaPost»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MetaPost&amp;action=edit&amp;section=2" title="Редактировать код раздела «Кириллица и MetaPost»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Внедрить кириллицу в метки MetaPost можно только с помощью LaTeX. Для этого mp-файл должен иметь примерно следующий заголовок:
</p>
<div class="mw-highlight mw-highlight-lang-latex mw-content-ltr" dir="ltr"><pre><span></span>verbatimtex 
<span class="k">\documentclass</span><span class="na">[12pt]</span><span class="nb">{</span>minimal<span class="nb">}</span> 
<span class="c">%простейшая кириллизация</span>
<span class="k">\usepackage</span><span class="na">[koi8-r]</span><span class="nb">{</span>inputenc<span class="nb">}</span>

etex;

<p>Этот заголовок будет использоваться каждый раз, когда MetaPost доходит до текста, находящегося между метками <code>btex</code> и <code>etex</code>. Если для создание метки требуется какой-либо пакет LaTeX, то, соответственно, необходимо добавить этот пакет в заголовок стандартным образом.
</p><p>Для того, чтобы при создании надписи использовался именно <tt>latex</tt>, интерпретатор <tt>mpost</tt> должен запускаться с опцией <tt>-tex=latex</tt>. Если эта опция отсутствует, то информацию о том, что следует запускать, <tt>mpost</tt> ищет в переменной окружения <tt>TEX</tt>. По умолчанию вместо <tt>latex</tt> запускается <tt>tex</tt>.
</p><p>Если в тексте определяется переменная <code>prologues</code>, то она должна быть равна 0. В этом случае все необходимые шрифты «подшиваются» к картинке в момент, когда создаются eps и pdf-файлы.
</p>

</p>

</p>
<div class="mw-highlight mw-highlight-lang-bash mw-content-ltr" dir="ltr"><pre><span></span>make<span class="w"> </span>&lt;имя<span class="w"> </span>mp-файла&gt;.&lt;номер<span class="w"> </span>картинки&gt;.<span class="o">[</span>eps<span class="p">|</span>pdf<span class="o">]</span>

</p><p>Скрипт для каждого <code>beginfig(n)</code>-блока сделает файлы <tt>filen.eps</tt> и <tt>filen.pdf</tt>, где <tt>file</tt> — имя исходного MetaPost-файла, <i>n</i> — номер блока. <h2><span id=".D0.AF.D0.B7.D1.8B.D0.BA_META"></span><span class="mw-headline" id="Язык_META">Язык META</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=MetaPost&amp;veaction=edit&amp;section=5" title="Редактировать раздел «Язык META»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MetaPost&amp;action=edit&amp;section=5" title="Редактировать код раздела «Язык META»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>В качестве базового языка, инструкции которого подаются на вход программы MetaPost, используется язык <a class="new" href="/w/index.php?title=META&amp;action=edit&amp;redlink=1" title="META (страница отсутствует)">META</a><sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup>.
</p><p>В MetaPost можно оперировать следующими типами данных:
</p>
<ul><li>boolean — логический (Истина/Ложь)</li>
<li>numeric — обычные числа</li>
<li>pen (перо) — то, чем компьютер рисует (в подавляющем большинстве случаев используется круглое перо <code>pencircle</code>)</li>
<li>pair (точка) — пара чисел (x, y) в случае декартовых координат или R*dir(α) в случае полярных координат</li>
<li>path (путь) — набор точек с описанием типа соединений между ними</li>
<li>color (цвет) — тройка чисел (r, g, b) соответствует <a href="/wiki/%D0%A6%D0%B2%D0%B5%D1%82%D0%BE%D0%B2%D0%B0%D1%8F_%D0%BC%D0%BE%D0%B4%D0%B5%D0%BB%D1%8C" title="Цветовая модель">цветовой модели</a> <a href="/wiki/RGB" title="RGB">RGB</a></li>
<li>picture (картинка) — совокупность путей и точек</li>
<li>string (строка) — ASCII строка,</li>
<li>transform (линейные преобразования) — линейные преобразования, которые можно применять к объектам типа <code>pair</code>, <code>pen</code>, <code>path</code> и <code>picture</code>.</li></ul>
<p>Имена переменных в META могут состоять из нескольких лексем. Лексемы могут быть либо буквенными, либо числовыми. Например, переменная <code>x1l</code> состоит из трёх лексем. Её можно переписать более понятным способом <code>x[1].l</code>, то есть числовая лексема по сути указывает на номер элемента в массиве, а следующая за ней буква уточняет элемент структуры. Возможность опускать «[].» в написании имён переменных упрощает в некоторых случаях восприятие кода (например, <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle x_{1l}}" xmlns="http://www.w3.org/1998/Math/MathML">

Взамен, если нужны просто переменные без подобных особенностей, то придётся ограничиться только буквенными комбинациями.
</p><p>Все переменные необходимо объявлять перед использованием. Исключением являются переменные типа <code>numeric</code>. Массивы объявляются и используются следующим образом:
</p>
<div class="mw-highlight mw-highlight-lang-latex mw-content-ltr" dir="ltr"><pre><span></span> pair w[];

 w2:=w1;
</pre></div>
<p>Взаимодействие переменных, чисел и операторов вполне естественно, но достаточно нетривиально. Описание этого достойно отдельного раздела. В любом случае следует действовать по правилу: если сомневаетесь, то расставляйте скобки в нужных местах.
</p><p>В META можно опускать некоторые из операторов для сокращения записей, например, <code>2*x</code> соответствует записи <code>2x</code>. При этом <code>1/2x</code> — это <code>0.5x</code>, что более естественно с точки зрения математики, но не программирования. В META сначала обрабатываются числовые лексемы.

</p><p>Программу <tt>mpost</tt> можно использовать в режиме калькулятора для вычислений на языке META. Это позволяет проверить правильность ваших предположений относительно языка. Пример сеанса представлен ниже:
</p>
<div class="mw-highlight mw-highlight-lang-text mw-content-ltr" dir="ltr"><pre><span></span> baldin@evgueni:~$ mpost
 This is MetaPost, Version 0.901 (Web2C 7.5.5)
 **\relax
 *a:=10;
 *b:=8;
 *c:=a+-+b;
 *show c;
 &gt;&gt; 6
 *show (3-sqrt 5)/2;
 &gt;&gt; 0.38197
 *show angle(1,sqrt 3);
 &gt;&gt; 60.00008
 *show 2**10;
 &gt;&gt; 1024.00003
 *show infinity;
 &gt;&gt; 4095.99998
 *show epsilon;
 &gt;&gt; 0.00002
 *show infinity-infinity;
 &gt;&gt; 0
 *end
 Transcript written on mpout.log.
</pre></div>
<p>После вывода приглашения <code>**</code> следует набрать команду <code>\relax</code>. Далее можно вводить команды MetaPost. Делать это надо аккуратно, так как этот режим не поддерживает «истории команд». В начале не предполагалось, что MetaPost можно использовать и так тоже. С помощью команды <code>show</code> можно вывести результат на экран. Закончить сеанс можно с помощью команды <code>end</code>. Обратите внимание, что на просьбу вывести бесконечность (<code>infinity</code>) MetaPost выдал 4095.99998 — это максимальное значение, которое может принимать переменная типа <code>numeric</code>. Причём в процессе вычисления результат может превышать «бесконечность», но ответ должен быть меньше или равен ей, иначе будет выдана ошибка. Минимальный шаг изменения типа <code>numeric</code> равен <code>epsilon</code>, или, точнее, 1/256/256. При создании рисунка эти ограничения не существенны, так как диапазон изменения чисел вполне велик, чтобы вместить все элементы. Но в любом случае это тоже необходимо учитывать.
</p><p>Если необходимо вычислить однострочное выражение, то на первоначальное приглашение <code>**</code> можно ввести <code>expr</code>. В этом случае <tt>mpost</tt> считает файл <tt>expr.mf</tt> и на любое действие будет выдаваться ответ:
</p>
<div class="mw-highlight mw-highlight-lang-text mw-content-ltr" dir="ltr"><pre><span></span> baldin@evgueni:~$ mpost
 This is MetaPost, Version 0.901 (Web2C 7.5.5)
 **expr
 (/usr/local/texlive/2005/texmf-dist/metafont/base/expr.mf
 gimme an expr: 2(a+3b)-2b
 &gt;&gt; 4b+2a
 gimme an expr: 1/3[a,b]
 &gt;&gt; 0.33333b+0.66667a
</pre></div>

<div class="gallerytext">Распределение материи во Вселенной. Пример <a class="mw-redirect" href="/wiki/%D0%9A%D1%80%D1%83%D0%B3%D0%BE%D0%B2%D0%B0%D1%8F_%D0%B4%D0%B8%D0%B0%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B0" title="Круговая диаграмма">круговой диаграммы</a></div>

<p>Код каждого примера приведён в описании соответствующей картинки.
</p><p>Известен пример нестандартного применения программы.  С 1996 по 1999 год авторы спелеопрограммы Therion<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> написали DSL METAPOST для преобразования 2D-карт в 3D-модели пещер, а все подписи реализовали в формате TEX<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup>.
</p>
<h2><span id=".D0.90.D0.BD.D0.B0.D0.BB.D0.BE.D0.B3.D0.B8"></span><span class="mw-headline" id="Аналоги">Аналоги</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=MetaPost&amp;veaction=edit&amp;section=7" title="Редактировать раздел «Аналоги»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MetaPost&amp;action=edit&amp;section=7" title="Редактировать код раздела «Аналоги»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>MetaPost имеет некоторое количество ограничений, доставшихся ему в наследство от METAFONT. Попытка обойти эти ограничения легла в основу создания программного интерпретатора <a href="/wiki/Asymptote" title="Asymptote">Asymptote</a><sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup>. Язык, используемый Asymptote, похож на META, но вследствие перехода от синтаксиса макро-языка к синтаксису <a href="/wiki/C%2B%2B" title="C++">C++</a> гораздо более многословен и сложен. Основное преимущество Asymptote заключается в лучшей поддержке возможностей <a href="/wiki/PostScript" title="PostScript">PostScript</a>.
</p><p><a class="new" href="/w/index.php?title=Functional_MetaPost&amp;action=edit&amp;redlink=1" title="Functional MetaPost (страница отсутствует)">Functional MetaPost</a> — <a class="mw-redirect" href="/wiki/%D0%9F%D1%80%D0%B5%D0%B4%D0%BC%D0%B5%D1%82%D0%BD%D0%BE-%D0%BE%D1%80%D0%B8%D0%B5%D0%BD%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D1%8B%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F" title="Предметно-ориентированный язык программирования">DSL</a> для графики встроенный в <a href="/wiki/Haskell" title="Haskell">Haskell</a>, который генерирует код MetaPost.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup>
</p><p><a class="new" href="/w/index.php?title=METAGRAF&amp;action=edit&amp;redlink=1" title="METAGRAF (страница отсутствует)">METAGRAF</a> — <a href="/wiki/%D0%93%D1%80%D0%B0%D1%84%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81_%D0%BF%D0%BE%D0%BB%D1%8C%D0%B7%D0%BE%D0%B2%D0%B0%D1%82%D0%B5%D0%BB%D1%8F" title="Графический интерфейс пользователя">графический интерфейс</a> над MetaPost. Написан на <a href="/wiki/Java" title="Java">Java</a>. По возможностям напоминает <a href="/wiki/Xfig" title="Xfig">xfig</a>. Картинки сохраняет в формате MetaPost.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup>
</p><p>Среди программного окружения LaTeX аналогичной MetaPost функциональностью также обладают пакеты <a href="/wiki/PSTricks" title="PSTricks">PSTricks</a> и <a class="new" href="/w/index.php?title=PGF/TikZ&amp;action=edit&amp;redlink=1" title="PGF/TikZ (страница отсутствует)">PGF/TikZ</a>.
</p><p>На основе программной базы MetaPost был создан инструмент <a class="new" href="/w/index.php?title=METATYPE1&amp;action=edit&amp;redlink=1" title="METATYPE1 (страница отсутствует)">METATYPE1</a> для разработки Type1 шрифтов.
</p>
