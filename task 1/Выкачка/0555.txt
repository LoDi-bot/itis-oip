Эффективные алгоритмы для решения задачи дискретного логарифмирования в общем случае неизвестны.

В случае, когда <span class="ts-math"><i>G</i></span> является мультипликативной группой кольца вычетов по модулю <span class="ts-math"><i>m</i></span>, решение называют также <b>индексом</b> числа <span class="ts-math"><i>a</i></span> по основанию <span class="ts-math"><i>g</i></span>. Индекс числа <span class="ts-math"><i>a</i></span> по основанию <span class="ts-math"><i>g</i></span> гарантированно существует, если <span class="ts-math"><i>g</i></span> является <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B2%D0%BE%D0%BE%D0%B1%D1%80%D0%B0%D0%B7%D0%BD%D1%8B%D0%B9_%D0%BA%D0%BE%D1%80%D0%B5%D0%BD%D1%8C_(%D1%82%D0%B5%D0%BE%D1%80%D0%B8%D1%8F_%D1%87%D0%B8%D1%81%D0%B5%D0%BB)" title="Первообразный корень (теория чисел)">первообразным корнем</a> по модулю <span class="ts-math"><i>m</i></span>.
</p>

<h2><span id=".D0.92.D1.8B.D1.87.D0.B8.D1.81.D0.BB.D0.B8.D1.82.D0.B5.D0.BB.D1.8C.D0.BD.D0.B0.D1.8F_.D1.81.D0.BB.D0.BE.D0.B6.D0.BD.D0.BE.D1.81.D1.82.D1.8C_.D0.B8_.D0.BF.D1.80.D0.B8.D0.BB.D0.BE.D0.B6.D0.B5.D0.BD.D0.B8.D1.8F_.D0.B2_.D0.BA.D1.80.D0.B8.D0.BF.D1.82.D0.BE.D0.B3.D1.80.D0.B0.D1.84.D0.B8.D0.B8"></span><span class="mw-headline" id="Вычислительная_сложность_и_приложения_в_криптографии">Вычислительная сложность и приложения в криптографии</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%94%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%BD%D0%BE%D0%B5_%D0%BB%D0%BE%D0%B3%D0%B0%D1%80%D0%B8%D1%84%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5&amp;veaction=edit&amp;section=10" title="Редактировать раздел «Вычислительная сложность и приложения в криптографии»"><span>править</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%94%D0%B8%D1%81%D0%BA%D1%80%D0%B5%D1%82%D0%BD%D0%BE%D0%B5_%D0%BB%D0%BE%D0%B3%D0%B0%D1%80%D0%B8%D1%84%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5&amp;action=edit&amp;section=10" title="Редактировать код раздела «Вычислительная сложность и приложения в криптографии»"><span>править код</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Задача дискретного логарифмирования является одной из основных задач, на которых базируется <a class="mw-redirect" href="/wiki/%D0%9A%D1%80%D0%B8%D0%BF%D1%82%D0%BE%D0%B3%D1%80%D0%B0%D1%84%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0_%D1%81_%D0%BE%D1%82%D0%BA%D1%80%D1%8B%D1%82%D1%8B%D0%BC_%D0%BA%D0%BB%D1%8E%D1%87%D0%BE%D0%BC" title="Криптографическая система с открытым ключом">криптография с открытым ключом</a>. Классическими криптографическими схемами на её основе являются схема выработки общего ключа <a class="mw-redirect" href="/wiki/%D0%90%D0%BB%D0%B3%D0%BE%D1%80%D0%B8%D1%82%D0%BC_%D0%94%D0%B8%D1%84%D1%84%D0%B8-%D0%A5%D0%B5%D0%BB%D0%BB%D0%BC%D0%B0%D0%BD%D0%B0" title="Алгоритм Диффи-Хеллмана">Диффи-Хеллмана</a><sup class="reference" id="cite_ref-_08b07f1facd8ea80_10-0"><a href="#cite_note-_08b07f1facd8ea80-10">[10]</a></sup>, <a href="/wiki/%D0%A1%D1%85%D0%B5%D0%BC%D0%B0_%D0%AD%D0%BB%D1%8C-%D0%93%D0%B0%D0%BC%D0%B0%D0%BB%D1%8F" title="Схема Эль-Гамаля">схема электронной подписи Эль-Гамаля</a><sup class="reference" id="cite_ref-_1e0a99d83fe29b44_11-0"><a href="#cite_note-_1e0a99d83fe29b44-11">[11]</a></sup><sup class="reference" id="cite_ref-_9096049fb2527be9_12-0"><a href="#cite_note-_9096049fb2527be9-12">[12]</a></sup>, <a class="mw-redirect" href="/wiki/%D0%9A%D1%80%D0%B8%D0%BF%D1%82%D0%BE%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0_%D0%9C%D1%8D%D1%81%D1%81%D0%B8-%D0%9E%D0%BC%D1%83%D1%80%D1%8B" title="Криптосистема Мэсси-Омуры">криптосистема Мэсси-Омуры</a><sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> для передачи сообщений. Их криптостойкость основывается на предположительно высокой <a href="/wiki/%D0%92%D1%8B%D1%87%D0%B8%D1%81%D0%BB%D0%B8%D1%82%D0%B5%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D1%81%D0%BB%D0%BE%D0%B6%D0%BD%D0%BE%D1%81%D1%82%D1%8C" title="Вычислительная сложность">вычислительной сложности</a> обращения показательной функции. Хотя сама показательная функция вычисляется достаточно эффективно, даже самые современные алгоритмы вычисления дискретного логарифма имеют очень высокую сложность, которая сравнима со сложностью наиболее быстрых алгоритмов <a href="/wiki/%D0%A4%D0%B0%D0%BA%D1%82%D0%BE%D1%80%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F_%D1%86%D0%B5%D0%BB%D1%8B%D1%85_%D1%87%D0%B8%D1%81%D0%B5%D0%BB" title="Факторизация целых чисел">разложения чисел на множители</a>.
</p><p>Другая возможность эффективного решения задачи вычисления дискретного логарифма связана с <a href="/wiki/%D0%9A%D0%B2%D0%B0%D0%BD%D1%82%D0%BE%D0%B2%D1%8B%D0%B9_%D0%BA%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80" title="Квантовый компьютер">квантовыми вычислениями</a>. Теоретически доказано, что с помощью <a href="/wiki/%D0%90%D0%BB%D0%B3%D0%BE%D1%80%D0%B8%D1%82%D0%BC_%D0%A8%D0%BE%D1%80%D0%B0" title="Алгоритм Шора">алгоритма Шора</a> дискретный логарифм можно вычислить за полиномиальное время<sup class="reference" id="cite_ref-_2fbb03e82ec651aa_14-0"><a href="#cite_note-_2fbb03e82ec651aa-14">[14]</a></sup><sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup><sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup>. В любом случае, если полиномиальный алгоритм вычисления дискретного логарифма будет реализован, это будет означать практическую непригодность криптосистем на его основе для долговременной защиты данных. Рассматривается ряд идей для создания <a href="/wiki/%D0%9F%D0%BE%D1%81%D1%82%D0%BA%D0%B2%D0%B0%D0%BD%D1%82%D0%BE%D0%B2%D0%B0%D1%8F_%D0%BA%D1%80%D0%B8%D0%BF%D1%82%D0%BE%D0%B3%D1%80%D0%B0%D1%84%D0%B8%D1%8F" title="Постквантовая криптография">новых алгоритмов с открытым ключом</a>.
</p>
